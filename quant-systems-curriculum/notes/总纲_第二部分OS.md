## 第二部分 · 操作系统（Linux）学习路线（总纲）

> 这是「量化系统底层修炼」专栏 OS 部分的导览根页。下面每个阶段是一棵子树，子课时挂在对应阶段下。

### 为什么 Linux 内核功底是低延迟量化的另一道门槛

C++ 决定单线程算得多快，**OS 决定整条 tick-to-trade 路径抖不抖**。低延迟交易的延迟杀手大多在系统层：中断、缺页、调度抢占、cache/TLB 失效、NUMA 跨节点、频率波动。把这些摁住，才谈得上稳定的 P99.9。

- **C 档 · 研究员**：达到《Linux 高性能服务器编程》水平，会用 epoll/mmap/socket 即可。
- **B 档 · HPC/平台**：懂 NUMA / cache / 虚存体系结构，会用 perf/火焰图定位瓶颈。
- **A 档 · 低延迟核心**：啃内核——绑核隔离、内核旁路、抖动控制，把交易核从调度器手里抢出来。

**分水岭在 O4「中断与内核旁路」**——kernel bypass 几乎是顶级低延迟岗的门槛标志。

### 八个阶段（父设计，广度优先）

| 阶段 | 主题 | 难度峰值 | 一句话 |
|---|---|---|---|
| O1 | Linux 系统编程地基 | 🟢 | 进程/线程 / socket / epoll / mmap，研究员关到此 |
| O2 | 进程调度与 CPU | 🔴 | 绑核 isolcpus / nohz_full / NUMA，把交易核抢出来 |
| O3 | 内存管理 | 🔴 | HugePages / mlockall 预热 / 运行期禁 malloc |
| O4 | 中断与内核旁路 | ⚫ | **A 档分水岭**：kernel bypass / DPDK / Onload / busy-poll |
| O5 | 网络深入 | 🔴 | 组播行情 / 零拷贝 / 硬件时间戳 / PTP 对时 |
| O6 | IO 与存储 | 🟡 | 日志异步化，热路径绝不同步写盘 |
| O7 | 性能分析 | 🔴 | perf / 火焰图 / ftrace / eBPF / cyclictest |
| O8 | 实时性与抖动控制 | ⚫ | 抖动全谱消除，tail latency P99.9 思维 |

### OS 侧分水岭题

1. kernel bypass 为什么省延迟、省了哪些开销（O4）
2. 如何精确测量 tick-to-trade 延迟（O5 硬件时间戳）
3. 延迟尖峰溯源：缺页 / 中断 / 调度，怎么定位（O7）
