## 第一部分 · C++ 学习路线（总纲）

> 这是「量化系统底层修炼」专栏 C++ 部分的导览根页。下面每个阶段是一棵子树，子课时挂在对应阶段下。

### 为什么 C++ 是低延迟量化的硬门槛

量化对 C++ 的要求按岗位分三档，深度天差地别：

- **C 档 · 量化研究员**：现代 C++「会用、写得干净高效」即可，多数研究员到此为止。
- **B 档 · HPC/平台**：模板/SIMD/内存池「重工程 + 性能」，平台组核心。
- **A 档 · 低延迟核心**：无锁/内存模型/纳秒优化「天花板」，幻方/九坤硬核交易 infra。

**分水岭在 C4「内存模型 + 无锁队列」**——过不了这关做不了交易系统。

### 六个阶段（父设计，广度优先）

| 阶段 | 主题 | 难度峰值 | 一句话 |
|---|---|---|---|
| C1 | 现代 C++ 地基 | 🟢 | RAII / 移动语义 / noexcept，研究员关到此 |
| C2 | 模板与泛型 | 🟡→🔴 | constexpr 是 zero-cost abstraction 核心抓手 |
| C3 | 对象模型与底层 | 🟡→🔴 | vtable / CRTP 静态多态 / 编译链接 |
| C4 | 内存模型与并发 | ⚫ | **A 档分水岭**：六种 memory_order + 手写无锁队列 |
| C5 | 性能优化 | 🔴 | 热路径零分配 / branchless / SIMD / 延迟测量 |
| C6 | 工具链与库 | 🔴 | perf / TSan / STL 内部 / folly |

### 三大必刷高频题

1. 手写无锁 SPSC 环形队列（C4）
2. 内存模型 / false sharing / cache line（C4）
3. STL 内部：vector 扩容、为什么不用 std::map、移动语义（C6）
