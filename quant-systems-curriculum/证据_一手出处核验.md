## 1. 无锁队列 / cache line / false sharing

**声明**（大纲 C4-20/21、O3-15）：cache line 64 字节；`alignas(64)` 减少 cache coherency traffic；用 `std::hardware_destructive_interference_size`。

- **出处**：Erik Rigtorp,《Correctly implementing a ring buffer》https://rigtorp.se/ringbuffer/
- **原文摘录**："read (readIdx_) and write (writeIdx_) indices are aligned to the size of a cache line (alignas(64))... to reduce cache coherency traffic. On AMD64 / x86_64 and ARM a cache line is 64 bytes... using std::hardware_destructive_interference_size is a good choice if it's available."
- **作者权威性**：Rigtorp 是业界广泛引用的无锁 SPSC 队列实现作者，其 rigtorp/SPSCQueue 是 C++ 低延迟圈事实参考实现。

**声明**（大纲 C4-22 Disruptor、C4-20 false sharing）：LMAX Disruptor 思想；false sharing 是性能杀手。

- **出处**：LMAX 官方技术白皮书 https://lmax-exchange.github.io/disruptor/disruptor.html （PDF 同源 GitHub LMAX-Exchange/disruptor）
- **原文摘录（false sharing）**："independent, but concurrently written, variables do not share the same cache-line if contention is to be minimised... This is a concept known as 'false sharing'."
- **原文摘录（性能硬数据）**："we have seen over 25 million messages per second and latencies lower than 50 nanoseconds... very close to the theoretical limit of a modern processor."
- **原文摘录（ring buffer 预分配）**："All memory for the ring buffer is pre-allocated on start up."（对应大纲 C5-28「热路径禁 new/malloc」「预分配」）
- **意义**：大纲「了解 LMAX Disruptor 思想加分」「无锁队列是标配」有一手出处；50ns/25M msg/s 是该领域被反复引用的标杆数字。

## 2. Linux 绑核 / CPU 隔离 / 抖动控制

**声明**（大纲 O2-8/9）：`isolcpus`、`nohz_full`、`sched_setaffinity` 把交易核从调度器抢出来。

- **出处**：Linux 官方内核参数文档 https://www.kernel.org/doc/html/latest/admin-guide/kernel-parameters.html
- **原文确认**：`isolcpus=` 与 `nohz_full=`（"nohz_full=all" 等价 "nohz_full=0-N"）均为官方记载的 boot 参数，语法原文可核验。
- **配套**：man7 `sched_setaffinity(2)` https://man7.org/linux/man-pages/man2/sched_setaffinity.2.html 官方手册页确认绑核 API。

**声明**（大纲 O3-13 HugePages、O3-14 mlockall 预热）：大页降 TLB miss；`mlockall` 锁页防 swap + 预热全内存。

- **出处**：Linux 官方 HugeTLB 文档 https://www.kernel.org/doc/html/latest/admin-guide/mm/hugetlbpage.html （HTTP 200 实测）
- **配套 API**：`mlockall(2)` man7 手册页确认锁定进程地址空间防换出。
