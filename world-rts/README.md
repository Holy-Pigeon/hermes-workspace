# World-RTS · 世界即一局 RTS 的全局态势分析引擎

## 一句话定位
把真实世界当成一局大型即时战略(RTS)游戏来建模：资源、资金、人口、科技、战争、领地一一对应。
目标不是"讲一个酷比喻"，而是**建立一张可校准、可证伪的全局态势矩阵**，让"资金/政策会被迫涌向哪里"这件事从拍脑袋变成从数据里自动浮现的结论。

## 核心信念（RTS 铁律 → 投资映射）
- **你的发展速度由最稀缺的资源决定，不由最强的资源决定。** → 资金/政策会被迫涌向缓解"绑定约束(binding constraint)"的方向；这种流向是被迫的、可预测的、对价格不敏感的——正是价值投资者最爱的确定性指向。
- **瓶颈不是分析师指认的，是数据里那个最低分自己浮出来的。** → 拒绝认知投影（我熟能源就只写能源），必须把每种资源都量到同一把 0–100 的刻度尺上，最低分才有意义。

## 项目阶段（严格分期，禁止跨期抢跑）
- **第一阶段（当前）：只建全局视野。** 交付物 = `10 阵营 × 22 维度` 的状态矩阵，每格挂硬数据 + 归一化分 + 一手来源，瓶颈列由公式自动算出。**这一阶段绝不跳到挑标的**——地基没夯完不砌墙。
- **第二阶段（未来）：读资金流。** 每条绑定约束 → 被迫的资金/政策流 → 叠加地理咽喉图 → 找"不可替代且路过少数供应商"的持久利润池。
- **第三阶段（未来）：落标的。** 每条流追 2–3 家"绕不开"的公司，走估值三情景全套，进 StockChoose 研究池。

## 数据严谨性铁律（继承 SOUL）
- 每个 `(阵营 × 维度)` 单元格必须挂**一手来源**（EIA/FAO/USGS/IMF/SIPRI/WHO 等），零编造，查不到标空。
- 区分"已核实硬数据"与"基于推断/估算"，后者显式标注。
- 一手文件 > 媒体口径；多源打架本身就是存疑信号，去源头终结矛盾。
