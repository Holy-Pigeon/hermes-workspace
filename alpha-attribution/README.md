# alpha-attribution · 呼叫α负读数的失败模式归因器

## 一句话
把 call-alpha-tracker 给出的**负呼叫α读数**机械拆成三类失败模式（方向错/时机错/选择错），回答系统最该问却一直回避的问题：**我们已下的研究呼叫，到底为什么在亏相对钱？**

## 为什么存在（元层能力缺口）
系统使命「超越巴菲特」=相对收益游戏。call-alpha-tracker 首跑（2026-06-23）给出负读数：12/13 进行中呼叫为负呼叫α、均值约 -3.19pp，StockChoose 选股口径 -4.45pp 跑输沪深300。但过去两周系统对这个负 verdict 的唯一回应是**继续加发现 lens**（quality-compounder / us-tech-scout / valuation-trigger），**没有任何一轮做归因**。这是确认偏误 + 缝补舒适区在架构层固化：用「再造更好的入口漏斗」回避「现有出口在亏钱」。本工具补的就是「发现-呼叫-计分-**归因**-改规则」闭环里缺的那一环。

## 三类失败模式（机械可判）
- **F1 方向错**：看空却跑赢基准 / 看多却跑输——典型是空在强势股（把「贵」当「该空」，但贵的票可以更贵）。
- **F2 时机错**：看多「质量/护城河/成长」派标的却跑输——买伟大公司付了过高价，reverse_dcf 隐含增速已 price-in 满仍进池。
- **F3 选择错**：看多「便宜/低估」派标的却跑输——低 PE 分位锚定选出真烂生意（便宜陷阱）。

## 首跑结论（2026-06-24）
| 模式 | 条数 | 累计呼叫α |
|---|---|---|
| F3 选择错(便宜陷阱) | 4 | **-16.8pp** ← 最痛 |
| F2 时机错(好生意买贵) | 3 | -10.8pp |
| F1 方向错(空在强势股) | 3 | -4.4pp |

**最痛系统性偏差 = F3 便宜陷阱**：坐实了系统被怀疑已久的深度价值锚定漏在出口侧。改规则方向：低 PE 分位是必要非充分条件，入池须叠加质量闸（ROE 持久性/毛利/现金含量）。

## ⚠️ 持有成熟度门 M0（2026-07-01 新增·纠正时间维度确认偏误）
**元层复盘发现**：上面首跑(06-24)的 -16.8pp F3 处方、以及后续 06-25→07-01 连续 5 轮围绕 StockChoose v1.0 质量闸 / validate_pick veto 死结的争论，其驱动证据（负呼叫α）**在下诊断时全部持有期仅 16-19 天、成熟度仅 21-24%**——而这些逆向价值呼叫的论点窗口多在中报 8/31 才闭合。**用建仓 2-3 周的 mark-to-market 浮动读数去改选股规则 = 在时间维度犯确认偏误（拿早期噪声当已实现失败结论）。** 事实印证：截至 07-01 平均呼叫α已从负转正 +1.68pp（7/12 正），早期负读数确为噪声。

**修复（可逆·纯只读工具）**：
- `call_alpha.py` 输出新增 `held_days / window_days / maturity_pct`（持有期占「建仓→verify_by」窗口比例）。
- `attribute.py` 加成熟度门 `MATURITY_GATE=50`：负α但成熟度 <50% → 归 **M0 未成熟观察桶**，**不开 F1/F2/F3/F4 改规则处方**，只登记观察、等成熟度≥50% 或论点证伪日再归因。「最痛系统性偏差」评选与处方**只对已成熟负α**开。
- 实测(07-01)：当前 5 条负α(-27.5pp)全部成熟度 21-24% → 全隔离 M0，「已成熟归因 0 条 → 不足以改规则」。这解释了 validate_pick veto 死结为何卡 3 轮推不动：系统直觉（好生意买早需时间兑现）一直在抵抗一个用早期噪声开出的错误处方。

## 用法
```
/opt/homebrew/bin/python3 attribute.py            # 人读报告
/opt/homebrew/bin/python3 attribute.py --quiet    # 无负α则静默exit0(挂cron用)
/opt/homebrew/bin/python3 attribute.py --json     # 机读
```

## 诚实边界
- 纯只读，零独立取数：完全复用 `call-alpha-tracker/call_alpha.py --json`，任一源失败即跳过、绝不编造。
- 失败模式按 `direction` 派系 + 已实现 excess **机械分类**，是确定性规则不是主观叙事；direction→派系映射在 `attribute.py` 顶部显式声明。
- 呼叫α是**至今浮动相对收益**，非已结算业绩（论点多在 8/31 才证伪）。负读数是**过程信号不是终局定论**；成熟度门确保只有窗口过半的负α才驱动改规则，早期读数只观察不结论。
- 结论应回灌 StockChoose 选股规则 / research-pipeline 入池门槛，**不是再开新发现镜头**；但**只有已成熟(≥50%)的负α才有资格触发改规则**。

## 挂 cron（待拍板）
建议周度（周日，跟在 call-alpha-tracker 之后），`--quiet`：发现侧扩张被冻结期间，定时把失败模式分布的漂移 surface 出来。挂 cron 涉及定时基础设施，按可逆性铁律须用户拍板。
