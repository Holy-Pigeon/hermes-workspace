## 今日知识点：Tanh 函数（双曲正切）——把 Sigmoid 挪到原点两侧的零中心开关

### 所属类目
类目 6：感知机与多层感知机

### 问题入口
上一节我们认识了 Sigmoid，它能把任意实数压进 $(0, 1)$，看起来已经是个很好的"开关"了。那为什么早期的神经网络（尤其是 RNN、LSTM）几乎全都用 **Tanh** 而不是 Sigmoid 做隐藏层的激活？如果只是"再压一次数"，为什么不直接用 Sigmoid？这背后藏着一个关于**梯度效率**的关键差别。

### 朴素想法
"既然 Sigmoid 已经能把数压到一个有限区间、还是平滑可导的，那隐藏层就一律用 Sigmoid 好了。" 这是最自然的想法：一个非线性、可导、有界的函数，拿来做每一层的激活，反正都能训练。

### 卡住的地方
问题出在 Sigmoid 的**输出范围**：它永远是正的（介于 0 和 1 之间）。这意味着一层神经元的输出全部是正数，喂给下一层时，会引发一个隐蔽但要命的毛病——**梯度更新方向被"绑架"**。

具体地说，假设某个神经元的输入是 $z = \sum_i w_i x_i + b$。如果所有 $x_i$（即上一层的激活）都恒为正，那么在反向传播时，损失对每个权重 $w_i$ 的梯度是

$$
\frac{\partial L}{\partial w_i} = \frac{\partial L}{\partial z} \cdot x_i
$$

其中 $x_i > 0$。于是所有 $w_i$ 的梯度符号**完全由 $\frac{\partial L}{\partial z}$ 这一个共同因子决定**：要么全体一起变大，要么全体一起变小。参数无法"有的加、有的减"，只能走"之"字形（zig-zag）逼近最优，收敛更慢。这就是所谓的**非零中心（non-zero-centered）问题**。

### 1. 概念
**Tanh** 是双曲正切函数，把任意实数压进 $(-1, 1)$，定义为：

$$
\tanh(z) = \frac{e^{z} - e^{-z}}{e^{z} + e^{-z}}
$$

符号说明：$z$ 是输入（一个实数，通常是神经元的加权和 $\sum w_i x_i + b$）；$e$ 是自然常数 $\approx 2.718$。分子分母都用到 $e^{z}$ 和 $e^{-z}$，本质是把正向和反向的指数做了一个"归一化的差"。

它和 Sigmoid 的关系极其干净——Tanh 就是**被拉伸并平移后的 Sigmoid**：

$$
\tanh(z) = 2\,\sigma(2z) - 1
$$

其中 $\sigma(\cdot)$ 是 Sigmoid。也就是说：把输入放大 2 倍、过一次 Sigmoid、再把输出从 $(0,1)$ 线性映射到 $(-1,1)$，就得到了 Tanh。

**通俗理解**：Sigmoid 像一个"投票器"，输出 0 到 1 表示"赞成的程度"；而 Tanh 像一个"态度表"，输出 $-1$ 到 $1$，能同时表达"反对（负）、中立（0）、赞成（正）"。关键就在于它以 **0 为中心**——中立态度对应输出 0，而不是 Sigmoid 那样对应 0.5。但类比到此为止，真正的价值不在"能表达负数"，而在下面要讲的梯度性质。

### 2. 推导与证明
我们来算 Tanh 的导数，看它为什么好用，又在哪里会失效。

关键步骤：

1. **利用商法则求导**。设 $f(z) = \tanh(z) = \frac{e^{z}-e^{-z}}{e^{z}+e^{-z}}$。分子导数是 $e^{z}+e^{-z}$，分母导数是 $e^{z}-e^{-z}$。套商法则：

$$
\tanh'(z) = \frac{(e^{z}+e^{-z})^2 - (e^{z}-e^{-z})^2}{(e^{z}+e^{-z})^2}
$$

2. **化简分子**。用平方差 $(A^2 - B^2)$ 展开，或直接观察：分子 $= (e^z+e^{-z})^2 - (e^z-e^{-z})^2 = 4$（两个平方相减，交叉项抵消，只剩 $4\cdot e^z \cdot e^{-z} = 4$）。于是

$$
\tanh'(z) = \frac{4}{(e^{z}+e^{-z})^2}
$$

3. **写成自身的函数**。注意到 $1 - \tanh^2(z) = 1 - \frac{(e^z-e^{-z})^2}{(e^z+e^{-z})^2} = \frac{4}{(e^z+e^{-z})^2}$。因此得到那个漂亮的结论：

$$
\tanh'(z) = 1 - \tanh^2(z)
$$

**为什么这一步重要**：导数可以直接用**前向传播已经算好的** $\tanh(z)$ 值来表达，反向传播时无需重新算指数，省计算。这和 Sigmoid 的 $\sigma'(z) = \sigma(z)(1-\sigma(z))$ 是同一种"自洽"美感。

4. **比较最大梯度**。在 $z=0$ 处，$\tanh'(0) = 1 - 0 = 1$；而 Sigmoid 在 $z=0$ 处 $\sigma'(0) = 0.5 \times 0.5 = 0.25$。**Tanh 的最大梯度是 1，是 Sigmoid 的 4 倍**。梯度更大，意味着在激活的中间区域信号衰减更少，训练更快。这是 Tanh 优于 Sigmoid 的第二个理由（第一个是零中心）。

### 3. 来源与历史动机
- 论文：双曲函数本身是 18 世纪的数学（Lambert, Riccati 等人），并非某篇深度学习论文的发明。它进入神经网络主流的关键推动，来自 Yann LeCun 等人对激活函数选择的系统建议。
- 作者 / 文献：LeCun, Bottou, Orr, Müller，《Efficient BackProp》(1998, 收录于 *Neural Networks: Tricks of the Trade*)。文中明确建议隐藏层用零中心的 Tanh 而非 Sigmoid，以加速收敛。
- 时间：1998 年（该建议）；Tanh 在 1990s–2010s 早期 RNN/LSTM 中长期是默认激活。
- 链接：http://yann.lecun.com/exdb/publis/pdf/lecun-98b.pdf

历史背景：上世纪 90 年代到 2010 年代初，网络还不深，主要是浅层 MLP 和循环网络。当时的核心痛点是**训练慢、易发散**。LeCun 的 *Efficient BackProp* 系统总结了一堆工程经验，其中"输入和激活都应该零中心化"是反复强调的一条——Tanh 正是把这条原则落到激活函数上的产物。后来 LSTM（1997）的记忆单元更新也选用 Tanh 来把候选状态压到 $(-1,1)$，让记忆可正可负地累加。

### 4. 核心用途
用途一：**RNN / LSTM / GRU 的隐藏状态激活**。这是 Tanh 至今仍在用的主战场。LSTM 用 Tanh 生成"候选记忆" $\tilde{c}_t = \tanh(W_c[h_{t-1}, x_t] + b_c)$，并在输出门后再用一次 $\tanh(c_t)$ 把细胞状态压回 $(-1,1)$。零中心保证记忆能对称地增减。

用途二：**需要输出有正有负的场景**。比如某些回归头、自编码器的隐层、或强化学习里把动作压到 $[-1,1]$ 区间的策略网络输出层。

用途三：**作为门控/缩放信号的一部分**。在一些注意力变体和门控结构里，Tanh 被用来产生一个 $(-1,1)$ 的调制系数，比只能给正权重的 Sigmoid 表达力更强。

### 5. 反直觉点与常见误区
反直觉点：**Tanh 和 Sigmoid 本质是同一个函数**（差一个线性变换 $\tanh(z)=2\sigma(2z)-1$），形状一模一样，但仅仅"把输出中心从 0.5 挪到 0"这一个看似无关紧要的平移，就能显著改善训练动力学——因为它改变的不是表达能力，而是**梯度更新的几何方向**（避免了参数梯度同号导致的 zig-zag）。同一个函数，换个坐标原点，训练效率就不同，这非常反直觉。

常见误区：
- **误区一：以为 Tanh 彻底解决了梯度消失**。并没有。Tanh 依然会**饱和**：当 $|z|$ 很大时 $\tanh(z) \to \pm 1$，导数 $1-\tanh^2(z) \to 0$。深层网络里多个小于 1 的梯度连乘，照样指数衰减。这正是后来 ReLU 取代它成为深层前馈网络默认激活的原因。
- **误区二：把"零中心的输出"和"零中心的输入数据"混为一谈**。Tanh 保证的是**激活输出**零中心，让下一层受益；而对网络的原始输入做零中心化（减均值）是另一件事（数据预处理）。两者动机相通（都为了梯度方向健康），但作用位置不同。

### 6. 小实验
不用写完整训练，手算三个点就能感受它的性质：

1. **心算对称性**：$\tanh(0) = 0$，$\tanh(-z) = -\tanh(z)$（奇函数）。取 $z=1$：$\tanh(1) \approx 0.762$，那么 $\tanh(-1) \approx -0.762$。对比 Sigmoid：$\sigma(1)\approx 0.731$，$\sigma(-1)\approx 0.269$——Sigmoid 关于 0.5 对称，Tanh 关于 0 对称。

2. **验证导数公式**：取 $z=1$，$\tanh(1)\approx 0.762$，则 $\tanh'(1) = 1 - 0.762^2 \approx 1 - 0.581 = 0.419$。取 $z=0$，$\tanh'(0)=1$（最大）。取 $z=3$，$\tanh(3)\approx 0.995$，$\tanh'(3)\approx 1-0.990=0.010$——几乎饱和，梯度快没了。亲眼看到"两端梯度趋 0"。

3. **一行 Python 验证与 Sigmoid 的关系**：

```python
import numpy as np
z = np.array([-2.0, -0.5, 0.0, 0.5, 2.0])
tanh = np.tanh(z)
sig  = 1 / (1 + np.exp(-2*z))     # sigmoid(2z)
print(tanh)
print(2*sig - 1)                   # 应与 tanh 完全一致
```

两行输出会逐元素相等，直接印证 $\tanh(z) = 2\sigma(2z) - 1$。

### 7. 批判性思考
局限性：Tanh 的两端饱和是硬伤——一旦神经元被推到 $|z|$ 很大的区域，梯度接近 0，这个神经元就很难再被更新（"卡死"）。在深层前馈网络里，这个问题被 ReLU 家族（不饱和的正半轴、恒等梯度 1）大幅缓解，所以现代 Transformer 的 FFN 用的是 GELU/SwiGLU 而非 Tanh。Tanh 之所以在 RNN 里还活着，是因为循环结构需要一个**有界**的激活来防止隐藏状态在时间上无限放大——ReLU 的无界性在这里反而危险。

深度问题：既然零中心这么好，为什么 ReLU（输出恒为非负、明显不是零中心）反而在深层网络里全面胜出？提示：想想 ReLU 用什么代价换来了什么——它牺牲了零中心和处处可导，换来了正半轴上**恒定不衰减的梯度 1**。这说明"避免梯度消失"比"零中心"在深层网络里是更优先的矛盾。而 Batch/Layer Normalization 的出现，又从另一条路补上了零中心这一课（把每层输入重新拉回零均值），让 ReLU 的非零中心缺陷不再致命。这三者（激活的零中心、不饱和、归一化）如何分工，是理解现代深度网络设计的一条主线。

### 8. 和后续知识的连接
- **向前**：Tanh 是 Sigmoid（上一节）的直接延伸，两者共享 S 形与饱和特性；它俩的饱和缺陷，是理解为什么后来要用 ReLU、以及为什么需要 Batch Normalization / Layer Normalization 的动机源头。
- **向 RNN/LSTM/GRU**：我们已经学过 LSTM、GRU，它们的候选状态和记忆压缩正是用 Tanh 实现的——现在你能明白为什么门控用 Sigmoid（$(0,1)$ 当"开关比例"）、而候选值用 Tanh（$(-1,1)$ 让记忆可正可负）。
- **向 Transformer**：Transformer 的 FFN 早期用 ReLU、如今用 GELU/SwiGLU，正是因为深层堆叠让 Tanh 的饱和问题不可接受。理解 Tanh 的局限，就理解了激活函数演进（Sigmoid → Tanh → ReLU → GELU → SwiGLU）这条主线为什么会发生。此外，GELU 的一个常用近似式里正好含有 $\tanh$，Tanh 以另一种形式活在了现代大模型里。
