## 今日知识点：LogSumExp 技巧（Log-Sum-Exp）——让 Softmax 和交叉熵在电脑里不溢出的稳定器

### 所属类目
类目 4：信息论基础

### 问题入口
你已经学过 Softmax（把一堆分数变成概率）和交叉熵（衡量预测和真值差多远）。它们的公式里都藏着一个 $\exp$（指数）。现在把它放进真实的计算机里跑一下：

假设模型某一步给三个词打的分（logits）是 $z = [1000, 1001, 1002]$。按 Softmax 定义：

$$
\text{softmax}(z)_i = \frac{e^{z_i}}{\sum_j e^{z_j}}
$$

你需要算 $e^{1000}$。但 64 位浮点数能表示的最大值大约是 $1.8 \times 10^{308}$，而 $e^{1000} \approx 10^{434}$——直接**溢出成 `inf`**。接下来 $\frac{\text{inf}}{\text{inf}} = \text{NaN}$（非数），整个训练就此崩掉。

反过来，如果 logits 是 $[-1000, -1001, -1002]$，$e^{-1000}$ 会**下溢成 0**，分母变成 $0$，又是 $\frac{0}{0} = \text{NaN}$。

问题很尖锐：**Softmax 的数学定义是完全正确的，但照着定义写代码，遇到大数或小数就会算出垃圾。** 我们需要一个既不改变结果、又不会溢出的算法。这就是 LogSumExp 技巧要解决的事。

### 朴素想法
最自然的想法有两个：

1. **"数值不会那么大吧？"**——先跑起来再说。但深层网络里 logits 动辄几十上百，训练不稳定时冲到几千是常事，赌它不溢出等于埋雷。
2. **"那我用更高精度的浮点数？"**——从 float32 换成 float64。但这只是把溢出的门槛从约 $e^{88}$ 推到约 $e^{709}$，logits 再大一点照样爆，而且内存和速度翻倍。治标不治本。

### 卡住的地方
朴素办法的根本问题在于：它们都试图**先算出 $e^{z_i}$ 这个可能巨大的中间量**，再去做除法。溢出发生在中间步骤，而不是最终结果——最终的概率明明都是 $0$ 到 $1$ 之间的正常数字，是我们自己在半路上把它算炸了。

关键观察：Softmax 的**结果**对 logits 的整体平移是不变的（下面会证明）。也就是说 $[1000, 1001, 1002]$ 和 $[0, 1, 2]$ 给出完全一样的概率。既然结果一样，我们为什么非要用那个会溢出的版本去算？**如果能在取指数之前，先把 logits 整体挪到一个安全的区间，就能既保住正确结果、又躲开溢出。** LogSumExp 正是把这个"先平移"的思想制度化。

### 1. 概念
**LogSumExp（简称 LSE）** 指的是这个函数：

$$
\text{LSE}(z_1, \dots, z_n) = \log \sum_{i=1}^{n} e^{z_i}
$$

它读作"对一堆数先取指数、求和、再取对数"。符号含义：$z_i$ 是第 $i$ 个输入（比如一个 logit），$e^{z_i}$ 是它的指数，$\sum$ 是求和，$\log$ 是自然对数。

**为什么它重要？** 因为 Softmax 和交叉熵在计算机里，最终都归结为算这个 LSE，而 LSE 有一个"平移不变"的恒等式，让我们能安全地计算它：

$$
\text{LSE}(z) = c + \log \sum_{i} e^{z_i - c}
$$

其中 $c$ 是我们**自己挑的任意常数**。实践中永远取 $c = \max_i z_i$（最大的那个 logit）。这样一来，$z_i - c$ 全都 $\leq 0$，$e^{z_i - c}$ 全都落在 $(0, 1]$ 里——**永不上溢**；而且至少有一项（最大那项）等于 $e^0 = 1$，求和结果 $\geq 1$，$\log$ 也**永不下溢成负无穷**。

通俗理解：LSE 其实是"最大值"的一个**平滑版本**。你可以把它近似看成 $\text{LSE}(z) \approx \max_i z_i$（当各项差距大时几乎相等），所以它常被叫作"soft maximum"（软最大值）。而 $c = \max$ 这个技巧，本质就是"先把最高的山峰移到海平面，再量各座山的相对高度"——相对高度算完，再把海平面加回去。类比之后回到机制：这一步平移之所以合法，是因为下面这个等式恒成立。

### 2. 推导与证明

**第一部分：证明平移恒等式（这是整个技巧的地基）**

我们要证 $\log \sum_i e^{z_i} = c + \log \sum_i e^{z_i - c}$，对任意 $c$ 成立。

关键步骤：
1. 从右边出发，把 $e^{z_i - c}$ 拆开：$e^{z_i - c} = e^{z_i} \cdot e^{-c}$。这一步用的是指数运算法则 $e^{a+b} = e^a e^b$。
2. 因为 $e^{-c}$ 和求和下标 $i$ 无关，可以提到求和号外面：
$$
\sum_i e^{z_i - c} = e^{-c} \sum_i e^{z_i}
$$
   **为什么要这么做**：把与 $i$ 无关的公共因子提出来，是为了让 $\log$ 能把乘法拆成加法。
3. 两边取对数，用 $\log(ab) = \log a + \log b$：
$$
\log \sum_i e^{z_i - c} = \log(e^{-c}) + \log \sum_i e^{z_i} = -c + \log \sum_i e^{z_i}
$$
4. 移项：$\log \sum_i e^{z_i} = c + \log \sum_i e^{z_i - c}$。证毕。

这说明：**减去任意 $c$ 再加回来，结果分毫不差。** 选 $c = \max_i z_i$ 只是为了数值安全，不影响正确性。

**第二部分：稳定的 Softmax 长什么样**

Softmax 的对数（log-softmax，训练里真正用的量）可以写成：
$$
\log \text{softmax}(z)_i = z_i - \text{LSE}(z) = z_i - \Big(c + \log \sum_j e^{z_j - c}\Big)
$$

带 $c = \max$ 之后，右边每个 $e^{z_j - c}$ 都安全。而 Softmax 本身则是：
$$
\text{softmax}(z)_i = \frac{e^{z_i - c}}{\sum_j e^{z_j - c}}
$$
分子分母同时减 $c$，比值不变（分子分母各乘了 $e^{-c}$，约掉了），但现在指数全都不溢出。

**第三部分：为什么交叉熵要用 log-softmax 而不是 log(softmax)**

分类的交叉熵损失是 $L = -\log \text{softmax}(z)_{\text{正确类}}$。如果你先算 softmax（得到一个可能极小的概率，比如 $10^{-40}$），再取 $\log$，那个极小概率可能已经下溢成 $0$，$\log 0 = -\infty$。而用 $\log \text{softmax}(z)_i = z_i - \text{LSE}(z)$ 这个式子，全程是加减法加一个稳定的 LSE，**永远不会碰到 $\log 0$**。这就是 PyTorch 里 `F.cross_entropy` 内部一定用 `log_softmax` 而不是 `log(softmax(...))` 的原因。

### 3. 来源与历史动机
- 论文：LogSumExp 没有单一"发明论文"，它是数值线性代数和统计计算里的经典技巧，属于凸分析中的标准工具（LSE 是一个凸函数，是最大值函数的光滑逼近）。
- 相关根源：在统计物理里，$\log \sum e^{z_i}$ 就是配分函数（partition function）的对数；在指数族分布里它是"对数配分函数 / 累积量母函数"。
- 系统性推广：Golub & Van Loan 的《Matrix Computations》、以及各类数值分析教材长期把"减最大值再取指数"列为标准防溢出手法。
- 现代影响：2022 年 Tri Dao 等人的 **FlashAttention**（arXiv:2205.14135）把这个技巧发挥到极致——用"在线（online）"更新的 LogSumExp，让注意力可以分块（tiling）计算而不必一次性把整行 softmax 展开，从而省显存、提速度。

历史背景：早期科学计算就反复被指数溢出坑过（似然函数连乘、隐马尔可夫模型的前向算法都靠它续命）。深度学习把 Softmax 放进每一层输出，让这个几十年的老技巧成了每天被调用亿万次的基础设施——你用的每一个大模型，每生成一个 token 都在偷偷做 LogSumExp。

### 4. 核心用途
用途一：**稳定的 Softmax 与 log-softmax。** 任何用到 Softmax 的地方（分类输出层、注意力权重、采样前的概率归一化）底层都先减最大值。这是 Transformer 每一层注意力都在做的事。

用途二：**稳定的交叉熵损失。** 训练分类器和语言模型时，损失函数 $-\log p_{\text{正确类}}$ 通过 $z_i - \text{LSE}(z)$ 计算，避免 $\log 0$ 和 $\text{NaN}$，是训练能收敛的前提。

用途三：**概率的对数域运算与注意力的分块计算。** 在隐马尔可夫、贝叶斯推断里，大量小概率连乘会下溢，改在对数域用 LSE 做"加法"就稳了；在 FlashAttention 里，用在线 LSE 把长序列注意力拆成小块累加，直接支撑了长上下文大模型的训练与推理。

### 5. 反直觉点与常见误区
反直觉点：**减去最大值这个"改动"竟然完全不改变结果。** 直觉上，你动了输入怎么可能输出不变？但正因为 Softmax 只关心 logits 之间的**相对差**（整体平移被指数化后变成公共因子约掉了），减最大值才既安全又无损。真正被"消灭"的只是那个会爆炸的中间量，不是信息。

常见误区：
- **误区一：以为高精度浮点数就能免疫溢出。** float64 只是把门槛推高，logits 够大照样爆。LogSumExp 是算法层面的根治，和精度是两码事。
- **误区二：以为 `log(softmax(x))` 和 `log_softmax(x)` 只是写法不同、结果一样。** 数学上等价，数值上天差地别——前者会经历"极小概率→下溢为0→log(0)=-∞"的死亡链，后者全程稳定。工程上永远用后者。
- **误区三：把 LSE 当成普通的 max。** 它是**平滑**的 max，当各项接近时（比如 $[2,2,2]$），$\text{LSE} = \log(3 e^2) = 2 + \log 3 \approx 3.1$，比真正的 max（$2$）大出一截。差距来自"有多少项在竞争"。

### 6. 小实验
不用写复杂代码，先用计算器或心算感受一下。取 $z = [0, 1, 2]$：

- 直接算：$e^0 + e^1 + e^2 = 1 + 2.718 + 7.389 = 11.107$，$\log 11.107 \approx 2.408$。所以 $\text{LSE} \approx 2.408$，比 $\max = 2$ 略大，符合"软最大值"。
- 用平移（取 $c = 2$）：$e^{-2} + e^{-1} + e^{0} = 0.135 + 0.368 + 1 = 1.503$，$\log 1.503 \approx 0.408$，再加回 $c = 2$，得 $2.408$。**两条路结果完全一致**——亲眼看到平移不改结果。

再验证溢出：想象 $z = [1000, 1001, 1002]$。直接算 $e^{1000}$ 溢出。但减去 $c = 1002$ 后变成 $[-2, -1, 0]$，指数是 $[0.135, 0.368, 1]$，求和 $\log$ 得 $0.408$，加回 $1002$ 得 $1002.408$——安全无溢出，而且和小数版本只差一个平移常数，概率分布完全相同。

想跑代码验证的话，三行 Python 即可：

```python
import numpy as np
z = np.array([1000.0, 1001.0, 1002.0])
print(np.exp(z) / np.exp(z).sum())          # 朴素版：输出 [nan nan nan]
zc = z - z.max()
print(np.exp(zc) / np.exp(zc).sum())         # 稳定版：输出 [0.09 0.24 0.67]
```

### 7. 批判性思考
局限性：LogSumExp 只解决**数值溢出/下溢**，不解决**信息丢失**。当两个 logit 的差距超过约 $700$（float64）或约 $88$（float32）时，较小那项的 $e^{z_i - c}$ 会精确下溢成 $0$，它对分布的微小贡献被彻底抹掉。多数时候这无所谓（它本就接近 0），但在需要极端尾部精度的场景要留意。此外，减最大值需要先扫一遍数据求 max，是一次额外的遍历——FlashAttention 的"在线 softmax"正是为了避免多次遍历而设计的增量更新。

深度问题：既然 LSE 是"软最大值"，那 Softmax 就是"软 argmax"（软性地选出最大项）。温度参数 $T$（你在采样策略那节学过）如何调节这个"软硬程度"？提示：把 logits 换成 $z_i / T$ 再做 LSE，$T \rightarrow 0$ 时 LSE 趋近硬 max、Softmax 趋近 one-hot；$T \rightarrow \infty$ 时趋近均匀分布。为什么？（想想平移不变性和相对差被 $T$ 缩放后会发生什么。）
