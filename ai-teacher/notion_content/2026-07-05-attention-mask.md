## 今日知识点：注意力掩码（Attention Mask / Padding Mask）——告诉模型"哪些位置根本不该看"

### 所属类目
类目 12：Self-Attention 详解

### 问题入口
真实训练里，一个 batch 要同时喂进很多句话，但句子长短不一：一句是 "I love NLP"（3 个词），另一句是 "The quick brown fox jumps"（5 个词）。GPU 只能吃规整的矩形张量，所以我们必须把短句用无意义的占位符 `<pad>` 补齐到同一长度。

问题来了：注意力机制会让每个位置去"看"序列里的所有位置，包括那些纯凑数的 `<pad>`。如果模型真的把 `<pad>` 当成有意义的词去分配注意力权重，等于往汤里掺沙子——它会学到"某些句子后面跟着一堆空气"这种错误规律。我们需要一种机制，明确告诉模型：**这些位置是假的，一个眼神都别给。**

### 朴素想法
最直觉的做法：既然 `<pad>` 是假的，那我在算完注意力、得到输出之后，把 `<pad>` 对应的输出行直接丢掉不就行了？或者，把 `<pad>` 的词向量全设成零，指望"零向量不会影响结果"。

### 卡住的地方
两个朴素想法都会失败，原因藏在 softmax 里。

回顾缩放点积注意力：一个位置对所有位置的原始打分是 $\mathbf{q} \cdot \mathbf{k}_j / \sqrt{d}$，然后过一遍 softmax 变成权重：

$$
\alpha_j = \frac{\exp(s_j)}{\sum_{k} \exp(s_k)}
$$

- **"把 pad 设成零向量"为什么不行**：零向量和 query 的点积确实是 0，但 $\exp(0) = 1$ 不是 0！softmax 的分母 $\sum_k \exp(s_k)$ 里，每个 `<pad>` 都会贡献一个 $\exp(0)=1$ 的正数份额。padding 越多，分母被撑得越大，真实词分到的权重就被稀释得越小。信息被污染，而且污染程度取决于你补了多少 pad——同一句话在不同 batch 里的输出会不一样，灾难。
- **"事后丢掉 pad 输出"为什么不行**：污染发生在 softmax 归一化那一步，等你算完再丢，真实词位置的输出**已经**混入了 pad 的 value 向量。覆水难收。

结论：必须在 **softmax 之前** 就把 pad 位置的贡献彻底清零，而不是事后补救。

### 1. 概念
**注意力掩码（Attention Mask）** 是一个和注意力打分矩阵同形状的布尔/数值矩阵，它在 softmax 之前作用于打分矩阵，把"不该看的位置"的分数改写成负无穷（$-\infty$），从而让它们在 softmax 之后的权重精确等于 0。

两种最常见的掩码：
- **Padding Mask（填充掩码）**：屏蔽 `<pad>` 占位符所在的列。凡是 key 是 pad 的位置，任何 query 都不许分给它注意力。
- **Causal Mask（因果掩码）**：屏蔽"未来"位置，让第 $i$ 个词只能看 $\le i$ 的词（这是前面 Decoder-only 一课讲过的因果注意力）。

两者可以叠加使用——解码器训练时既要遮未来、又要遮 padding。

通俗理解：想象考试时监考老师给你一副特殊眼镜，某些桌子（pad / 未来的题）在你眼里被涂成全黑，你物理上看不见。但类比到此为止，机制上它不是"变黑"，而是把这些位置的相似度分数强行拉到 $-\infty$，靠 $\exp(-\infty)=0$ 这个数学事实让 softmax 自动把权重归零。

### 2. 推导与证明
核心机制：**为什么加负无穷能精确清零，而不是"约等于零"？**

关键步骤：
1. **原始打分矩阵**。设序列长度 $n$，注意力打分矩阵 $S \in \mathbb{R}^{n \times n}$，其中 $S_{ij} = \mathbf{q}_i \cdot \mathbf{k}_j / \sqrt{d}$ 表示第 $i$ 个 query 对第 $j$ 个 key 的相似度。

2. **构造掩码并相加**。定义掩码矩阵 $M$，允许看的位置 $M_{ij}=0$，要屏蔽的位置 $M_{ij}=-\infty$（工程实现里用一个极大负数如 $-10^9$）。得到：
$$
S'_{ij} = S_{ij} + M_{ij}
$$
为什么用"加 $-\infty$"而不是"乘 0"？因为我们要作用在 **softmax 之前**的 logits 上，而 softmax 对加性平移敏感——把某个 logit 推到 $-\infty$，正好对应权重为 0。乘法做不到这一点（乘 0 会让分数变成 0，但如前所述 $\exp(0)=1$ 仍有份额）。

3. **过 softmax，屏蔽位自动归零**。对被屏蔽的位置 $j$：
$$
\alpha_{ij} = \frac{\exp(S_{ij} + M_{ij})}{\sum_{k} \exp(S_{ik} + M_{ik})} = \frac{\exp(-\infty)}{\sum_k \exp(S'_{ik})} = \frac{0}{\text{正数}} = 0
$$
分子 $\exp(-\infty) = 0$，权重精确为 0。而分母里这些屏蔽项也贡献 0，所以它们**根本不参与归一化竞争**——真实词之间的相对权重完全不受 padding 数量影响。这正是朴素想法做不到的：无论补 1 个还是 100 个 pad，真实词的注意力分布一模一样。

4. **加权求和输出干净**。最终输出 $\mathbf{o}_i = \sum_j \alpha_{ij} \mathbf{v}_j$，因为 pad 位的 $\alpha_{ij}=0$，pad 的 value 向量 $\mathbf{v}_j$ 被乘以 0，对输出零贡献。信息纯净。

### 3. 来源与历史动机
- 论文：《Attention Is All You Need》
- 作者：Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin（Google Brain / Google Research）
- 时间：2017 年 6 月
- 链接：https://arxiv.org/abs/1706.03762

历史背景：原 Transformer 论文在描述解码器时明确提到"masking, combined with the fact that the output embeddings are offset by one position"来保证自回归性质——这就是因果掩码的出处。而 padding mask 更多是工程实现层面的必需品：一旦你想批量训练变长序列，掩码就是绕不开的基础设施。它不是什么惊天算法，但没有它，注意力在真实数据上根本训不对。今天所有 Transformer 库（HuggingFace 的 `attention_mask` 参数、PyTorch 的 `key_padding_mask`）都把它做成一等公民。

### 4. 核心用途
用途一：**变长序列的批量训练/推理**。真实语料句子长短不齐，padding mask 让我们能把它们塞进同一个矩形张量高效并行，同时保证短句的输出不被填充污染。这是几乎每一次 Transformer 前向传播都在用的东西。

用途二：**自回归生成的因果约束**。因果掩码保证训练时第 $i$ 个位置预测下一个词时，绝不会偷看到答案（未来的词），否则模型会"作弊"——训练损失极低但推理时完全不会生成。这是 GPT 类模型能正确学习的前提。

用途三：**结构化的注意力控制**。掩码本质是"哪些位置能互相看"的开关矩阵，可以推广到更复杂的场景：Sparse Attention 用掩码实现局部窗口/块稀疏、双向-单向混合（如 UniLM）、文档分隔（让一个 batch 里拼接的多篇文档互不串味）。掩码是实现各种注意力拓扑的通用手柄。

### 5. 反直觉点与常见误区
反直觉点：掩码用的是 **加法 + 负无穷**，而不是你直觉上的"乘以 0 屏蔽"。原因是它必须作用在 softmax 之前的 logits 上——只有把 logit 打到 $-\infty$ 才能让归一化后的权重真正为 0；在 softmax 之后再乘 0，虽然那一项没了，但归一化分母已经被污染、其它权重的相对比例已经错了。位置错一步，结果全错。

常见误区：
- **误区一："把 pad 词向量置零就够了。"** 错。如推导所示 $\exp(0)=1$，pad 仍会瓜分 softmax 的概率质量。必须在打分上加 $-\infty$，不是在向量上做文章。
- **误区二：混淆 padding mask 和 causal mask。** 前者屏蔽"假位置（pad）"、和数据内容有关、query 方向无所谓；后者屏蔽"未来位置"、和序列顺序有关、是个下三角结构。两者形状、来源、语义都不同，训练解码器时要**同时**用。漏一个都出 bug。
- **误区三：以为 $-\infty$ 可以随便填。** 实现里若填的负数不够大（比如 $-100$），在低精度（fp16）下 $\exp$ 可能没被压到严格 0，残留微弱泄漏；但若填真正的 `-inf` 又可能在某行**全部**被 mask 时让 softmax 出 NaN（$0/0$）。这是工程上真实存在的数值陷阱。

### 6. 小实验
手算即可，感受 padding 污染有多严重。假设 query 对 3 个真实词的打分都是 $2.0$，现在补了 2 个 pad，pad 的打分（因为向量非零）假设也是 $2.0$。

**不加掩码**：5 个位置权重相等，每个真实词分到 $\frac{\exp(2)}{5\exp(2)} = \frac{1}{5} = 0.2$。真实词总共只拿到 $0.6$ 的注意力，$0.4$ 被 pad 白白吃掉。

**加掩码**（pad 打分 $\rightarrow -\infty$）：只有 3 个真实词竞争，每个分到 $\frac{1}{3} \approx 0.333$，真实词总权重 $= 1.0$。

再验证"pad 数量无关性"：把 pad 从 2 个加到 10 个，加掩码后每个真实词仍是 $\frac{1}{3}$ 纹丝不动；不加掩码则跌到 $\frac{1}{13}\approx 0.077$，随 pad 数量剧烈漂移。这就是为什么掩码不是可选优化，而是正确性的一部分。

### 7. 批判性思考
局限性：掩码只是把不该看的位置"关掉"，但打分矩阵 $S$ 本身仍是完整的 $n \times n$——也就是说，即使一半是 padding，你依然付出了 $O(n^2)$ 的计算和显存代价去算那些注定被 mask 掉的分数，纯属浪费。这在长序列 + 大量 padding 的 batch 里尤其肉疼，也是变长序列打包（sequence packing）、FlashAttention 的变长内核、以及分桶（bucketing，把长度相近的句子放一个 batch 减少 padding）等优化想解决的问题。

深度问题：如果一整行都被 mask（比如某个 query 位置本身就是 pad，它对所有 key 又叠加了 causal/padding 约束导致无一可看），softmax 的分子分母全为 0，得到 $0/0=\text{NaN}$，会毒化整个反向传播。真实框架怎么防？（提示：要么保证每行至少留一个可见位置，要么对全 mask 行做特殊处理把输出直接置零而不过 softmax。这个边界情况坑过无数人。）

### 8. 和后续知识的连接
- 向前接 **缩放点积注意力 / QKV / 因果注意力**：掩码就插在"算完 $QK^\top/\sqrt{d}$、softmax 之前"这一步，是注意力流水线的固定环节。因果掩码是掩码思想在 Decoder 上的特例。
- 向后接 **Sparse Attention / Linear Attention / FlashAttention**：稀疏注意力本质就是用结构化掩码规定"只让邻近或特定块互相看"；FlashAttention 则要在不显式存下整个 $n\times n$ 矩阵的前提下正确应用掩码——掩码怎么和 IO 优化共存，是高效注意力的关键工程细节。
- 向后接 **KV Cache / 自回归生成**：推理时因果掩码天然满足（只缓存过去的 K/V），理解掩码有助于理解为什么 KV Cache 是合法且无损的加速。
- 向后接 **长上下文与序列打包**：掩码的 $O(n^2)$ 浪费直接催生了变长打包、分桶、block-diagonal mask 等长文本训练技巧。
