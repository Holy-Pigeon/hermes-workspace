## 今日知识点：InfoNCE 与对比学习——不靠标签，靠"拉近正例、推远负例"学表示

### 所属类目
类目 4：信息论基础

### 问题入口
我们已经学过：交叉熵能训练分类器，但前提是**每个样本都有一个明确的标签**。可现实里最海量、最廉价的数据恰恰是**没有标签的**——几十亿张网页图片、几万亿 token 的纯文本。

问题来了：**没有人告诉模型"这是猫、那是狗"，模型能不能自己学出"什么样的东西是相似的、什么样的东西是不同的"？** 更具体地说，我们想让模型把语义相近的句子映射到相近的向量（这正是 RAG 检索、语义搜索、句向量的地基）。可"相近"本身就没有标签，怎么定义损失、怎么求梯度？

### 朴素想法
一个很自然的念头：**既然没有类别标签，那我就自己造标签。**

- 拿一句话 $x$，对它做个小改动（换个同义词、随机遮几个词、换种语序）得到 $x^+$，规定"$x$ 和 $x^+$ 是一对，应该相似"。
- 然后训练模型，让编码后的向量 $f(x)$ 和 $f(x^+)$ 越靠近越好——比如直接最小化它们的欧氏距离 $\|f(x) - f(x^+)\|^2$。

这样似乎不用任何人工标签就能定义"相似"，听起来很美。

### 卡住的地方
朴素想法有一个**致命的漏洞**：如果损失只有"把正例拉近"，模型会找到一个作弊的捷径——**把所有输入都映射到同一个点**（比如全部输出零向量）。

此时任意两个向量的距离都是 0，损失完美地降到最低，但这个表示**毫无用处**：它把整个世界压成了一个点，分不出任何东西。这叫"表示坍缩"（representation collapse）。

根本原因：**只有"吸引"没有"排斥"的系统必然坍缩。** 你需要一股反向的力——不仅要让正例靠近，还要让**不相关的样本互相推远**，撑开整个表示空间。问题是：负例从哪来？推远的"力"该用什么损失来表达，才能既防坍缩、又有良好的梯度、还能和我们学过的信息论对上号？

### 1. 概念

**对比学习（Contrastive Learning）**：不预测绝对标签，而是学习一个编码器 $f$，使得"正例对"的相似度**远高于**"负例对"的相似度。

**InfoNCE 损失**（Info Noise-Contrastive Estimation）把这个目标写成一个"多选一分类"问题。给定一个锚点 $x$，它的正例 $x^+$，以及 $N-1$ 个负例 $x_1^-, \dots, x_{N-1}^-$，我们要求模型从这 $N$ 个候选里**把正例挑出来**：

$$
\mathcal{L}_{\text{InfoNCE}} = -\log \frac{\exp(\text{sim}(f(x), f(x^+)) / \tau)}{\exp(\text{sim}(f(x), f(x^+)) / \tau) + \sum_{i=1}^{N-1} \exp(\text{sim}(f(x), f(x_i^-)) / \tau)}
$$

符号逐个解释：
- $f(\cdot)$：编码器，把输入映射成向量（比如一个 Transformer encoder）。
- $\text{sim}(u, v)$：相似度函数，最常用**余弦相似度**（我们学过：$\text{sim}(u,v)=\frac{u \cdot v}{\|u\|\,\|v\|}$）。
- $\tau$（tau，温度）：一个正的缩放超参数，控制分布的"尖锐程度"，后面细讲。
- 分母：正例 + 所有负例的相似度指数和，起归一化作用。

**通俗理解**：这就像一场"认亲"游戏。给你看一张锚点照片 $x$，再给你 $N$ 个候选人（其中恰好一个是它的"孪生兄弟" $x^+$，其余都是路人 $x_i^-$），你要指认出那个兄弟。InfoNCE 就是这场认亲游戏的交叉熵损失。

但类比要落回机制：注意分子那一项、分母第一项**完全相同**——这个式子的本质是一个 $N$ 类的 **Softmax 分类**，其中"正确类别"永远是正例那一格。所以 InfoNCE 不是什么新损失，它就是**在 $N$ 个候选上做 Softmax，再取正例的交叉熵**（都是我们学过的东西）。

### 2. 推导与证明

**关键步骤一：为什么它等价于一个 Softmax 交叉熵。**

回忆 Softmax：给一组打分 $s_0, s_1, \dots, s_{N-1}$，第 $k$ 类的概率是 $\frac{\exp(s_k)}{\sum_j \exp(s_j)}$。回忆交叉熵：真实标签是"第 0 类"（正例）时，损失为 $-\log(\text{第 0 类的概率})$。

现在令打分 $s_i = \text{sim}(f(x), f(\cdot_i))/\tau$，把正例排在第 0 位。代进去正好得到上面的 $\mathcal{L}_{\text{InfoNCE}}$。**所以 InfoNCE = 以相似度为 logits、以"正例是正确答案"为标签的交叉熵。** 这一步告诉我们：它天然有良好、稳定的梯度（跟训练分类器一模一样），而且梯度会**同时**"抬高正例打分、压低负例打分"——排斥力就藏在 Softmax 的分母里，坍缩被自动禁止。

**关键步骤二：它是互信息的一个下界（信息论灵魂）。**

上一课我们学了**互信息** $I(X; Y)$：知道 $Y$ 能减少 $X$ 多少不确定性。InfoNCE 的名字里那个 "Info" 正来自这里。可以证明（van den Oord et al., 2018）：

$$
I(X; X^+) \geq \log N - \mathcal{L}_{\text{InfoNCE}}
$$

含义：**最小化 InfoNCE 损失，等价于最大化锚点与正例之间互信息的一个下界。** 换句话说，对比学习在偷偷做一件信息论上很干净的事——**让"正例对"共享尽可能多的信息**。

这个不等式还顺带解释了一个实践现象：下界里有个 $\log N$，$N$（负例数量）越大，界越紧、能挤出的互信息越多——这就是为什么对比学习**吃 batch size**，SimCLR、CLIP 都靠超大 batch 或负例队列取胜。

**关键步骤三：温度 $\tau$ 在干什么。**

把相似度除以 $\tau$：
- $\tau$ 很小 → logits 被放大 → Softmax 变尖锐 → 只要有一个负例跟锚点稍微像一点，就被狠狠惩罚（对"难负例"极度敏感）。
- $\tau$ 很大 → logits 被压平 → Softmax 趋于均匀 → 正负例区分变模糊，梯度变弱。

所以 $\tau$ 是一个"对比强度旋钮"，常见取值在 $0.05 \sim 0.1$。它和我们讲采样时的温度是同一个数学机制，只是用在了损失里。

### 3. 来源与历史动机
- **NCE 起源论文**：《Noise-Contrastive Estimation》，Gutmann & Hyvärinen，2010——最早提出"用区分真实数据和噪声"来估计概率模型。
- **InfoNCE 论文**：《Representation Learning with Contrastive Predictive Coding》（CPC），Aaron van den Oord, Yazhe Li, Oriol Vinyals，2018，arXiv:1807.03748——正式提出 InfoNCE 并证明互信息下界。
- **视觉里程碑**：SimCLR（Chen et al., 2020）、MoCo（He et al., 2020）把它推成自监督视觉的主流。
- **跨模态里程碑**：CLIP（Radford et al., 2021）用图文对比学习，把图像和文本对齐到同一空间。
- **NLP 应用**：SimCSE（Gao et al., 2021）——用 Dropout 当数据增强做句向量，成为语义检索的经典基线。

历史背景：2018 年前，无监督表示学习主要靠"重建"（自编码器、语言模型预测下一个词）。重建的问题是逼模型关注像素级/字符级的琐碎细节。对比学习换了个思路——**不重建细节，只要求区分"是不是一对"**，反而学到了更抽象、更有用的语义表示。这条路线直接催生了后来 RAG、语义搜索赖以生存的高质量句向量/图文向量。

### 4. 核心用途
用途一：**语义检索与 RAG 的向量地基。** 用 InfoNCE 训练的句向量编码器（如 SimCSE、E5、BGE），把"问题"和"相关文档"映射到相近向量，检索时用余弦相似度找最近邻。我们学过的 RAG，其"检索"这一步的向量质量，很大程度就靠对比学习撑起来。

用途二：**多模态对齐（CLIP 类）。** 把成对的图-文映射到同一空间，实现"用文字搜图""零样本图像分类"。CLIP 的损失就是双向 InfoNCE（图找文 + 文找图）。

用途三：**自监督预训练。** 在没有标签的海量数据上先用对比学习学好表示，再用少量标签微调下游任务，大幅降低对标注的依赖（视觉领域的 SimCLR/MoCo）。

### 5. 反直觉点与常见误区
反直觉点：**负例往往比正例更重要。** 直觉上我们关心"把相似的拉近"，但真正决定表示质量的常常是"推开难负例"的能力。一个和锚点很像但其实无关的"难负例"，提供的梯度信号远大于一堆一眼就能排除的"易负例"。这也是为什么 MoCo 要维护一个大负例队列、CLIP 要用巨大 batch——**都是为了见到足够多、足够难的负例。**

常见误区：
- **误区一：以为对比学习"没有标签、完全无监督"。** 更准确的说法是**自监督**：标签是从数据结构里自动构造的（正例来自数据增强或天然配对，负例来自 batch 内其他样本）。设计"怎么造正例对"（增强策略）恰恰是成败关键。
- **误区二：以为 batch 里可以随便抓负例。** 如果 batch 内混进了和锚点其实同义的样本当"负例"（假负例 / false negative），模型会被逼着把本该相近的东西推远，反而伤害表示。大规模训练里如何处理假负例是一个真实难题。
- **误区三：把温度 $\tau$ 当无关紧要的小超参。** 实践中 $\tau$ 对结果极其敏感，调错一个量级，表示质量可能天差地别。

### 6. 小实验
不用训练，用心算体会"排斥力"和"温度"。

设锚点向量与 3 个候选做点积后的相似度打分为：正例 $s^+ = 8$，两个负例 $s_1^- = 6$、$s_2^- = 2$。取温度 $\tau = 1$，先算 Softmax：
- 分母 $\approx e^{8} + e^{6} + e^{2} \approx 2981 + 403 + 7.4 \approx 3391$。
- 正例概率 $\approx 2981 / 3391 \approx 0.879$，损失 $-\log 0.879 \approx 0.129$。

现在把温度调小到 $\tau = 0.5$（等于把打分翻倍：16, 12, 4）：
- 分母 $\approx e^{16} + e^{12} + e^{4} \approx 8.9\times10^{6} + 1.6\times10^{5} + 54.6$。
- 正例概率 $\approx 8.9\times10^6 / 9.06\times10^6 \approx 0.982$，损失 $\approx 0.018$。

对比两次：温度调小后，**同样的相似度差距被放大**，正例概率从 0.88 飙到 0.98，损失骤降——模型对"正例比负例高多少"变得极度挑剔。再把那个难负例 $s_1^-$ 从 6 抬到 7.5（更难），你会发现损失明显上升，直观感受到"难负例"如何主导梯度。这两个心算就把温度和难负例的作用摸清了。

### 7. 批判性思考
局限性：
- **强烈依赖数据增强/正例构造的质量。** 增强太弱，正负例几乎一样，学不到东西；增强太强，破坏语义，正例对不再"同义"。这套先验是人工设计的，换个领域可能失效。
- **对 batch size / 负例数量敏感**，训练成本高（下界里的 $\log N$ 是双刃剑：要紧的界就得大 $N$）。
- **只学"区分"，不学"生成"。** 对比学习给的是判别式表示，本身不能生成内容，需要和生成式目标（如语言模型）配合。

深度问题：InfoNCE 最大化的是互信息的**下界**，而"最大化互信息"本身并不必然带来"对下游任务有用"的表示——存在一些反例：互信息很高但表示很差。那么，对比学习真正学好表示，到底是因为最大化了互信息，还是因为"数据增强隐含的不变性先验"和"负例带来的均匀性"这两股力量？（这正是 2020 年后表示学习理论争论的焦点——Wang & Isola 的 alignment 与 uniformity 分析给了另一个视角。）

### 8. 和后续知识的连接
- 向上承接：**互信息**（正是它的理论目标）、**Softmax 与交叉熵**（它的数学骨架）、**余弦相似度与点积**（它的相似度度量）、**温度**（和采样里同一个旋钮）——今天这一课几乎把前面信息论和线代的零件全串了起来。
- 向下铺路：**RAG 的检索**（句向量的来源）、**CLIP 式多模态**、以及句向量微调。当你之后看到 "embedding model"、"retriever"、"dual-encoder"，背后训练目标十有八九就是 InfoNCE 或它的变体。
- 与我们学过的 **DPO** 遥相呼应：DPO 也是"把偏好对里的好答案打分抬高、差答案压低"，本质同样是一个对比式的分类损失——理解了 InfoNCE，再看 DPO 会觉得亲切。
