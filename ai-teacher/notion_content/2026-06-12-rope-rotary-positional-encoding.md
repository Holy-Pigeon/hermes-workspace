## 今日知识点：RoPE（旋转位置编码）——把位置信息"转"进注意力的相对角度里

### 所属类目
类目 14：位置编码

### 问题入口
我们已经学过正弦位置编码和学习位置编码，它们都是把"第几个位置"做成一个向量，**加**到词向量上。但这里藏着两个真实困难：

1. **外推问题**：如果训练时句子最长 512，推理时来了 2048 个 token，学习位置编码根本没有第 513 号位置的向量，正弦编码虽能算出来，但模型从没见过那段"高位组合"，效果会崩。
2. **相对位置才是注意力真正关心的**：判断"它"指代谁，关键是"它"和候选词**相隔多远**，而不是它们各自的绝对坐标是 37 还是 412。可加性绝对编码把位置塞进每个向量，却没有直接、干净地告诉注意力"这两个词差几步"。

有没有一种方法，能让注意力分数**天然只依赖两个 token 的相对距离**，同时又能优雅地外推到没见过的长度？这就是 RoPE 要解决的问题。

### 朴素想法
如果不知道 RoPE，最自然的做法有两种：

- **办法 A：继续用绝对位置编码相加。** 给位置 m 一个向量 p_m，把它加到词向量 x_m 上，让模型自己从绝对位置里"算"出相对距离。
- **办法 B：直接给注意力分数加一个相对偏置。** 算完 query·key 之后，根据 i 和 j 的距离 (i−j) 查一张表，加上一个偏置数。

### 卡住的地方
- 办法 A 的问题：相对距离是模型**间接推断**出来的，不是结构上保证的。而且绝对编码外推差——位置 513 对模型是全新的输入分布。
- 办法 B（这其实就是后面要学的 ALiBi 的思路）能编码相对距离，但它作用在**标量分数**上，没法让 query 和 key 向量本身携带"相对几何"，表达能力受限，也不能让不同维度有不同的"旋转频率"去捕捉远近不同的关系。

我们真正想要的是：让 query 和 key 在做点积**之前**，就把各自的位置"刻"进向量的几何结构里，并且点积的结果**自动**只剩下相对位置 (m−n)。RoPE 用"旋转"做到了这一点。

### 1. 概念

**RoPE（Rotary Position Embedding，旋转位置编码）的核心思想**：不要把位置信息"加"到向量上，而是按照位置把向量"**旋转**"一个角度。位置越靠后，转得越多。

关键性质：当 query（在位置 m）和 key（在位置 n）做点积时，两次旋转的效果会**合并成只依赖角度差** (m−n)θ 的形式。于是注意力分数天然只看相对距离。

**通俗理解**：想象每个词的特征是钟表上的一根指针。绝对位置编码是"在指针旁边贴一张写着几点的便利贴"；而 RoPE 是"直接把指针转到对应的时刻"。当你比较两根指针时，你关心的是它们**夹角**有多大——而夹角只取决于两者的时间差，跟你把表盘整体转多少度无关。这个"夹角只依赖时间差"的性质，就是 RoPE 把绝对旋转变成相对编码的魔法。

但类比要回到数学。先看二维情形：把向量 $x=(x_1, x_2)$ 在位置 $m$ 旋转角度 $m\theta$，用旋转矩阵：

$$R_m = \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix}$$

其中 $m$ 是 token 的位置序号，$\theta$ 是这一对维度选定的固定频率。旋转后的向量是 $R_m x$。

### 2. 推导与证明

我们要证明的核心结论：**经过 RoPE 旋转后的 query 和 key 做点积，结果只依赖相对位置 (m−n)。**

关键步骤：

1. **设置。** 设 query 向量在位置 $m$ 经旋转后为 $R_m q$，key 向量在位置 $n$ 经旋转后为 $R_n k$。这里 $q, k$ 是原始（未加位置）的二维向量，$R_m, R_n$ 是上面定义的旋转矩阵。

2. **写出注意力打分（点积）。** 注意力需要算 $(R_m q)^\top (R_n k)$。展开：
$$(R_m q)^\top (R_n k) = q^\top R_m^\top R_n k$$
这里用了转置的性质 $(R_m q)^\top = q^\top R_m^\top$。

3. **利用旋转矩阵的关键代数性质。** 旋转矩阵满足 $R_m^\top = R_{-m}$（转一个角度的逆等于转回去），并且两个旋转相乘等于角度相加：$R_{-m} R_n = R_{n-m}$。于是：
$$q^\top R_m^\top R_n k = q^\top R_{n-m} k$$

4. **读出结论。** 右边 $q^\top R_{n-m} k$ 里，旋转矩阵的下标是 $n-m$——**只有相对位置**！绝对位置 $m$ 和 $n$ 单独消失了，只剩它们的差。这正是我们想要的：无论这两个词在句子的开头还是结尾，只要相隔的距离一样，注意力打分对位置的依赖就一样。

5. **推广到高维。** 真实向量维度 $d$ 远大于 2。RoPE 的做法是把 $d$ 维向量**两两分组**成 $d/2$ 个二维平面，每一组用一个不同的频率 $\theta_i$ 旋转。频率按下面方式设定（借鉴正弦编码）：
$$\theta_i = 10000^{-2i/d}, \quad i = 0, 1, \dots, d/2-1$$
低频维度（$\theta_i$ 小）旋转得慢，负责捕捉长距离关系；高频维度旋转得快，负责捕捉近距离细节。每个平面单独满足"点积只依赖相对位置"，整体加起来仍然只依赖相对位置。

每一步的"为什么"：第 2 步把旋转挪到点积里，是为了让两个旋转有机会相遇相消；第 3 步是整个证明的命门——旋转矩阵的群性质（角度可加、可逆）保证了绝对量自动抵消；第 5 步用多频率，是为了让模型在不同"尺度"上都能感知距离，这和正弦编码用不同波长是同一个智慧。

### 3. 来源与历史动机
- 论文：《RoFormer: Enhanced Transformer with Rotary Position Embedding》
- 作者：Jianlin Su（苏剑林）、Yu Lu、Shengfeng Pan、Ahmed Murtadha、Bo Wen、Yunfeng Liu（追一科技）
- 时间：2021 年 4 月（arXiv 首发），后续多次修订
- 链接：https://arxiv.org/abs/2104.09864

历史背景：2021 年，绝对位置编码的外推弱点越来越突出，长文本需求上升。苏剑林在中文博客（科学空间）上从"用复数表示位置"的角度推导出 RoPE，思路极其优雅。它最大的工程胜利在后来——LLaMA、GPT-NeoX、PaLM、Qwen、DeepSeek 等几乎所有主流开源大模型都采用 RoPE，它已成为 decoder-only 大模型的事实标准。后续的长上下文技术（如 NTK-aware 插值、YaRN）也都是在 RoPE 的频率上做文章。

### 4. 核心用途
用途一：**大语言模型的标准位置编码。** LLaMA、Qwen、DeepSeek、Mistral 等都用 RoPE。它不需要额外的可学习参数（频率是固定公式），却能稳定编码相对位置，是现代 LLM 的"默认件"。

用途二：**长上下文外推。** 因为 RoPE 是连续函数式的旋转，可以通过"位置插值"（把位置序号按比例缩小）让 2K 训练的模型扩展到 32K、128K 上下文，这是 ChatGPT/Claude 类长文本能力的底层技术之一。

用途三：**相对位置敏感的任务。** 代码补全、数学推理、长文档问答这类强依赖"相隔多远"的任务，RoPE 的相对性带来直接收益。

### 5. 反直觉点与常见误区
反直觉点：**RoPE 编码的是绝对位置（每个 token 转一个绝对角度 mθ），但注意力感受到的却是相对位置。** 这听起来矛盾，关键在于"旋转的角度会在点积里相减"。你给每个 token 一个绝对旋转，系统在交互时自动把绝对量减成相对量——这是数学结构（旋转群）送的免费午餐，不是硬加上去的。

常见误区：
- **误区一：以为 RoPE 像正弦编码那样是"加"到词向量上的。** 不是。RoPE 是**乘**（旋转矩阵作用），而且只作用在 query 和 key 上，**不作用在 value 上**。value 不旋转，因为我们只想让"注意力打分"具有相对性，而不想扭曲被加权求和的内容。
- **误区二：以为旋转会改变向量长度，破坏信息。** 旋转是正交变换，**保长度、保夹角**，不丢失任何模长信息，只改变方向。这正是它安全的原因。

### 6. 小实验
不用代码，纸笔即可验证"点积只依赖相对位置"。

取二维向量 $q = (1, 0)$，$k = (1, 0)$，设 $\theta = 90°$（为了好算）。

- 情形 A：query 在位置 $m=1$，key 在位置 $n=2$，相对距离 $n-m=1$。
  - 旋转 $q$：转 $1\times90°=90°$，$(1,0)\to(0,1)$。
  - 旋转 $k$：转 $2\times90°=180°$，$(1,0)\to(-1,0)$。
  - 点积：$(0,1)\cdot(-1,0)=0$。
- 情形 B：query 在位置 $m=3$，key 在位置 $n=4$，相对距离同样是 $1$。
  - 旋转 $q$：转 $270°$，$(1,0)\to(0,-1)$。
  - 旋转 $k$：转 $360°=0°$，$(1,0)\to(1,0)$。
  - 点积：$(0,-1)\cdot(1,0)=0$。

两种情形绝对位置完全不同（1,2 对 3,4），但相对距离都是 1，点积都等于 0——亲手验证了"绝对量消失、只剩相对"。你可以再试相对距离为 2 的两组，会得到另一个相同的值。

### 7. 批判性思考
局限性：
- RoPE 的外推**不是无限免费**的。直接把 4K 训练的模型喂 100K token，注意力会因为高频维度旋转过快而失真，必须配合位置插值/NTK/YaRN 等技巧。
- 旋转频率是固定公式，不可学习，在某些任务上不如完全可学习的相对编码灵活。
- 它本质仍假设"距离衰减是合理的归纳偏置"，对需要长程精确定位的任务可能不是最优。

深度问题：RoPE 让注意力分数只依赖相对位置，那它如何还能区分"绝对开头"？比如一句话的第一个 token 往往有特殊作用（如 BOS）。提示：思考一下，当所有交互都只看相对距离时，模型靠什么识别"句首"这种绝对锚点？（答案线索：因果注意力的可见范围本身、以及 BOS token 的内容向量，承担了部分绝对信息。）

### 8. 和后续知识的连接
- **承接前面**：RoPE 直接建立在正弦位置编码（频率 $10000^{-2i/d}$ 同款）、向量旋转（线性代数的正交变换）、QKV 与缩放点积注意力之上——它就插在 Q、K 算出来之后、点积之前那一步。
- **连接后续**：学到类目 18「Decoder-only 模型（GPT）」时，你会发现现代 GPT 类模型几乎都用 RoPE 而非原始正弦编码；学到「长上下文/高效注意力」时，位置插值、YaRN、FlashAttention 的实现都要和 RoPE 协同。下一节我们会学同类目的 **ALiBi**，它走的是"在分数上加线性偏置"的另一条路，正好和 RoPE 形成对照——两种让注意力感知相对距离的不同哲学。
