## 今日知识点：贝叶斯定理（Bayes' Theorem）——用新证据修正旧信念的概率推理法则

### 所属类目
类目 3：概率与统计

### 问题入口
你做了一次癌症筛查，结果是阳性。这个检测「准确率 99%」。你现在该有多担心？很多人会直接说「99% 可能得病了」。但如果这种癌症在人群中的发病率只有万分之一，真相可能恰恰相反——你真正患病的概率可能还不到 1%。同样的逻辑遍布机器学习：垃圾邮件分类器看到「免费」这个词，该把邮件判为垃圾邮件的概率提高多少？我们需要一个法则，能把「检测结果」这种新证据，和「发病率」这种已有背景知识，正确地融合起来。直觉在这里频频翻车，必须有数学。

### 朴素想法
没有贝叶斯定理时，人们会把「在患病的前提下检测为阳性的概率」直接当成「检测为阳性时患病的概率」。也就是看到检测准确率 99%，就认为阳性结果意味着 99% 患病。本质上是把「证据有多支持假设」直接等同于「假设有多可能成立」，忽略了这个假设本身一开始有多罕见。

### 卡住的地方
这种朴素做法在「先验概率悬殊」的场景彻底失败：

- **混淆了两个方向的条件概率。** $P(\text{阳性} \mid \text{患病})$ 和 $P(\text{患病} \mid \text{阳性})$ 是两个完全不同的量。前者是检测的性能，后者才是你真正想知道的。把它们划等号叫「条件概率倒置谬误」（base rate fallacy）。
- **完全无视基础发病率。** 如果一种病一万人才一个，那么即便检测很准，海量健康人里也会冒出大量「假阳性」，把真阳性淹没。不考虑先验，结论必然偏离现实。

我们需要一个公式，明确告诉我们：新证据应该把旧信念**修正多少**，而不是直接替换掉它。这正是贝叶斯定理。

### 1. 概念

**直觉：** 贝叶斯定理是一台「信念更新机」。你对某件事原本有个估计（先验），然后来了新证据，定理告诉你该把估计调整成多少（后验）。核心思想：**新信念 = 旧信念 × 证据带来的支持度**，再做归一化。

**正式表达：** 对事件 $A$（假设）和 $B$（证据），

$$P(A \mid B) = \frac{P(B \mid A)\, P(A)}{P(B)}$$

每个符号的含义：

- $P(A)$：**先验概率**，看到证据前对 $A$ 的信念（如发病率）。
- $P(B \mid A)$：**似然**，假设 $A$ 成立时观察到证据 $B$ 的概率（如患病时检测阳性的概率）。
- $P(A \mid B)$：**后验概率**，看到证据 $B$ 后对 $A$ 的更新信念（这才是我们想求的）。
- $P(B)$：**证据的边际概率**，起归一化作用，保证后验是合法概率。它可用全概率公式展开：

$$P(B) = P(B \mid A)P(A) + P(B \mid \neg A)P(\neg A)$$

**通俗类比但不止于类比：** 把先验想成法庭上「无罪推定」的初始立场，每条证据都让陪审团把「有罪概率」往上或往下调一点。类比到此为止——关键在于调整的幅度不是拍脑袋，而由似然比 $P(B\mid A)/P(B\mid \neg A)$ 精确决定：证据越是「只在假设为真时才会出现」，对信念的撬动就越大。

### 2. 推导与证明

贝叶斯定理的推导短得惊人，核心只是条件概率定义。

**第 1 步：写出条件概率的定义。** 条件概率表示「在 $B$ 已发生的条件下 $A$ 发生」的比例：

$$P(A \mid B) = \frac{P(A \cap B)}{P(B)}, \qquad P(B \mid A) = \frac{P(A \cap B)}{P(A)}$$

*为什么*：这是条件概率的原始定义，把「联合发生」除以「条件事件」的概率，是整个推导唯一的公理基础。

**第 2 步：用同一个联合概率搭桥。** 两式都含 $P(A \cap B)$。由第二式得

$$P(A \cap B) = P(B \mid A)\, P(A)$$

*为什么*：$P(A \cap B)$ 是「$A$ 和 $B$ 同时发生」，它和顺序无关，所以可以从任一方向拆开。这一步把两个不同方向的条件概率连接了起来。

**第 3 步：代入消元。** 把第 2 步的结果代入第 1 步的第一式：

$$P(A \mid B) = \frac{P(B \mid A)\, P(A)}{P(B)}$$

定理得证。
*为什么重要*：注意结果——我们用「容易测的」$P(B \mid A)$（检测性能）和「已知的」$P(A)$（发病率），算出了「真正想要的」$P(A \mid B)$。贝叶斯定理本质是**翻转条件概率方向**的工具。

**第 4 步（实用形式）：展开归一化项。** 把 $P(B)$ 用全概率公式展开，得到只依赖先验和似然的完整公式：

$$P(A \mid B) = \frac{P(B \mid A)P(A)}{P(B \mid A)P(A) + P(B \mid \neg A)P(\neg A)}$$

*为什么*：实际问题里 $P(B)$ 往往不直接知道，但「患病时阳性率」「健康时假阳性率」「发病率」都能查到，这个形式可以直接代数字计算。

### 3. 来源与历史动机
- 论文：Thomas Bayes, "An Essay towards solving a Problem in the Doctrine of Chances"，发表于《Philosophical Transactions of the Royal Society》，1763 年（贝叶斯去世后，由 Richard Price 整理发表）。
- 作者：Thomas Bayes（英国长老会牧师、数学家）；后由 Pierre-Simon Laplace 独立且更系统地发展并推广。
- 时间：1763 年（身后发表）；Laplace 在 1774 年后将其发展成完整理论。
- 链接：经典概率论教材，如 Sheldon Ross《A First Course in Probability》。

历史背景：18 世纪的核心难题是「逆概率」——已知原因求结果（正问题）相对容易，但从观测到的结果反推原因的概率（逆问题）一直缺乏方法。Bayes 想回答的是：观察到若干次成功后，如何更新对「成功概率」的估计。Laplace 后来用它估算行星质量、人口比例，奠定了现代统计推断的基础。

### 4. 核心用途

**用途一：朴素贝叶斯分类器。** 在文本分类、垃圾邮件过滤中，用 $P(\text{类别} \mid \text{词语}) \propto P(\text{词语} \mid \text{类别}) P(\text{类别})$，结合各词的似然来判类。简单、快速、在小数据上出奇地有效，至今仍是基线模型。

**用途二：医学诊断与风险评估。** 把检测结果作为证据，结合疾病先验得到真实患病概率，避免上文那种「阳性即患病」的误判。同样的框架用于反欺诈、故障诊断等所有「从信号推原因」的场景。

**用途三：贝叶斯推断与机器学习。** 参数估计中，后验 $\propto$ 似然 $\times$ 先验，这是贝叶斯神经网络、变分推断、A/B 测试、卡尔曼滤波、强化学习中信念更新的统一框架。它让模型能表达「不确定性」，而不只给一个点估计。

### 5. 反直觉点与常见误区

**反直觉点（base rate fallacy）：** 回到开头的例子，量化一下。设发病率 $P(D)=0.0001$，检测敏感度 $P(+\mid D)=0.99$，假阳性率 $P(+\mid \neg D)=0.01$。代入：

$$P(D \mid +) = \frac{0.99 \times 0.0001}{0.99 \times 0.0001 + 0.01 \times 0.9999} \approx \frac{0.000099}{0.010098} \approx 0.0098$$

阳性结果下真正患病的概率只有约 **1%**！因为健康人太多，1% 的假阳性绝对数量远超 0.01% 的真阳性。这彻底颠覆「检测 99% 准就 99% 患病」的直觉。

**误区一：把似然当后验。** 误以为 $P(B \mid A) = P(A \mid B)$。这是最常见也最致命的错误，忽略了先验和归一化。

**误区二：忽略先验或随意设定先验。** 有人完全不考虑基础概率，有人则用极端先验（如认定某事概率为 0）。注意：先验为 0 的假设，无论多少证据都无法翻盘（$0 \times$ 任何似然仍是 0），这叫「先验吞噬证据」，设先验要留余地。

### 6. 小实验

**心算版（用自然频数，绕开公式直觉更顺）。** 想象 10000 人：

- 发病率万分之一 → 约 1 人真患病，检测准 → 这 1 人阳性。
- 其余 9999 名健康人，假阳性率 1% → 约 100 人也被测成阳性。
- 所以阳性的共 101 人，其中真患病只有 1 人 → $1/101 \approx 1\%$。

不用任何公式，仅靠数人头就重现了上面的反直觉结论。

**Python 版（验证连续更新）。**

```python
def update(prior, likelihood, false_pos):
    num = likelihood * prior
    return num / (num + false_pos * (1 - prior))

p = 0.0001                       # 先验：发病率
p = update(p, 0.99, 0.01)        # 第一次阳性
print(round(p, 4))               # 约 0.0098
p = update(p, 0.99, 0.01)        # 再独立测一次仍阳性
print(round(p, 4))               # 约 0.495，证据累积后大幅上升
```

可以看到：单次阳性只把信念推到约 1%，但第二次独立阳性后骤升到约 50%——这正是贝叶斯「逐步更新信念」的精髓。

### 7. 批判性思考

**局限性：** 贝叶斯结果对**先验**和**条件独立假设**敏感。先验选得不合理，后验就会系统性偏差；朴素贝叶斯假设特征间相互独立，现实中词语高度相关，这个假设常被违反（虽然分类边界往往仍可用）。此外，复杂模型的后验积分（归一化常数 $P(B)$）在高维下往往无法解析求解，必须靠 MCMC、变分推断等近似，带来计算和收敛上的挑战。

**深度问题：** 先验从哪里来？如果先验本身就是主观的，贝叶斯推断还算「客观」吗？思考方向：频率派批评贝叶斯「掺入主观」，贝叶斯派回应「任何推断都隐含假设，不如把它显式写出来」；随着数据增多，似然会逐渐压过先验（贝叶斯一致性），但在小样本、强先验时分歧依然显著。这场争论至今活跃，也直接影响我们如何为深度模型设计正则化（很多正则项本质就是某种先验）。

### 8. 和后续知识的连接

贝叶斯定理是概率视角理解深度学习的总纲。**最大似然估计（MLE）与最大后验估计（MAP）** 的区别正是「要不要乘先验」；**L2 正则化**等价于给权重加高斯先验下的 MAP，**Dropout** 可被解释为近似贝叶斯推断。在生成模型里，**VAE** 的核心就是对潜变量后验 $P(z \mid x)$ 做变分近似；**扩散模型**的训练与采样也建立在贝叶斯式的逐步去噪上。理解**交叉熵损失**为何是「负对数似然」、为何 softmax 输出可被当作后验概率，乃至大模型里的**校准（calibration）**与不确定性估计，都要回到贝叶斯这把尺子。
