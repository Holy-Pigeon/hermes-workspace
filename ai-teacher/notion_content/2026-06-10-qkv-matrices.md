## 今日知识点：Query / Key / Value 矩阵（QKV）——把"我想找什么、我能被什么找到、我携带什么"拆成三件事

### 所属类目
类目 12：Self-Attention 详解

### 问题入口
上一节我们学了 Luong 注意力：解码器的某个状态去和编码器每个位置打分，再加权求和。但这套机制里，**"用来打分的向量"和"被加权的向量"是同一个东西**——都是那个隐状态本身。

这就埋下一个问题：一个词向量要同时干三件不同的事——

1. 主动去问别人："我在找什么样的信息？"
2. 被别人匹配："我这里有什么标签，能被谁找到？"
3. 真正交付内容："如果被选中，我把什么信息给你？"

如果这三件事都用同一个向量来完成，模型就没有自由度去分别优化它们。**有没有办法让同一个词，根据"扮演的角色"不同，呈现出三副不同的面孔？** 这正是 QKV 要解决的问题。

### 朴素想法
最自然的想法：直接拿词向量本身两两算相似度（比如点积），相似度高的就多关注。也就是用 $x_i \cdot x_j$ 当注意力分数，然后用 $x_j$ 本身当作要聚合的内容。

这其实就是"无参数自注意力"，看起来很省事，也确实能跑。

### 卡住的地方
朴素办法在三个地方卡死：

1. **角色绑死**：用 $x_i$ 既当"提问者"又当"被匹配者"还当"内容载体"，三个角色被强行绑在同一个向量上。模型想让"提问时关注语法、交付时关注语义"，根本做不到——只有一套数。

2. **相似度被语义主导**：原始词向量的点积，主要反映"两个词意思像不像"。但注意力真正想捕捉的往往不是"像不像"，而是"该不该关联"。例如 "it" 应该关注它指代的名词，可 "it" 和那个名词的原始词向量未必相似。

3. **没有可学习参数去塑造关注模式**：朴素点积是固定的，模型无法通过训练去改变"谁该关注谁"。

一句话：朴素自注意力没有"旋钮"，模型拧不动它。

### 1. 概念
**QKV 就是给每个输入向量配三个可学习的线性投影，把它变成三个角色专用的向量：**

- **Query（查询，Q）**：代表"我作为提问者，想找什么"。
- **Key（键，K）**：代表"我作为信息源，挂着什么标签供别人匹配"。
- **Value（值，V）**：代表"如果我被选中，我实际交付的内容"。

正式表达：设输入是一行向量 $x \in \mathbb{R}^{d}$，我们引入三个权重矩阵 $W^Q, W^K, W^V$，得到

$$q = x W^Q, \quad k = x W^K, \quad v = x W^V$$

其中 $W^Q \in \mathbb{R}^{d \times d_k}$、$W^K \in \mathbb{R}^{d \times d_k}$、$W^V \in \mathbb{R}^{d \times d_v}$。$d$ 是输入维度，$d_k$ 是 Q/K 的维度（必须相同，因为要做点积），$d_v$ 是 V 的维度。

对整个序列（$n$ 个 token 堆成矩阵 $X \in \mathbb{R}^{n \times d}$）就是：

$$Q = X W^Q, \quad K = X W^K, \quad V = X W^V$$

**通俗理解**：把图书馆找书想象成这套机制。你心里的检索需求是 **Query**（"我想找深度学习入门"）；每本书书脊上的索引标签是 **Key**（"机器学习 / 神经网络"）；书里真正的内容是 **Value**。你拿需求（Q）去和所有书脊标签（K）比对，匹配度高的书你多翻几页，把它们的内容（V）按匹配度加权综合起来——这就是一次注意力。

但别停在类比：关键在于 Q、K、V 是**三个独立学习的矩阵变换**。同一本书（同一个 $x$），它"被检索时露出的标签"（K）和"它去检索别人时的需求"（Q）可以完全不同。这种解耦正是朴素办法给不了的自由度。

### 2. 推导与证明
我们来看 QKV 如何嵌入完整的自注意力计算，并说明每一步为什么需要它。

**关键步骤：**

1. **三次投影，制造角色。** 对每个 token 算 $Q = XW^Q,\ K=XW^K,\ V=XW^V$。
   - 为什么：让"提问"和"被匹配"和"交付内容"各自有独立参数，训练时能分别调整。

2. **Q 与 K 点积，得到原始注意力分数。**
   $$S = Q K^\top \in \mathbb{R}^{n \times n}$$
   $S_{ij} = q_i \cdot k_j$ 表示"第 $i$ 个 token 的查询"和"第 $j$ 个 token 的键"的匹配度。
   - 为什么用点积：点积衡量两个向量的方向一致程度，是最廉价的相似度度量；而且因为 Q、K 是学出来的，这个"相似度"已经不是原始语义相似度，而是模型认为"该不该关注"的分数。

3. **缩放。** 除以 $\sqrt{d_k}$：
   $$S' = \frac{Q K^\top}{\sqrt{d_k}}$$
   - 为什么：当 $d_k$ 较大时，点积是 $d_k$ 个乘积之和，方差随 $d_k$ 增长，数值会很大，导致 softmax 进入梯度极小的饱和区。除以 $\sqrt{d_k}$ 把方差拉回 1 量级。（这个细节下一节"缩放点积注意力"会专门展开。）

4. **softmax 归一化成权重。**
   $$A = \mathrm{softmax}(S')\quad(\text{按行归一化})$$
   每一行加起来等于 1，$A_{ij}$ 就是 token $i$ 分给 token $j$ 的注意力权重。

5. **用权重对 V 加权求和。**
   $$\mathrm{Attention}(Q,K,V) = A V$$
   - 为什么聚合的是 V 而不是 K：K 的任务只是"被匹配"，它可以专门为匹配优化（比如编码句法位置）；真正要传递的语义信息由 V 携带。**分数来自 Q·K，内容来自 V**——这是 QKV 设计最精妙的解耦。

合起来就是那个著名公式：

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V$$

**一个小证明式的观察**：如果令 $W^Q=W^K=W^V=I$（恒等矩阵），这个公式就退化成朴素的"原始词向量点积 + 加权求和"。也就是说，朴素自注意力是 QKV 的一个特例。QKV 没有发明新机制，而是在朴素机制外面包了三层"可学习的化妆"，让模型能自己决定每个角色长什么样。

### 3. 来源与历史动机
- 论文：《Attention Is All You Need》
- 作者：Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin（Google）
- 时间：2017 年 6 月
- 链接：https://arxiv.org/abs/1706.03762

历史背景：在此之前的注意力（Bahdanau 2014、Luong 2015）都依附在 RNN 之上——注意力只是"解码器看编码器"的一个增强模块，Query 来自解码器、Key/Value 来自编码器，三者没有被明确抽象成统一概念。Transformer 的关键贡献之一，就是把注意力提炼成纯粹的 **QKV 三元组运算**，并指出："序列内部自己关注自己"（self-attention，Q、K、V 全来自同一个序列）就足以替代循环结构。"Query/Key/Value"这套命名直接借鉴了**检索系统/字典查询**的语言，让机制一目了然。

### 4. 核心用途
**用途一：自注意力（Self-Attention）。** Q、K、V 全部来自同一序列。让句子里每个词根据上下文动态聚合其他词的信息，比如让 "it" 找到它指代的名词。这是 Transformer 编码器和解码器的核心。

**用途二：交叉注意力（Cross-Attention）。** Q 来自一个序列、K/V 来自另一个序列。机器翻译里解码器（Q）去关注编码器输出（K/V）；多模态模型里文本（Q）关注图像特征（K/V）。QKV 的解耦让"谁问、谁答"可以来自不同来源。

**用途三：可控的信息路由。** 因为 K 和 V 分离，可以缓存 K/V 复用（即 LLM 推理时的 **KV Cache**）——生成新 token 时，历史 token 的 K、V 不用重算，极大加速自回归生成。这是今天所有 LLM 推理优化的基石。

### 5. 反直觉点与常见误区
**反直觉点**：**用来打分的东西（K）和被聚合的东西（V）故意是两个不同的向量。** 直觉上你会觉得"我关注谁，就该拿谁的信息"，所以用同一个向量算分又当内容。但 QKV 偏要把它们拆开——因为"判断该不该关注"和"被关注后给什么"是两个不同的优化目标。K 可以专心编码"我是不是你要找的"，V 可以专心携带"找到我之后你拿到什么"。这种分离是注意力表达力的关键来源。

**常见误区：**
- **误区一**："Q、K、V 是三种不同的输入数据。" 错。在自注意力里它们来自**同一份输入 $X$**，只是经过三个不同的投影矩阵。是"一个东西的三副面孔"，不是三个东西。
- **误区二**："$W^Q,W^K,W^V$ 是人工设计的固定矩阵。" 错。它们是**随机初始化、靠反向传播训练出来**的参数，模型自己学会"提问该看哪些维度、匹配该看哪些维度"。
- **误区三**："Q 和 K 必须和 V 维度一样。" 不一定。Q、K 维度必须相等（要做点积），但 V 的维度 $d_v$ 可以不同。实践中常设成相等只是为了方便。

### 6. 小实验
不用框架，用 2 个 token、维度 2 手算一遍，亲眼看到 QKV 的解耦。

设输入两个 token：$x_1 = [1, 0]$，$x_2 = [0, 1]$。

为了看清"K 和 V 分离"的效果，我们故意让投影矩阵不同：
- $W^Q = W^K = I$（恒等，简化打分）
- $W^V = \begin{bmatrix}10 & 0\\ 0 & 20\end{bmatrix}$（放大内容）

那么：$q_1=k_1=[1,0]$，$q_2=k_2=[0,1]$；而 $v_1=[10,0]$，$v_2=[0,20]$。

算 token 1 的注意力（暂时忽略缩放）：
- 分数：$q_1\cdot k_1 = 1$，$q_1\cdot k_2 = 0$。
- softmax([1, 0]) ≈ [0.73, 0.27]。
- 输出：$0.73\times[10,0] + 0.27\times[0,20] = [7.3, 5.4]$。

**观察**：token 1 主要关注自己（权重 0.73），但因为 $W^V$ 放大了内容，输出的数值规模和原始 $x_1=[1,0]$ 完全不同。这说明"我关注谁"（由 Q·K 决定）和"我拿到什么内容"（由 V 决定）是两条独立的链路。你可以试着只改 $W^V$ 而不动 $W^Q,W^K$——注意力权重一点不变，但输出内容大变。这就是解耦的威力。

进阶可动手：把 $W^Q$ 改成 $\begin{bmatrix}0&1\\1&0\end{bmatrix}$（交换维度），再算一次，你会发现 token 1 反而开始更关注 token 2——仅仅改了"提问的视角"，关注对象就变了。

### 7. 批判性思考
**局限性：**
- **二次复杂度**：$QK^\top$ 是 $n\times n$ 矩阵，序列越长，计算和显存都按 $O(n^2)$ 暴涨。这是长文本的根本瓶颈，催生了类目 19 的"高效注意力"（Sparse / Linear / FlashAttention）。
- **三套投影 = 三倍参数**：相比朴素无参注意力，QKV 引入了大量参数，需要更多数据才能学好。
- **点积相似度的假设**：QKV 假定"该不该关注"能被一个线性投影后的点积刻画。对某些需要复杂非线性匹配的关系，单层 QKV 未必够，所以才要堆多层、多头。

**深度问题：**
- 如果让 $W^Q = W^K$（共享提问和匹配的投影），会发生什么？此时打分矩阵 $S=XW^Q(XW^Q)^\top$ 变成对称的——"i 关注 j"和"j 关注 i"的分数相等。语言里关注关系往往**不对称**（"it"关注名词，但名词不一定同等关注"it"），所以分开 $W^Q$ 和 $W^K$ 是有必要的。你能想到哪些任务里对称反而是好的吗？

### 8. 和后续知识的连接
- **下一节"缩放点积注意力"**：会把第 2 步里的 $\sqrt{d_k}$ 缩放讲透，证明为什么不缩放会让训练崩。
- **多头注意力（类目 12 / 15）**：把 QKV 投影成多组（多个头），每组在不同子空间里做注意力，让模型同时关注语法、语义、位置等多种关系。QKV 是多头的最小单元。
- **Transformer 架构（类目 13）**：编码器用自注意力（Q/K/V 同源），解码器用因果自注意力 + 交叉注意力（Q 来自解码器、K/V 来自编码器）。
- **位置编码（类目 14）**：QKV 本身对顺序无感（打乱 token 顺序，注意力结果只是跟着换位），所以必须额外注入位置信息——这正是位置编码存在的理由。RoPE 干脆直接作用在 Q、K 上。
- **KV Cache（类目 18/20）**：自回归生成时缓存历史 K、V，是 LLM 推理提速的关键工程，直接建立在 K/V 分离之上。
