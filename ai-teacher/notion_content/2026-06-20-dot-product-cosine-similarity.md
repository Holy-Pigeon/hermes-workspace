## 今日知识点：向量点积与余弦相似度（Dot Product & Cosine Similarity）——衡量"两个方向有多像"的核心运算

### 所属类目
类目 1：线性代数基础

### 问题入口
Transformer 的注意力机制里，最关键的一步是算 $QK^\top$——让每个 Query 向量去和每个 Key 向量"打分"，分数高的就多关注。那么问题来了：**给定两个向量，怎么用一个数字衡量它们"有多像"？** 这个看似简单的需求，正是注意力、检索、推荐、相似度搜索背后共同的数学引擎。如果我们连"两个向量像不像"都说不清楚，整个注意力机制就无从谈起。

### 朴素想法
最直觉的办法：把两个向量逐元素相减，看差距。比如 $\mathbf{a} = [1, 2]$ 和 $\mathbf{b} = [1, 3]$，差是 $[0, -1]$，差距很小，所以"像"。这就是用**欧氏距离**衡量相似度：差越小越像。

另一个朴素想法：既然向量有方向，那"像"应该是指"指向同一个方向"。但怎么把"方向一致"变成一个可计算的数？我们可能会想：把对应位置的数字相乘再相加？于是无意中就发明了点积。

### 卡住的地方
**用欧氏距离衡量"像不像"会在两个地方翻车：**

1. **它对长度（模长）过度敏感。** 设想两段文本的词频向量，一段是 $[2, 4]$，另一段是 $[20, 40]$。它们的"方向"完全一致（都是 1:2 的比例），语义上高度相似，但欧氏距离 $\sqrt{18^2+36^2} \approx 40$ 巨大，会被判成"完全不像"。长文档和短文档天然模长不同，距离会被文档长度主导，而不是内容。

2. **它不会自然产生"打分"语义。** 注意力需要的是"匹配度越高、分数越大"，而距离是"越像越小"，方向反了，还得额外取负号、做变换。

我们真正想要的是一个**直接衡量"方向一致程度"、且能给出"越像越大"分数**的运算。点积和它的归一化版本——余弦相似度——正好做到这件事。

### 1. 概念

**点积（内积）**：两个等长向量 $\mathbf{a} = [a_1, a_2, \dots, a_n]$ 和 $\mathbf{b} = [b_1, b_2, \dots, b_n]$ 的点积定义为对应元素相乘再求和：

$$
\mathbf{a} \cdot \mathbf{b} = \sum_{i=1}^{n} a_i b_i
$$

其中 $a_i$、$b_i$ 是两个向量第 $i$ 个分量，$n$ 是维度。结果是**一个标量（单个数字）**，不再是向量。

它还有一个等价的几何定义，这才是理解的关键：

$$
\mathbf{a} \cdot \mathbf{b} = \|\mathbf{a}\| \, \|\mathbf{b}\| \cos\theta
$$

这里 $\|\mathbf{a}\|$ 是向量 $\mathbf{a}$ 的模长（长度，$\|\mathbf{a}\| = \sqrt{\sum_i a_i^2}$），$\theta$ 是两个向量之间的夹角。这个公式说：**点积 = 两个长度相乘，再乘以夹角的余弦。**

**余弦相似度**：如果我们只想要"方向一致程度"，把长度这个干扰因素除掉，就得到余弦相似度：

$$
\cos\theta = \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \, \|\mathbf{b}\|}
$$

它的取值范围是 $[-1, 1]$：$1$ 表示方向完全相同，$0$ 表示垂直（正交、毫不相关），$-1$ 表示方向完全相反。

**通俗理解**：把每个向量想象成一支射出的箭。点积在问两件事的组合——"这两支箭有多长" × "它们指的方向有多接近"。余弦相似度则把"多长"剥掉，只留下"方向有多接近"，像两个人各自指着远方，我们只关心他们手臂指的方向夹角，不关心谁手臂长。但类比到此为止，回到机制：余弦本质就是把两个向量都缩放成单位长度后再做点积。

### 2. 推导与证明

我们来证明点积的代数定义 $\sum_i a_i b_i$ 和几何定义 $\|\mathbf{a}\|\|\mathbf{b}\|\cos\theta$ 是同一回事。这用**余弦定理**最干净。

关键步骤：

1. **画出三角形。** 把 $\mathbf{a}$、$\mathbf{b}$ 从同一原点出发，那么从 $\mathbf{a}$ 的尖端到 $\mathbf{b}$ 的尖端的向量就是 $\mathbf{b} - \mathbf{a}$。这三条边构成一个三角形，夹角 $\theta$ 在原点处。

2. **写出余弦定理。** 对这个三角形，对边长度的平方满足：
$$
\|\mathbf{b} - \mathbf{a}\|^2 = \|\mathbf{a}\|^2 + \|\mathbf{b}\|^2 - 2\|\mathbf{a}\|\,\|\mathbf{b}\|\cos\theta
$$
这是为什么要用余弦定理——它天然把夹角 $\theta$ 和三条边的长度联系起来。

3. **把左边用坐标展开。** 利用 $\|\mathbf{v}\|^2 = \sum_i v_i^2$：
$$
\|\mathbf{b} - \mathbf{a}\|^2 = \sum_i (b_i - a_i)^2 = \sum_i b_i^2 - 2\sum_i a_i b_i + \sum_i a_i^2
$$
即 $\|\mathbf{b} - \mathbf{a}\|^2 = \|\mathbf{b}\|^2 - 2\sum_i a_i b_i + \|\mathbf{a}\|^2$。

4. **两式对比。** 把第 3 步代入第 2 步左边，两边的 $\|\mathbf{a}\|^2$ 和 $\|\mathbf{b}\|^2$ 抵消，剩下：
$$
-2\sum_i a_i b_i = -2\|\mathbf{a}\|\,\|\mathbf{b}\|\cos\theta
$$
约掉 $-2$，立刻得到 $\sum_i a_i b_i = \|\mathbf{a}\|\,\|\mathbf{b}\|\cos\theta$。证毕。

这个证明的意义在于：它告诉我们**逐元素相乘求和（一个纯代数操作）竟然精确编码了"夹角"这个几何信息**。注意力之所以能用一次矩阵乘法 $QK^\top$ 同时算出所有 Query-Key 对的"方向匹配度"，根基就在这里。

### 3. 来源与历史动机
- 论文：点积/内积是经典线性代数概念，无单一来源论文；现代向量内积形式可追溯到 19 世纪 Hermann Grassmann 的线性扩张论与 William Rowan Hamilton 的四元数工作，后由 Josiah Willard Gibbs 与 Oliver Heaviside 在 1880 年代整理成今天的向量代数。
- 作者：Grassmann、Hamilton、Gibbs、Heaviside 等
- 时间：19 世纪中后期
- 链接：可参考 Wikipedia "Dot product" 词条 https://en.wikipedia.org/wiki/Dot_product 与 "Cosine similarity" https://en.wikipedia.org/wiki/Cosine_similarity

历史背景：当时物理学家（如电磁学）急需一种简洁方式表达"力在某方向上的投影""做功"等概念。功 $W = \mathbf{F} \cdot \mathbf{d}$ 正是点积——力和位移方向越一致做功越多。这个"投影/对齐"直觉后来被信息检索（1970 年代向量空间模型，Salton 等）借用：把文档表示成词频向量，用余弦相似度衡量查询和文档的相关性。再后来，这条线一路延伸到词嵌入（word2vec）和 Transformer 注意力——同一个运算，换了舞台。

### 4. 核心用途

用途一（注意力打分）：Transformer 中 $\text{Attention}(Q,K,V) = \text{softmax}(QK^\top / \sqrt{d_k})V$。这里 $QK^\top$ 的每个元素就是一个 Query 向量和一个 Key 向量的点积，衡量"这个 token 该多关注那个 token"。整个注意力的"相关性"判断完全建立在点积之上。

用途二（语义检索与 RAG）：把文本编码成嵌入向量后，检索"最相关文档"就是找与查询向量余弦相似度最高的那些向量。向量数据库（FAISS、Milvus 等）的核心操作就是大规模余弦/点积相似度搜索。这正是我们前面学过的 RAG 的检索环节的底层运算。

用途三（推荐系统与去重）：用户向量与物品向量做点积预测偏好（矩阵分解类推荐）；判断两段文本/图片是否重复，也靠余弦相似度设阈值。

### 5. 反直觉点与常见误区

反直觉点：**点积大，不一定方向就接近。** 因为点积 $= \|\mathbf{a}\|\|\mathbf{b}\|\cos\theta$ 同时被长度放大。一个方向只是"还行"（$\cos\theta = 0.5$）但模长很大的向量，点积可能比一个方向几乎完全一致（$\cos\theta = 0.99$）但很短的向量还大。这就是为什么需要余弦相似度把长度除掉——以及为什么注意力里要除以 $\sqrt{d_k}$ 来控制点积随维度增长而变大的问题（我们在"缩放点积注意力"那节学过）。

常见误区：
- **误区一：以为余弦相似度永远比点积好。** 在很多检索/注意力场景，向量已经被归一化（或我们就是想保留模长信息），此时直接用点积更快也更合适。是否归一化是一个设计选择，不是"余弦一定对"。
- **误区二：把点积结果当成概率或百分比。** 点积是任意实数（可正可负可大可大），余弦才被限制在 $[-1, 1]$。要变成"注意力权重"那种概率，还得过一层 softmax。点积本身只是"未归一化的打分"。

### 6. 小实验

纯手算，三组二维向量，体会"点积"和"余弦"的区别：

- $\mathbf{a} = [1, 0]$，$\mathbf{b} = [1, 0]$：点积 $= 1\times1 + 0\times0 = 1$；模长都是 1；余弦 $= 1/(1\cdot1) = 1$。方向完全相同。✓
- $\mathbf{a} = [1, 0]$，$\mathbf{c} = [0, 1]$：点积 $= 1\times0 + 0\times1 = 0$；余弦 $= 0$。两者垂直，毫不相关。
- $\mathbf{a} = [1, 0]$，$\mathbf{d} = [100, 0]$：点积 $= 100$（很大！）；但余弦 $= 100/(1 \cdot 100) = 1$。**方向完全相同，余弦说"一模一样"，而点积被 $\mathbf{d}$ 的长度撑到 100。**

最后一组最关键：它让你亲眼看到点积如何被模长主导，而余弦如何把这个干扰剥离。

如果愿意用一行代码验证：
```python
import numpy as np
a, d = np.array([1, 0]), np.array([100, 0])
print(np.dot(a, d))                                    # 100
print(np.dot(a, d) / (np.linalg.norm(a) * np.linalg.norm(d)))  # 1.0
```

### 7. 批判性思考

局限性：余弦/点积只衡量**线性方向的对齐**，捕捉不了复杂的非线性相似关系。两个语义相关但在嵌入空间里恰好接近正交的向量，余弦会判它们"不相关"——所以相似度的质量完全依赖于嵌入本身学得好不好。此外，高维空间里有"距离集中"现象：维度极高时，随机向量之间的余弦都趋近于某个值，区分度下降，这给大规模向量检索带来挑战。

深度问题：注意力为什么选择用点积而不是欧氏距离来打分？提示：点积能写成一次干净的矩阵乘法 $QK^\top$，可被 GPU 高度并行；而且它是 $Q$、$K$ 的**双线性**函数，方便求导和反向传播。如果换成欧氏距离，还能这么优雅地批量计算和优化吗？这个"为了可微、可并行而选运算"的思路，是理解整个深度学习架构设计的一把钥匙。

### 8. 和后续知识的连接

点积是 Transformer 的"原子操作"，几乎处处出现：
- **缩放点积注意力**：$QK^\top$ 的每个元素就是一次点积，除以 $\sqrt{d_k}$ 正是为了驯服点积随维度变大的副作用。
- **多头注意力**：每个头在自己的低维子空间里独立做点积打分。
- **嵌入相似度 / RAG 检索**：向量数据库的召回基于余弦/点积相似度。
- **线性层与矩阵乘法**：神经网络里每个神经元的加权求和 $\sum_i w_i x_i$ 本质就是权重向量和输入向量的点积——我们早先学的"神经元模型""矩阵乘法"，底层都是它。

可以说，理解了点积衡量"方向对齐"，你就握住了从单个神经元到注意力机制的那条主线。
