## 今日知识点：注意力的计算复杂度（Computational Complexity）——为什么序列一长，Transformer 就开始喘不过气

### 所属类目
类目 15：多头注意力机制

### 问题入口
我们已经知道，多头注意力让模型能从多个子空间同时观察一句话，效果惊艳。但有一个现实问题绕不开：**当输入序列变长时，注意力到底要付出多大代价？**

- 处理一条 512 词的句子，和处理一篇 8000 词的文章、甚至 100 万 token 的代码库，计算量差多少倍？
- 为什么 GPT 早期的上下文窗口只有 2048，而不是直接开到 100 万？
- 为什么"长上下文"会成为一个专门的研究方向，催生出 FlashAttention、稀疏注意力这些技术？

要回答这些问题，必须先把注意力的**时间复杂度和空间复杂度**算清楚。这不是炫技的数学，而是决定一个模型"能读多长"的硬约束。

### 朴素想法
直觉上，很多人会这样估计："注意力不就是几次矩阵乘法吗？神经网络层数固定，参数量固定，那计算量应该和序列长度成正比吧——句子长一倍，算得久一倍。"

这个"线性"的直觉，来自我们对前馈网络（FFN）的经验：FFN 对每个 token 独立处理，token 多一倍，确实就多算一倍。听起来很合理。

### 卡住的地方
这个朴素估计**错得很危险**。

注意力的本质是：**序列里每一个 token 都要和其他所有 token 算一次相关性**。如果序列长度是 $n$，那么"每个 token 看所有 token"就意味着 $n \times n$ 对组合。

- 序列长 512：要算约 $512^2 \approx 26$ 万对。
- 序列长 8000：要算约 $8000^2 = 6400$ 万对。
- 序列长 100 万：要算 $10^{12}$ 对——一万亿次。

序列长 16 倍（512 → 8192），计算量不是 16 倍，而是 $16^2 = 256$ 倍。这就是**平方级增长**。朴素的"线性"直觉直接漏掉了注意力最核心的成本来源。更糟的是，**显存占用也是平方级**——那张 $n \times n$ 的注意力矩阵要实实在在地存在显卡里。这才是长序列真正的拦路虎。

### 1. 概念

**计算复杂度**衡量的是：当输入规模增大时，算法所需的运算次数（时间复杂度）和存储空间（空间复杂度）如何增长。我们用大 $O$ 记号 $O(\cdot)$ 描述增长的"数量级"，忽略常数系数，只看最主导的项。

对于标准的自注意力（self-attention），设：
- $n$：序列长度（token 数）
- $d$：模型隐藏维度（每个 token 向量的长度）

它的时间复杂度是：

$$
O(n^2 \cdot d)
$$

空间复杂度（存注意力矩阵）是：

$$
O(n^2)
$$

**通俗理解**：想象一个 $n$ 人的会议室，规则是"每个人都必须和每个其他人单独握一次手"。人数翻倍，握手次数大约变四倍——因为握手次数是 $\frac{n(n-1)}{2}$，正比于 $n^2$。注意力就是这场"全员两两握手"的会议：每个 token 都要和所有 token 算一次 $\mathbf{q} \cdot \mathbf{k}$ 点积。

但类比要回到机制本身：握手是一次性的简单动作，而注意力的每一次"握手"其实是一个 $d$ 维向量点积（$d$ 次乘加），所以总成本里还要乘上 $d$，得到 $O(n^2 d)$。

### 2. 推导与证明

我们一步步算清楚自注意力的成本。回忆缩放点积注意力的公式：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^\top}{\sqrt{d}}\right) V
$$

其中 $Q, K, V$ 都是 $n \times d$ 的矩阵（$n$ 个 token，每个 $d$ 维）。

**关键步骤：**

1. **算 $QK^\top$（相关性打分）**
   $Q$ 是 $n \times d$，$K^\top$ 是 $d \times n$，相乘得到 $n \times n$ 的分数矩阵 $S$。
   矩阵乘法 $(n \times d) \times (d \times n)$ 的成本是 $n \times n \times d = O(n^2 d)$ 次乘加。
   **为什么这么做**：这一步就是"全员两两握手"，$S_{ij}$ 表示第 $i$ 个 token 对第 $j$ 个 token 的关注度。$n^2$ 个分数，每个要 $d$ 次乘加，平方项就在这里诞生。

2. **算 softmax（归一化成概率）**
   对 $S$ 的每一行做 softmax，$n^2$ 个元素各做一次指数与归一化，成本 $O(n^2)$。
   **为什么这么做**：把原始分数变成"注意力权重"，每行加起来为 1。这一步成本低于第 1 步，不影响主导项。

3. **算 $\text{softmax}(S) \cdot V$（加权求和）**
   权重矩阵 $n \times n$ 乘以 $V$（$n \times d$），又是一个 $(n \times n) \times (n \times d)$ 乘法，成本 $n \times n \times d = O(n^2 d)$。
   **为什么这么做**：用注意力权重把所有 token 的 value 加权融合，得到每个位置的新表示。

把三步加起来，主导项是 $O(n^2 d)$。

**关于 $d$ 这一项的补充**：还有一个常被忽略的成本——生成 $Q, K, V$ 本身需要用输入 $X$（$n \times d$）乘以权重矩阵 $W_Q, W_K, W_V$（$d \times d$），这部分是 $O(n d^2)$。所以完整的单层注意力成本是：

$$
O(n^2 d + n d^2)
$$

- 当 $n \gg d$（长序列）时，$n^2 d$ 主导 → 瓶颈是序列长度的平方。
- 当 $d \gg n$（短序列、大模型）时，$n d^2$ 主导 → 瓶颈是隐藏维度。

对长上下文场景，我们关心的正是第一种情况：**$O(n^2 d)$ 的平方诅咒**。

**多头不改变量级**：把 $d$ 拆成 $h$ 个头、每头维度 $d_k = d/h$，每个头的成本是 $O(n^2 d_k)$，$h$ 个头加起来是 $O(n^2 \cdot h \cdot d_k) = O(n^2 d)$。所以多头注意力的总复杂度和单头同量级——多头是"免费"重新分配了维度，没有增加数量级成本。

### 3. 来源与历史动机
- 论文：《Attention Is All You Need》（Transformer 原始论文）
- 作者：Vaswani, Shazeer, Parmar, Uszkoreit, Jones, Gomez, Kaiser, Polosukhin（Google）
- 时间：2017 年 6 月
- 链接：https://arxiv.org/abs/1706.03762

历史背景：原论文里有一张著名的复杂度对比表（Table 1），把 Self-Attention、循环层（Recurrent）、卷积层（Convolutional）放在一起比较"每层复杂度"和"最大路径长度"。当时作者强调的卖点是：自注意力的**最大路径长度是 $O(1)$**——任意两个 token 一步直达，不像 RNN 要走 $O(n)$ 步才能让首尾相连，这极大缓解了长距离依赖问题。

但这张表也诚实地写下了代价：自注意力每层复杂度是 $O(n^2 d)$，而 RNN 是 $O(n d^2)$。当序列短于维度（$n < d$）时，注意力反而更快；可一旦序列变长，平方项就会反噬。Transformer 用"平方的计算成本"换来了"常数的信息传播距离"——这是一笔在 2017 年看起来非常划算的交易，因为当时的序列大多只有几百 token。后来 LLM 时代追求超长上下文，这笔交易的"账单"才真正浮现，直接催生了类目 19 的全部高效注意力研究。

### 4. 核心用途

**用途一：决定上下文窗口的上限。**
模型能处理多长的输入，本质受限于显存能否放下 $n \times n$ 的注意力矩阵。一个 4 层、16 头的模型处理 $n=8192$ 的序列，光注意力矩阵就要存 $8192^2 \times 16 \times 4$ 个浮点数。理解平方关系，你就明白为什么"把上下文从 8K 扩到 128K"不是改个参数那么简单——显存需求是 256 倍。

**用途二：指导架构选型与成本预算。**
工程上估算训练/推理成本时，$O(n^2 d)$ 告诉你：处理长文档的 FLOPs 主要烧在注意力上。如果你的任务序列很短（如分类 128 token），注意力其实很便宜，没必要上稀疏注意力；如果要处理整本书，就必须考虑 FlashAttention 或线性注意力。

**用途三：定义高效注意力研究的目标函数。**
所有高效注意力方法（Sparse、Linformer、Performer、FlashAttention）的核心目标，都是把 $O(n^2)$ 降到 $O(n \log n)$、$O(n \sqrt{n})$ 或 $O(n)$。不理解原始复杂度，就无法理解这些方法"省"在哪里。

### 5. 反直觉点与常见误区

**反直觉点：FlashAttention 并没有降低时间复杂度，但它快了好几倍。**
FlashAttention 的时间复杂度**仍然是** $O(n^2 d)$——它一次点积都没省。它真正解决的是**空间复杂度和显存读写（IO）**：通过分块计算（tiling），它避免把完整的 $n \times n$ 矩阵写回显存（HBM），把空间从 $O(n^2)$ 降到 $O(n)$。速度提升来自"少搬数据"，而非"少算数据"。这说明：**在现代 GPU 上，瓶颈常常是内存带宽而非计算量本身**——大 $O$ 算的是运算次数，但真实速度还取决于数据怎么在显存层级间流动。

**常见误区：**
- **误区一**："多头注意力比单头慢好几倍。" 错。多头是把同一个 $d$ 维空间切成 $h$ 份并行算，总复杂度仍是 $O(n^2 d)$，量级不变。多头增加的是少量 reshape 和拼接开销，不是数量级。
- **误区二**："序列翻倍，计算翻倍。" 错。这是注意力最致命的误解。序列翻倍，注意力计算变 **4 倍**，显存也变约 4 倍。这正是长上下文昂贵的根源。
- **误区三**："$d^2$ 项可以忽略。" 不一定。在 $n$ 较小、模型很宽（$d$ 大）的场景，$n d^2$ 项（来自 QKV 投影和 FFN）反而是主导。该忽略谁取决于 $n$ 和 $d$ 的相对大小。

### 6. 小实验

不用写复杂代码，用一张纸或心算就能感受平方诅咒。

**手算实验：填下面这张"成本翻倍表"。**

设基准序列长度 $n=512$，记它的注意力计算量为 1 个单位（忽略 $d$，只看 $n^2$ 比例）。

| 序列长度 $n$ | 相对 512 倍数 | 注意力相对成本（$\propto n^2$） |
|---|---|---|
| 512 | 1× | $1^2 = 1$ |
| 1024 | 2× | $2^2 = 4$ |
| 2048 | 4× | $4^2 = 16$ |
| 8192 | 16× | $16^2 = 256$ |
| 131072 (128K) | 256× | $256^2 = 65536$ |

读出结论：把上下文从 512 扩到 128K（256 倍），注意力成本暴涨 **6.5 万倍**。这一栏数字会让你瞬间理解为什么长上下文是个工程噩梦。

**极短 Python 验证（可选）：**

```python
import torch, time

d = 64
for n in [512, 1024, 2048, 4096]:
    Q = torch.randn(n, d)
    K = torch.randn(n, d)
    t0 = time.time()
    for _ in range(10):
        S = Q @ K.T            # n x n 分数矩阵
        A = torch.softmax(S, dim=-1)
    dt = time.time() - t0
    print(f"n={n:5d}  时间={dt:.4f}s  矩阵元素数={n*n}")
```

运行后你会看到：$n$ 翻倍时，耗时和矩阵元素数都大约变成 4 倍，亲眼验证 $O(n^2)$。

### 7. 批判性思考

**局限性：**
大 $O$ 记号只描述"增长趋势"，忽略常数系数和硬件特性。两个都是 $O(n^2 d)$ 的实现，实际速度可能差 10 倍（FlashAttention vs 朴素实现就是例子）。所以复杂度分析是**必要但不充分**的——它告诉你"扩展性"，但不直接告诉你"在这块 GPU 上跑多快"。真实性能还要看内存带宽、并行度、kernel 融合等。

**深度问题：**
既然注意力的平方成本这么痛，为什么 Transformer 没有被纯线性的架构（如 RNN、状态空间模型 Mamba）彻底取代？提示：$O(n^2)$ 的全连接握手，换来了任意两 token 间 $O(1)$ 的信息路径和无损的全局上下文；而 $O(n)$ 的方法往往要把历史压缩进一个固定大小的状态里，存在信息瓶颈。这是一场"计算成本"与"信息保真度"的根本权衡——至今没有免费的午餐。当你读到任何"线性注意力"论文时，都该追问：**它在哪里、用什么代价，换来了这份速度？**

### 8. 和后续知识的连接

- **类目 16（前馈神经网络 FFN）**：FFN 的复杂度是 $O(n d^2)$，与注意力的 $O(n^2 d)$ 形成对照。理解两者占比，才能判断一个 Transformer 层的计算到底烧在哪。
- **类目 18（Decoder-only / 因果注意力）**：自回归生成时的 KV Cache 优化，正是为了避免每生成一个 token 都重算整个 $n^2$ 矩阵——它把单步成本从 $O(n^2 d)$ 降到 $O(n d)$，是平方诅咒在推理侧的应对。
- **类目 19（高效注意力）**：这是本节内容的直接延续。Sparse Attention（只让部分 token 握手）、Linear Attention（用核技巧把 $\text{softmax}(QK^\top)V$ 重排成 $O(n)$）、FlashAttention（降 IO 不降 FLOPs）——每一种都是对今天这个 $O(n^2)$ 的正面进攻。今天算清楚的这笔账，就是理解它们的地基。
