## 今日知识点：词嵌入（Word Embedding）——把离散的词变成可计算的稠密向量

### 所属类目
类目 20：大语言模型应用

### 问题入口
上一课我们学了 Tokenization 与 BPE：一句话被切成一串 token，每个 token 在词表里有一个整数编号（id）。比如 "cat" 可能是 3187，"dog" 是 9042。

现在问题来了：神经网络只会做矩阵乘法、加法这类**连续数值运算**。可你手里拿到的是一堆整数 id。你能不能直接把 3187、9042 这些编号塞进网络里算？

如果不能，那从「token id」到「网络能吃的输入」之间，缺的那一步到底是什么？这就是词嵌入要解决的核心问题。

### 朴素想法
**朴素办法一：直接用 id 当数值。**
"cat" = 3187，"dog" = 9042，那就把这些整数当作网络的输入特征，直接喂进去。

**朴素办法二：One-Hot 编码。**
意识到「id 大小没意义」之后，人们会想：那就用 one-hot 向量吧。词表有 $V$ 个词，每个词用一个长度为 $V$ 的向量表示，只有它自己那一位是 1，其余全是 0。

比如词表大小 $V = 50000$，"cat" 就是第 3187 位为 1、其余 49999 位为 0 的向量。这样每个词都是独一无二的，互不干扰。

### 卡住的地方
**朴素办法一的致命问题：id 的大小是人为编号，没有任何语义。**
3187 和 9042 这两个数字之间的「大小关系」「差值」纯属偶然。如果网络把 9042 当成「比 3187 大三倍」的特征，它学到的就是垃圾。"dog" 并不比 "cat" 大三倍。把类别编号当连续数值，是机器学习里的经典错误。

**朴素办法二（one-hot）的三个问题：**

1. **维度爆炸。** 词表动辄五万、十万。每个词就是一个五万维向量，一句 100 个词的话就是 $100 \times 50000$ 的稀疏矩阵，存储和计算都极其浪费。

2. **彼此正交，没有相似度。** 任意两个不同词的 one-hot 向量点积都是 0。也就是说在 one-hot 的世界里，"cat" 和 "dog" 的距离，跟 "cat" 和 "democracy" 的距离**完全一样**。模型无法知道猫和狗都是动物、语义相近。这一点最致命——它把语言里最宝贵的「相似性结构」全部抹掉了。

3. **无法泛化。** 训练时见过 "cat sat on the mat"，测试时遇到 "dog sat on the mat"，one-hot 下这俩输入毫无关联，模型学到的关于 "cat" 的知识一点也传不到 "dog" 上。

我们需要一种表示，既**低维**，又能让**语义相近的词在向量空间里也靠得近**。

### 1. 概念
**词嵌入（Word Embedding）**：把每个词（token）映射成一个低维、稠密、连续的实数向量，且让语义/用法相近的词，其向量在空间中也彼此接近。

形式上，嵌入就是一张查找表（embedding table）——一个矩阵 $E \in \mathbb{R}^{V \times d}$：
- $V$ 是词表大小（有多少个不同 token）。
- $d$ 是嵌入维度（每个词用多长的向量表示，常见 128、768、4096 等）。
- 矩阵的第 $i$ 行 $E_i \in \mathbb{R}^d$，就是第 $i$ 个 token 的嵌入向量。

给定 token id $i$，「查嵌入」这个动作就是取出矩阵的第 $i$ 行：

$$
\mathbf{e}_i = E_i = \text{onehot}(i)^\top E
$$

这里 $\text{onehot}(i)$ 是第 $i$ 位为 1 的列向量，用它乘 $E$ 恰好挑出第 $i$ 行。所以**嵌入查表，数学上等价于 one-hot 向量乘一个矩阵**——只是实现时我们直接按行索引，不真的做这个大矩阵乘法。

通俗理解：one-hot 把每个词扔到一个五万维空间里互相孤立的角落；词嵌入则把它们重新安置到一个只有几百维、但**布局有讲究**的房间里——猫和狗被放在邻座，标点符号被放到另一个角落。但类比到此为止，回到机制：这些向量的每个分量是什么含义？答案是——没有人为指定含义，它们是**训练出来的参数**，靠「相近的词应该有相近的向量」这个目标自动学出来的。

### 2. 推导与证明
词嵌入是怎么「学」出来的？核心思想叫**分布假说（Distributional Hypothesis）**：

> "You shall know a word by the company it keeps."（一个词的含义，由它周围经常出现的词决定。）—— Firth, 1957

也就是说：如果两个词总是出现在相似的上下文里（"cat" 和 "dog" 都常跟 "feed"、"pet"、"furry" 一起出现），它们语义就相近，向量也应该相近。Word2Vec 把这个哲学变成了可优化的目标。

以 Word2Vec 的 **Skip-gram** 为例，目标是：用中心词去预测它周围的上下文词。

关键步骤：

1. **给每个词两套向量。** 中心词向量 $\mathbf{v}_w$（取自输入嵌入表）和上下文词向量 $\mathbf{u}_o$（取自输出嵌入表）。为什么要两套？因为一个词作「中心」和作「邻居」时扮演的角色不同，分开建模更灵活；最终一般用 $\mathbf{v}_w$ 当词嵌入。

2. **用点积定义「相容度」。** 中心词 $w$ 旁边出现上下文词 $o$ 的概率，用 softmax 定义：

$$
P(o \mid w) = \frac{\exp(\mathbf{u}_o^\top \mathbf{v}_w)}{\sum_{k=1}^{V} \exp(\mathbf{u}_k^\top \mathbf{v}_w)}
$$

点积 $\mathbf{u}_o^\top \mathbf{v}_w$ 越大，表示这两个向量方向越一致，模型就认为它们越可能共现。这一步把「语义相近」翻译成了「向量点积大」——这正是后面注意力机制 QK 点积打分的同款直觉。

3. **最大化真实共现的概率。** 扫过整个语料，对每个中心词 $w_t$ 和它窗口内的真实邻居 $w_{t+j}$，最大化对数似然：

$$
J = \frac{1}{T}\sum_{t=1}^{T} \sum_{-m \leq j \leq m,\, j \neq 0} \log P(w_{t+j} \mid w_t)
$$

其中 $T$ 是语料总词数，$m$ 是窗口半径。用梯度上升（等价于对 $-J$ 做梯度下降）更新 $\mathbf{v}$ 和 $\mathbf{u}$。

4. **结果自然涌现。** 训练收敛后，凡是出现在相似上下文的词，为了让点积都变大，它们的 $\mathbf{v}$ 会被「拉」到相近的方向。语义结构就这样从纯粹的共现统计里浮现出来——没有人手工标注「猫是动物」，它是统计的副产品。

（实际中分母对整个词表求和太贵，Word2Vec 用 **负采样（negative sampling）** 近似：每次只把真实邻居拉近、随机抽几个「假邻居」推远，把 $V$ 项的 softmax 砍成几项的二分类。这是工程上的关键提速。）

### 3. 来源与历史动机
- 论文一：*Efficient Estimation of Word Representations in Vector Space*（Word2Vec）
- 作者：Tomas Mikolov 等（Google）
- 时间：2013 年
- 链接：https://arxiv.org/abs/1301.3781

- 论文二：*GloVe: Global Vectors for Word Representation*
- 作者：Pennington, Socher, Manning（Stanford）
- 时间：2014 年
- 链接：https://nlp.stanford.edu/projects/glove/

历史背景：分布假说（Firth, 1957）和更早的向量空间模型早就存在，但在 2013 年之前，词向量要么维度太高（基于共现计数矩阵），要么训练太慢。Mikolov 的 Word2Vec 用一个极简的浅层网络 + 负采样，第一次能在数十亿词的语料上高效训练出几百维的稠密词向量，并展示了惊艳的「向量算术」性质，引爆了整个 NLP 表示学习的浪潮。它是从「符号 NLP」走向「向量 NLP」的分水岭，也是后来 BERT、GPT 里那个 embedding 层的思想源头。

### 4. 核心用途
**用途一：神经网络的输入层（最核心）。** 今天每一个 Transformer——BERT、GPT、LLaMA——第一层都是一张嵌入表 $E$。token id 进来，查表变成向量，才进入后续的注意力和 FFN。我们前面学的 QKV、位置编码，全都是在「嵌入向量」之上做文章。没有这一步，整个网络无从启动。

**用途二：迁移学习与冷启动。** 在大语料上预训练好的词向量（Word2Vec/GloVe）可以直接拿来初始化下游小任务（情感分类、命名实体识别），让数据稀少的任务也能借到大语料的语义知识。这是 BERT 之前 NLP 迁移学习的主流做法。

**用途三：相似度检索与召回。** 把词/句子映射成向量后，用余弦相似度就能做语义检索——找近义词、聚类、推荐。这条线一路发展成今天 RAG 系统里的「向量数据库 + embedding 召回」，我们在 RAG 那一课已经见过它的身影。

### 5. 反直觉点与常见误区
**反直觉点：词嵌入支持「向量算术」，语义关系变成了向量加减法。**
最著名的例子：

$$
\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}} + \mathbf{v}_{\text{woman}} \approx \mathbf{v}_{\text{queen}}
$$

「国王 − 男人 + 女人 ≈ 王后」。没人教模型「性别」这个概念，但「从男性词指向对应女性词」的方向，在向量空间里居然近似是一个固定的平移向量。语义的某些维度，被自动编码成了空间里的方向。这说明稠密向量里藏着结构化的语义，而这正是 one-hot 永远做不到的。

常见误区：
- **误区一：以为嵌入向量的每一维都有明确的人类可解释含义**（比如「第 5 维代表性别」）。其实单个维度通常没有清晰语义，语义是分布在整个向量、靠方向和组合体现的。可解释的「方向」是涌现的，不是预设的。
- **误区二：以为 Word2Vec 这种静态词嵌入和 BERT/GPT 的嵌入是一回事。** 静态词嵌入里 "bank" 永远是同一个向量，分不清「河岸」还是「银行」。而 Transformer 的嵌入只是**起点**，经过多层自注意力后会变成**随上下文变化的动态表示**——同一个 "bank" 在不同句子里输出不同向量。词嵌入解决了「入口」，上下文化才解决了「一词多义」。

### 6. 小实验
不用训练，手算就能体会「相似 = 方向接近」。假设我们有一套 2 维玩具嵌入：

- cat = $[0.9, 0.1]$
- dog = $[0.8, 0.2]$
- car = $[0.1, 0.9]$

算两两余弦相似度（$\cos\theta = \frac{\mathbf{a}\cdot\mathbf{b}}{\|\mathbf{a}\|\|\mathbf{b}\|}$）：

- cat · dog $= 0.9\times0.8 + 0.1\times0.2 = 0.74$，两者模长都约 0.906，余弦 $\approx 0.74/(0.906\times0.825)\approx 0.99$ —— 非常接近 1，说明猫狗方向几乎一致。
- cat · car $= 0.9\times0.1 + 0.1\times0.9 = 0.18$，余弦 $\approx 0.18/(0.906\times0.906)\approx 0.22$ —— 接近 0，方向几乎垂直。

结论一目了然：在这套向量里，猫和狗「挨着」，猫和车「不搭」。这就是词嵌入想要的几何结构。你可以在纸上把这三个点画出来，直观看到 cat、dog 挤在右下，car 孤零零在左上。

（想动手的话，三行 PyTorch 即可建一张真嵌入表：）
```python
import torch
emb = torch.nn.Embedding(num_embeddings=10000, embedding_dim=64)
ids = torch.tensor([3187, 9042])   # cat, dog 的 token id
vectors = emb(ids)                  # 形状 [2, 64]，就是查表取出的两行
```

### 7. 批判性思考
局限性：
- **静态词嵌入无法处理一词多义。**「苹果」是水果还是公司？单个固定向量无能为力——这是促使后来 ELMo、BERT 引入「上下文嵌入」的直接动因。
- **会继承并放大语料中的偏见。** 既然向量算术能学出 king−man+woman≈queen，它同样会学出带有性别/种族刻板印象的方向（如 doctor 偏男性）。偏见是数据里的，嵌入只是忠实地把它编码了出来，这是 NLP 公平性研究的重要课题。
- **未登录词（OOV）问题。** 词表里没有的词查不到向量。这也正是上一课 BPE 子词切分要解决的——把罕见词拆成已知子词，每个子词都有嵌入，从而拼出任意词。

深度问题：既然自注意力层会把嵌入「上下文化」成动态表示，那最初这层静态嵌入表到底还重要吗？如果把嵌入层随机初始化、和整个 Transformer 一起从零训练，最后学出来的嵌入，和单独预训练的 Word2Vec 会有何异同？（提示：现代大模型确实是端到端联合训练嵌入层的，但「相近词向量相近」这个结构依然会自发涌现——因为下游任务的梯度同样在奖励这种结构。）

### 8. 和后续知识的连接
- **向上承接**：直接接住上一课 Tokenization/BPE 的输出——token id。「切词 → 查嵌入」是大模型输入流水线的前两道工序。
- **复用旧知识**：嵌入查表本质是 one-hot 乘矩阵（类目 1 矩阵乘法）；相似度靠点积与余弦（类目 1 向量点积与余弦相似度）；训练目标用 softmax + 最大似然 + 梯度下降（类目 3、6、7 已学）。词嵌入是这些基础工具的一次漂亮合奏。
- **向下铺路**：嵌入向量是自注意力 QKV 的原料——Q、K、V 都是嵌入向量乘上各自的权重矩阵得到的。位置编码（类目 14）也是直接加到嵌入上。可以说，我们之前学的整个 Transformer 内部机制，处理的对象从第一步起就是这里产生的嵌入向量。理解了词嵌入，就接通了「离散语言」和「连续神经网络」之间那座最关键的桥。
