## 今日知识点：指数移动平均（Exponential Moving Average, EMA）——用一个数记住整段历史，还越近越重

### 所属类目
类目 8：优化算法

### 问题入口
训练一个大模型时，你几乎处处都要面对"带噪声的信号"：每个 mini-batch 算出来的梯度都在抖，Batch Normalization 每一批统计出来的均值和方差都在跳，验证集上的 loss 曲线更是上蹿下跳。

真实问题是：**我怎么从一串不断到来、还带噪声的数值里，估计出一个稳定又能跟上变化的"当前水平"？**

而且有个硬约束——你不能把全部历史都存下来慢慢平均。梯度有几十亿个分量，序列有几十万步，你只想为每个量维护**一个数**，用极小的内存实时更新。EMA 就是为这个约束量身定做的答案。

### 朴素想法
最自然的想法是"算平均值"。想知道梯度的稳定方向？把过去所有步的梯度加起来除以步数就好：

$$
\bar{g}_t = \frac{1}{t}\sum_{i=1}^{t} g_i
$$

这里 $g_i$ 是第 $i$ 步的观测值（比如梯度、loss），$\bar{g}_t$ 是到第 $t$ 步为止的算术平均。听起来很合理——多平均几个，噪声就被抵消了。

退一步，如果嫌"从头平均"太笨重，可能会想到"滑动窗口平均"：只保留最近 $k$ 个值取平均，老的就丢掉。

### 卡住的地方
朴素想法在两个地方卡死：

1. **全局平均对"变化"反应太慢。** 训练早期梯度方向变化剧烈，你希望估计值紧跟当下；但 $\frac{1}{t}$ 这个权重意味着第 1000 步时，新来的观测只占 $\frac{1}{1000}$ 的话语权，估计值几乎冻住了。它把"很久以前的陈旧信息"和"刚刚发生的新情况"一视同仁——这在**非平稳**（分布随时间漂移）的训练过程里是致命的。

2. **滑动窗口要存 $k$ 个历史值，内存翻 $k$ 倍。** 对一个有几十亿参数的模型，给每个参数再存 $k$ 个历史梯度，显存直接爆炸。而且"窗口内等权、窗口外骤降到 0"这个硬边界也不自然——为什么第 $k$ 个值满分重要、第 $k+1$ 个就完全不算？

我们真正想要的是：**离现在越近的观测权重越大、越远越轻，但永远不用存历史，每步只更新一个数。** 这正是 EMA 的设计目标。

### 1. 概念
指数移动平均（EMA）维护一个"running 估计值" $v_t$，每来一个新观测 $x_t$ 就做一次凸组合更新：

$$
v_t = \beta\, v_{t-1} + (1-\beta)\, x_t
$$

符号含义：
- $x_t$：第 $t$ 步的新观测（梯度、loss、BN 的批统计量……）。
- $v_t$：更新后的平滑估计值。
- $v_{t-1}$：上一步的估计值（唯一需要存的历史）。
- $\beta \in [0,1)$：衰减系数（decay），越接近 1 记忆越长、越平滑；越接近 0 越贴近最新观测、越灵敏。

**通俗理解**：想象你在给一个班级持续更新"平均成绩印象"。每来一次新考试，你不会把过去所有成绩重算一遍，而是"大部分保留原来的印象（$\beta$），掺进一点点这次的新成绩（$1-\beta$）"。这样最新的表现总能影响你，但老印象也不会瞬间被推翻。

但类比要回到数学：EMA 的关键不在"掺一点"，而在于**这个"掺"的动作反复叠加后，历史权重会按指数衰减**——这正是它名字里"指数"的来源。下一节把它展开证明。

### 2. 推导与证明
我们来证明：EMA 其实是**对所有历史观测的加权平均，且权重随时间指数衰减**。

关键步骤：

1. **把递推式一层层展开。** 从 $v_t = \beta v_{t-1} + (1-\beta) x_t$ 出发，把 $v_{t-1}$ 也用它的定义代入：
$$
v_t = (1-\beta)x_t + \beta(1-\beta)x_{t-1} + \beta^2 v_{t-2}
$$
继续把 $v_{t-2}$ 展开，一直递归到底（设 $v_0 = 0$），得到：
$$
v_t = (1-\beta)\sum_{i=1}^{t} \beta^{\,t-i}\, x_i
$$
为什么这么做：展开是为了看清"每个历史观测 $x_i$ 到底占多少权重"。

2. **读出权重结构。** 观测 $x_i$ 的权重是 $(1-\beta)\beta^{\,t-i}$。距离现在越远（$t-i$ 越大），$\beta^{\,t-i}$ 越小——权重按 $\beta$ 的幂**指数级衰减**。这就同时解决了朴素法的两个毛病：既是加权平均（抗噪），又天然重视近期（跟得上变化），还完全不用存历史。

3. **验证权重归一（近似）。** 把所有权重加起来是一个几何级数：
$$
(1-\beta)\sum_{i=1}^{t}\beta^{\,t-i} = (1-\beta)\cdot\frac{1-\beta^t}{1-\beta} = 1-\beta^t
$$
当 $t$ 很大时 $\beta^t \rightarrow 0$，权重之和 $\rightarrow 1$，说明它确实是一个合格的"平均"。但注意早期 $t$ 小的时候，权重和明显小于 1——这就是著名的**冷启动偏差**。

4. **推导偏差修正（bias correction）。** 假设所有观测都等于常数 $c$，理想的平均值应该就是 $c$。但由上式，$v_t = (1-\beta^t)\,c$，在早期被系统性压低了。修正方法是除掉这个因子：
$$
\hat{v}_t = \frac{v_t}{1-\beta^t}
$$
这样 $\hat{v}_t = c$，早期不再偏低。这正是 Adam 优化器里对一阶、二阶矩做偏差修正的那一步的来历。

5. **算一下"有效记忆长度"。** 权重衰减到 $\frac{1}{e}$（约 37%）需要的步数近似为 $\frac{1}{1-\beta}$。所以 $\beta=0.9$ 大约"记住最近 10 步"，$\beta=0.99$ 大约"记住最近 100 步"，$\beta=0.999$ 大约 1000 步。这个 $\frac{1}{1-\beta}$ 是调参时最实用的直觉。

### 3. 来源与历史动机
- 思想源头：EMA 源自统计学与信号处理中的**指数平滑**（exponential smoothing），最早由 Robert G. Brown、Charles Holt 在 1950 年代为需求预测与库存控制提出。
- 论文（深度学习中的关键落地）：Adam 优化器 *"Adam: A Method for Stochastic Optimization"*。
- 作者：Diederik P. Kingma, Jimmy Ba。
- 时间：2014（ICLR 2015）。
- 链接：https://arxiv.org/abs/1412.6980

历史背景：EMA 本身不是深度学习发明的，但深度学习把它用到了极致。随机梯度天生带噪、训练分布天生非平稳，正好是 EMA 最擅长的场景。从 Momentum（对梯度做 EMA）、RMSProp（对梯度平方做 EMA）、到 Adam（两者都做并加偏差修正）、再到 BatchNorm 的 running 统计、以及现代大模型训练里对权重本身做 EMA 得到更平滑的 checkpoint——EMA 是贯穿整个优化器谱系的同一根线。

### 4. 核心用途
用途一：**优化器的动量与自适应步长。** Momentum 把梯度的 EMA 当作更新方向（惯性），RMSProp/Adam 把梯度平方的 EMA 当作每个参数的自适应缩放。没有 EMA，这些优化器就无从谈起。实际场景：几乎每一次现代神经网络训练。

用途二：**Batch Normalization 的推理统计量。** 训练时 BN 用当前 batch 的均值方差，同时用 EMA 累积一份"running mean / running var"；推理时不再依赖 batch，直接用这份 EMA 统计量。实际场景：CNN、部分 Transformer 变体的归一化层。

用途三：**权重 EMA（Model EMA / Polyak averaging）。** 训练过程中额外维护一份参数的 EMA 副本，最终用这份"平滑权重"做评估或部署，往往比最后一步的原始权重泛化更好、更稳。实际场景：扩散模型、半监督学习（Mean Teacher）、许多 SOTA 视觉与语言模型的发布 checkpoint。

### 5. 反直觉点与常见误区
反直觉点：**$\beta$ 越大不代表"越准"，而是"越平滑但越滞后"。** 很多人以为把 $\beta$ 调到 0.999 能得到更好的估计，其实它只是让估计更钝——面对真正的趋势变化（比如学习率突然下降后梯度尺度骤变），大 $\beta$ 会让 EMA 迟迟跟不上。平滑和灵敏是一对必须权衡的矛盾，没有"越大越好"。

常见误区：
- **误区一：以为 EMA 存了历史。** 它只存一个数 $v_{t-1}$，"记住整段历史"是数学上的等价效果，不是真的把历史堆在内存里。这正是它相对滑动窗口的最大优势。
- **误区二：忽略偏差修正导致早期估计严重偏低。** 尤其在 Adam 里，如果去掉 $1-\beta^t$ 修正，训练最初几十步的自适应步长会被系统性压小，导致启动异常。凡是 $\beta$ 接近 1、又关心早期行为的场景，都别漏掉偏差修正。

### 6. 小实验
不用写复杂代码，拿计算器或心算就能验证 EMA 的"指数衰减"。设 $\beta=0.5$、$v_0=0$，喂进观测序列 $x = 8, 0, 0, 0, \dots$（只有第一步有个脉冲，之后全是 0），看这个脉冲的影响如何衰减：

- $v_1 = 0.5\times 0 + 0.5\times 8 = 4$
- $v_2 = 0.5\times 4 + 0.5\times 0 = 2$
- $v_3 = 0.5\times 2 = 1$
- $v_4 = 0.5$，$v_5 = 0.25\dots$

每一步正好减半——这就是 $\beta^{\,t-i}$ 的指数衰减在眼前发生。把 $\beta$ 换成 0.9 再算一遍，你会看到脉冲衰减得慢得多（每步只乘 0.9），直观感受到"$\beta$ 越大记忆越长"。

极短 Python 版（可选）：
```python
beta, v = 0.5, 0.0
for x in [8, 0, 0, 0, 0]:
    v = beta * v + (1 - beta) * x
    print(round(v, 4))
# 4.0, 2.0, 1.0, 0.5, 0.25
```

### 7. 批判性思考
局限性：EMA 假设"近期比远期更重要"，这在非平稳训练里通常成立，但并非总对。如果信号里有**周期性**结构（比如按 epoch 循环的数据顺序引入的周期噪声），单纯 EMA 无法分辨周期，可能把有意义的周期波动当噪声抹平。此外 $\beta$ 是超参数，需要人为设定；选错会引入系统性滞后。

深度问题：EMA 用一个**标量** $\beta$ 给所有历史强加了"指数衰减"这一固定形状的权重。但为什么"最优的记忆形状"一定是指数衰减？有没有任务里，理想权重应该是"先升后降"（最近一两步反而不可信，要看稍早的稳定趋势）？想想这个问题，你就会理解为什么会出现 Nesterov（先探再跳）、以及为什么有人研究可学习的、随数据自适应的衰减权重——EMA 的固定指数形状既是它的优雅，也是它的天花板。

### 8. 和后续知识的连接
- **向前**：它直接串起你已经学过的 Momentum（梯度的 EMA）、RMSProp / AdaGrad（梯度平方的 EMA）、Adam / AdamW（两者 + 偏差修正）、Batch Normalization（running 统计）——这些不再是孤立技巧，而是同一个 EMA 骨架的不同应用。Nesterov 加速则是在 EMA 惯性上再加一层"预判"。
- **向后**：在大模型训练里，权重 EMA 是很多发布 checkpoint 的标准操作；扩散模型几乎离不开 Model EMA。理解 EMA 的偏差修正，也能帮你读懂现代优化器实现里那几行看似神秘的 $1-\beta^t$ 除法。当你之后接触分布式训练里的参数平均、以及推理阶段的统计量冻结时，会一再遇到这同一个"用一个数记住整段历史"的思想。
