## 今日知识点：L1/L2 正则化（权重正则化）——给模型的"贪心"加一道紧箍咒

### 所属类目
类目 9：正则化技术

### 问题入口
我们训练一个神经网络，目标是让训练集上的损失越小越好。可现实里经常出现一个尴尬场景：模型在训练集上准确率 99%，一拿到没见过的测试数据，准确率掉到 70%。

这就是**过拟合（overfitting）**：模型没在"学规律"，而是在"背答案"。

一个具体的问题：当数据里有噪声、特征很多、样本却不够多时，模型会把权重调得很大、很极端，去精确拟合每一个训练点（包括噪声点）。我们能不能在不删数据、不改网络结构的前提下，直接告诉优化器"别把权重调那么离谱"？

L1/L2 正则化就是回答这个问题的——它在损失函数里加一项"惩罚",让模型在"拟合数据"和"保持简单"之间找平衡。

### 朴素想法
如果不知道正则化，遇到过拟合，人们会自然想到这些办法：

1. **多收集数据**：数据越多越难背，但收集数据贵且慢。
2. **把网络改小**：减少层数和神经元，让模型"没能力"过拟合。可改小了又可能欠拟合，而且每改一次都要重新调架构。
3. **早点停止训练**：看到验证集 loss 开始上升就停（这其实是 Early Stopping，一种正则化）。但它治标不治本，不能控制权重的"形状"。

这些都对，但都比较"粗"。我们想要一个更连续、更可调、不用改架构的旋钮。

### 卡住的地方
朴素想法的共同瓶颈：**它们都没有直接约束"权重本身长什么样"。**

过拟合的一个典型信号是：某些权重的绝对值变得异常大。想象一维拟合，一个高次多项式为了穿过每个带噪点，系数会剧烈震荡（一会儿 +5000，一会儿 -8000）。函数曲线变得极度扭曲，对输入的微小变化极其敏感——这正是泛化差的根源。

朴素办法里没有任何一个会对优化器说："你想把权重调到 8000？可以，但要付出代价。"我们缺的，正是一种**让大权重变贵**的机制。

### 1. 概念
**正则化**：在原始损失函数 $L_{\text{data}}$ 之外，额外加上一个只和参数有关的惩罚项 $\Omega(w)$，让总损失变成：

$$L_{\text{total}}(w) = L_{\text{data}}(w) + \lambda \, \Omega(w)$$

其中 $w$ 是模型权重，$\lambda \ge 0$ 是**正则化强度**（超参数）。$\lambda$ 越大，越强调"权重要小/简单"；$\lambda = 0$ 就退化回原始训练。

两种最常见的惩罚项：

**L2 正则化（权重衰减 / Ridge）**：惩罚权重的平方和。
$$\Omega_{L2}(w) = \frac{1}{2}\sum_i w_i^2 = \frac{1}{2}\lVert w \rVert_2^2$$

**L1 正则化（Lasso）**：惩罚权重的绝对值之和。
$$\Omega_{L1}(w) = \sum_i |w_i| = \lVert w \rVert_1$$

符号说明：$w_i$ 是第 $i$ 个权重分量；$\lVert w \rVert_2$ 是欧几里得范数（向量长度）；$\lVert w \rVert_1$ 是各分量绝对值之和（曼哈顿距离）。$\frac{1}{2}$ 只是为了求导后系数变干净，无本质影响。

**通俗理解**：把训练想象成"在崎岖地形上找最低点"。原始 loss 是地形本身；正则化相当于在地形中心（原点 $w=0$）放了一块磁铁，把解往原点拉。L2 的拉力随距离线性增长（离原点越远拉得越狠），L1 的拉力是恒定大小但方向始终指向原点。但类比要回到机制：真正起作用的是惩罚项对梯度的修改，下面推导。

### 2. 推导与证明
我们看正则化项如何改变参数更新，这才是它"起作用"的地方。

**关键步骤：**

1. **写出带正则化的梯度。** 对 L2，总损失对单个权重 $w_i$ 求偏导：
$$\frac{\partial L_{\text{total}}}{\partial w_i} = \frac{\partial L_{\text{data}}}{\partial w_i} + \lambda w_i$$
正则化贡献了一项 $\lambda w_i$——和权重本身成正比。

2. **代入梯度下降更新。** 学习率为 $\eta$：
$$w_i \leftarrow w_i - \eta\left(\frac{\partial L_{\text{data}}}{\partial w_i} + \lambda w_i\right) = (1 - \eta\lambda)\, w_i - \eta\frac{\partial L_{\text{data}}}{\partial w_i}$$
看清楚了吗？每一步先把 $w_i$ 乘上一个略小于 1 的因子 $(1-\eta\lambda)$，再做正常更新。这就是"**权重衰减（weight decay）**"名字的由来——每步都主动让权重缩水一点点。这解释了为什么 L2 让权重"整体变小但很少恰好为 0"。

3. **对 L1 做同样的事。** $|w_i|$ 的导数是符号函数 $\operatorname{sign}(w_i)$（$w_i>0$ 时为 $+1$，$<0$ 时为 $-1$）：
$$\frac{\partial L_{\text{total}}}{\partial w_i} = \frac{\partial L_{\text{data}}}{\partial w_i} + \lambda \operatorname{sign}(w_i)$$
更新变成：
$$w_i \leftarrow w_i - \eta\frac{\partial L_{\text{data}}}{\partial w_i} - \eta\lambda \operatorname{sign}(w_i)$$
关键区别：L1 每步减去的是一个**固定大小** $\eta\lambda$（方向朝 0），不管 $w_i$ 当前多大。所以即使 $w_i$ 已经很小，L1 还是用同样的力把它往 0 推——足够小的、对降低 data loss 没贡献的权重会被直接压到**精确的 0**。这就是 L1 产生**稀疏性（sparsity）**的根源。

4. **几何直观补充。** 把约束写成等价形式"在 $\Omega(w) \le C$ 的区域内最小化 $L_{\text{data}}$"。L2 的约束区域是一个**圆**（光滑），最优解通常落在圆边上某个一般位置；L1 的约束区域是一个**菱形**（有尖角，尖角正好在坐标轴上），等高线更容易先碰到尖角——而尖角意味着某些坐标为 0。这是 L1 产生稀疏解的几何解释。

### 3. 来源与历史动机
- 论文：
  - L2 在统计学叫"岭回归"：Hoerl & Kennard, *Ridge Regression: Biased Estimation for Nonorthogonal Problems* (1970)。
  - L1 即 Lasso：Robert Tibshirani, *Regression Shrinkage and Selection via the Lasso*, JRSS-B (1996)。
  - 神经网络语境下的"权重衰减"：Krogh & Hertz, *A Simple Weight Decay Can Improve Generalization* (NeurIPS 1991)。
- 作者：Hoerl、Kennard、Tibshirani、Krogh、Hertz 等。
- 时间：L2 思想可追溯到 1940s 的 Tikhonov 正则化（解病态方程）；统计学正式化在 1970（Ridge）与 1996（Lasso）；深度学习广泛使用至今。
- 链接：Lasso 原文 https://www.jstor.org/stable/2346178 ；Tikhonov 正则化背景见数值分析教材。

历史背景：早期统计学家遇到"特征比样本还多"或特征高度相关时，最小二乘解会变得极不稳定（方差爆炸）。Tikhonov 在求解病态积分方程时提出加惩罚项稳定解，这是 L2 的数学前身。1996 年 Tibshirani 用 L1 替代 L2，意外发现它能自动做"变量选择"——把无关特征系数压成 0，在高维统计里掀起一场革命。

### 4. 核心用途
**用途一：抑制过拟合，提升泛化。** 这是最普遍的用途。几乎所有现代深度学习优化器都内置 weight decay（L2），例如训练 BERT、GPT 时 AdamW 的 `weight_decay` 参数，默认就是给所有权重加 L2 惩罚。

**用途二：特征选择与模型稀疏化（L1 专属）。** 在高维场景（基因数据、推荐系统的海量特征）里，L1 能自动把无关特征权重清零，得到一个只用少数特征的可解释模型。压缩神经网络、做模型剪枝时也常借助 L1 诱导稀疏。

**用途三：数值稳定 / 解病态问题。** 当特征高度相关（多重共线性）导致解不唯一或剧烈震荡时，L2 给优化问题加上"凸性",让解唯一且稳定。这在线性回归、协方差矩阵估计中很关键。

### 5. 反直觉点与常见误区
**反直觉点：L2 几乎永远不会把权重变成精确的 0，但 L1 会。** 直觉上两者都"惩罚大权重",似乎都该让权重变小变零。但从推导第 2、3 步看：L2 的惩罚梯度是 $\lambda w_i$，当 $w_i$ 趋近 0 时惩罚力度也趋近 0，所以它"懒得"把小权重清零，只是让所有权重温和缩小；L1 的惩罚梯度是固定的 $\lambda\,\operatorname{sign}(w_i)$，到 0 附近力度不减，于是干脆推到 0。**"惩罚函数在 0 点是否光滑"决定了稀疏与否**，这是最容易被忽略的本质。

**常见误区：**
- **误区一：把 weight decay 和 L2 在所有优化器里都当成完全等价。** 在朴素 SGD 里两者数学等价，但在 Adam 这类自适应优化器里**并不等价**——Adam 里直接加 L2 到梯度会被二阶矩缩放扭曲。Loshchilov & Hutter 的 AdamW（2017）正是为修正这点而生：把权重衰减从梯度里拆出来单独施加。用 Adam 时务必分清。
- **误区二：以为 $\lambda$ 越大越好。** $\lambda$ 太大会从"防过拟合"滑向"欠拟合"——权重被压得太狠，模型连真实规律都学不到，训练和测试 loss 双双升高。$\lambda$ 是要在验证集上调的超参数，不是越大越安全。
- **误区三：对偏置项（bias）也施加正则化。** 通常**不**惩罚 bias，只惩罚连接权重。bias 只是平移，不会造成对输入的过度敏感，惩罚它常常有害无益。
- **误区四：把 Dropout、Early Stopping 和 L1/L2 混为一谈。** 它们都是正则化思想，但 L1/L2 是直接约束参数大小，机制不同，可叠加使用。

### 6. 小实验
不用复杂代码，手算就能体会 L1 与 L2 的差别。

设某个权重 $w = 0.1$，当前 data loss 对它的梯度恰好为 0（说明它对拟合已经没贡献了），取 $\eta = 0.1$，$\lambda = 1$。

**L2 更新**：$w \leftarrow (1 - \eta\lambda)w = (1 - 0.1\times 1)\times 0.1 = 0.9 \times 0.1 = 0.09$。
再来一步：$0.081$，再一步 $0.0729$……每次乘 0.9，**指数衰减，永远逼近 0 但碰不到 0**。

**L1 更新**：$w \leftarrow w - \eta\lambda\,\operatorname{sign}(w) = 0.1 - 0.1\times 1 \times 1 = 0$。**一步就到 0**（实际实现会用近端算子/软阈值防止越过 0 变负）。

这就直观看到了：L2 是"温柔缩水",L1 是"果断清零"。你可以把 $w$ 换成 5、把梯度设成非零再算几步，观察两者轨迹差异。

### 7. 批判性思考
**局限性：**
- L1/L2 假设"小权重 = 好模型",但这只是一个**先验偏好**，不总成立。某些任务最优解本就需要个别大权重，硬压会损害性能。
- L1 的稀疏性在特征高度相关时不稳定：一组相关特征里它可能随机只留一个，换个随机种子就换一个，可解释性打折扣（这催生了 Elastic Net = L1+L2 的组合）。
- 在超大模型（如 LLM）里，单纯的 weight decay 收益边际递减，正则化主力逐渐转向数据规模、Dropout、数据增强等。

**深度问题：**
- 从贝叶斯视角，L2 正则化等价于给权重加一个**高斯先验**，L1 等价于**拉普拉斯先验**——你能从"最大后验估计 MAP"推出这个对应关系吗？（提示：对先验取负对数。）
- 如果损失地形非凸（神经网络都是），正则化改变的不只是最优解位置，还改变了"优化器更容易走到哪个极小值"。正则化是在选解，还是在选路径？两者如何区分？

### 8. 和后续知识的连接
- **优化器（已学，类目 8）**：Adam 里的 `weight_decay` 就是 L2 的工程化形态；理解了本节才能看懂为什么要用 **AdamW** 训练 Transformer——这是几乎所有 LLM 预训练的标配优化器。
- **Batch Normalization（本类目下一节）**：BN 也有正则化的副作用，且会和 weight decay 产生微妙交互（BN 后权重尺度不影响输出，使 L2 的作用机制发生改变），是个值得深挖的话题。
- **Transformer 训练（类目 13 及以后）**：训练 BERT/GPT 时的超参数表里几乎都有 `weight_decay`（典型值 0.01～0.1），且通常对 LayerNorm 参数和 bias 关闭 weight decay——这正是本节"不惩罚 bias"原则的实战体现。
- **信息论（已学，类目 4）**：正则化可理解为对模型复杂度施加"描述长度"惩罚，和最小描述长度（MDL）原理一脉相承，把"简单模型更可能泛化"量化了。
