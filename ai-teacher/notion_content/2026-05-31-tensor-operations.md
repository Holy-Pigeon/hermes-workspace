## 今日知识点：张量操作（Tensor Operations）——让多维数据变成可批量计算的对象

### 所属类目
类目 5：Python 与 PyTorch

### 问题入口
训练神经网络时，我们面对的不是一个数字，而是一大批样本：一批图片、一批句子、一批词向量。

如果每个样本都用 Python 循环一个个处理，程序会很慢，也很难把计算交给 GPU。Transformer 更是这样：一个 batch 里可能有多句话，每句话有很多 token，每个 token 又是一个高维向量。

所以问题来了：有没有一种统一的数据容器，既能表示标量、向量、矩阵，也能表示更高维的数据，并且可以一次性做批量计算？

张量操作要解决的就是这个困难：把深度学习里的多维数据组织成可以高效计算的形状。

### 朴素想法
如果还不知道张量，人们很自然会想到用 Python 列表。

比如一个向量可以写成：

```python
[1, 2, 3]
```

一个矩阵可以写成：

```python
[[1, 2, 3],
 [4, 5, 6]]
```

如果有很多句子的词向量，也可以继续套列表。

这就像用很多小纸条记录班级成绩：一个学生一张纸，一个科目一行，看起来能保存信息，但真正要统计全班平均分、按科目加权、批量排序时，就会非常别扭。

### 卡住的地方
Python 列表的问题有三个。

第一，它不擅长批量数学计算。两个列表相加，Python 默认不是逐元素相加，而是拼接列表。

第二，它没有清晰的“形状”约束。深度学习很依赖 shape，比如 `[batch_size, seq_len, hidden_dim]`。如果某一层期望三维输入，你给了二维输入，模型就会出错。

第三，它不能自然利用 GPU。GPU 擅长同时处理大量规则排列的数字，但普通 Python 列表太松散，不适合直接交给 GPU 做并行计算。

所以我们需要张量：一种有形状、有数据类型、能批量运算、能放到 CPU 或 GPU 上的多维数组。

### 1. 概念
张量，英文是 Tensor，可以先理解成“多维数字表格”。

- 0 维张量：一个数，也叫标量，例如 `3.14`。
- 1 维张量：一串数，也叫向量，例如 `[1, 2, 3]`。
- 2 维张量：一张表，也叫矩阵，例如 `[[1, 2], [3, 4]]`。
- 3 维及以上张量：多层表格，例如一批句子的词向量。

在 PyTorch 里，张量通常由三个关键信息描述：

- `shape`：形状，说明每个维度有多长。
- `dtype`：数据类型，比如浮点数 `float32` 或整数 `int64`。
- `device`：存放位置，比如 CPU 或 GPU。

通俗理解：张量像一个有标签的货架。货架有几层、每层几格，这是 shape；每个格子里放整数还是小数，这是 dtype；货架放在普通仓库还是高速仓库，这是 device。

但回到机制本身，张量最重要的价值不是“能装很多数字”，而是让这些数字以规则形状排列，从而支持批量并行计算。

### 2. 推导与证明
我们用一个很小的例子说明为什么张量操作能简化计算。

假设有 2 个学生，每个学生有 3 门课成绩：

$$
X = \begin{bmatrix} 80 & 90 & 70 \\ 60 & 75 & 85 \end{bmatrix}
$$

我们想给三门课不同权重：

$$
w = [0.3, 0.4, 0.3]
$$

目标是得到每个学生的加权总分。

关键步骤：

1. 先把数据看成矩阵和向量

`X` 的 shape 是 `[2, 3]`，意思是 2 个学生、3 门课。

`w` 的 shape 是 `[3]`，意思是每门课一个权重。

为什么要先看 shape？因为 shape 告诉我们哪些维度可以对齐。这里 `X` 的第二个维度是 3，正好和 `w` 的长度 3 对齐。

2. 对每个学生做逐元素乘法

第一个学生：

$$
[80, 90, 70] \times [0.3, 0.4, 0.3] = [24, 36, 21]
$$

第二个学生：

$$
[60, 75, 85] \times [0.3, 0.4, 0.3] = [18, 30, 25.5]
$$

为什么可以一次性做？因为 `w` 可以广播到每一行。广播（broadcasting）指的是：在不真的复制数据的情况下，把较小形状的张量按规则扩展到可计算的形状。

3. 沿课程维度求和

$$
\begin{aligned}
\text{第一个学生：} & \quad 24 + 36 + 21 = 81 \\
\text{第二个学生：} & \quad 18 + 30 + 25.5 = 73.5
\end{aligned}
$$

用张量表达就是：

```python
scores = (X * w).sum(dim=1)
```

其中 `dim=1` 表示沿着第 1 个维度求和，也就是“把每个学生的 3 门课合成一个总分”。

这个例子很小，但 Transformer 里的计算本质类似：把很多 token 的向量放成规则张量，然后通过矩阵乘法、逐元素操作、reshape、sum、softmax 等操作一次性处理。

### 3. 来源与历史动机
- 论文：PyTorch: An Imperative Style, High-Performance Deep Learning Library
- 作者：Adam Paszke, Sam Gross, Francisco Massa 等
- 时间：2019
- 链接：https://arxiv.org/abs/1912.01703

历史背景：早期深度学习框架需要在“好写”和“跑得快”之间取舍。PyTorch 的重要贡献之一，是把类似 NumPy 的张量操作、GPU 加速和自动求导结合起来，让研究者可以像写普通 Python 一样搭模型，同时仍然获得高性能计算能力。

张量本身不是 PyTorch 发明的数学概念，但 PyTorch 把张量变成了深度学习工程里的核心操作对象。

### 4. 核心用途
用途一：表示模型输入和中间表示

在 Transformer 里，一批文本通常会被整理成 `[batch_size, seq_len]` 的 token id 张量。经过 embedding 后，会变成 `[batch_size, seq_len, hidden_dim]`。这个三维张量就是模型后续计算的主干数据。

用途二：批量执行神经网络计算

矩阵乘法、逐元素激活函数、归一化、softmax 都依赖张量操作。批量计算让模型不必一条样本一条样本地算，而是把一批样本打包送入 GPU，提高训练和推理效率。

用途三：连接自动求导

PyTorch 张量可以记录计算历史。当 `requires_grad=True` 时，PyTorch 会追踪张量经过了哪些操作，并在反向传播时自动计算梯度。这是后面学习神经网络训练的基础。

### 5. 反直觉点与常见误区
反直觉点：张量的维度越高，不代表概念越高级。高维张量只是“嵌套更多层的规则数组”。真正困难的是理解每个维度代表什么。

常见误区：
- 误区一：把张量等同于矩阵。矩阵只是 2 维张量，Transformer 里经常出现 3 维、4 维甚至更高维张量。
- 误区二：只看数值，不看 shape。很多深度学习 bug 不是公式错了，而是维度对错了。
- 误区三：以为 reshape 会改变数据含义。reshape 只是重新解释排列方式，如果没有想清楚维度含义，代码可能能跑但语义是错的。

### 6. 小实验
你可以用极短的 Python 代码观察 shape 的变化：

```python
import torch

X = torch.tensor([[80., 90., 70.],
                  [60., 75., 85.]])
w = torch.tensor([0.3, 0.4, 0.3])

print(X.shape)          # torch.Size([2, 3])
print(w.shape)          # torch.Size([3])
print((X * w).shape)    # torch.Size([2, 3])
print((X * w).sum(dim=1))
```

你会看到 `w` 虽然只有 `[3]`，但可以和 `[2, 3]` 的 `X` 相乘。这就是广播在起作用。

如果把 `w` 改成：

```python
w = torch.tensor([0.3, 0.4])
```

再运行 `X * w`，PyTorch 会报错。原因不是数字本身错了，而是 shape 无法对齐。

### 7. 批判性思考
局限性：张量操作要求数据尽量规则。但真实世界里的数据经常不规则，比如句子长短不同、图片尺寸不同、图结构邻居数量不同。工程上常用 padding、mask、打包序列等办法把不规则数据变成规则张量。

深度问题：为了让 GPU 高效计算，我们常常把数据整理成规则张量。但这种整理会不会引入额外假设？比如把短句补齐到同样长度后，模型如何知道哪些位置是真 token，哪些只是填充？这会引出后面的 mask 机制。

### 8. 和后续知识的连接
张量操作是 PyTorch 和 Transformer 的共同地基。

后面学习自动求导时，你会看到张量如何记录计算图；学习 Self-Attention 时，你会频繁遇到 `[batch, heads, seq_len, seq_len]` 这样的注意力权重张量；学习位置编码和多头注意力时，reshape、transpose、matmul 会成为核心操作。

理解张量，就是开始学会用“形状”思考神经网络。
