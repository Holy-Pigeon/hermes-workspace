## 今日知识点：向量（Vector）——高维空间里同时携带方向与大小的数学箭头

### 所属类目
类目 1：线性代数基础

### 问题入口
你想用一个数字描述一个点的位置，没问题——但你住在三维世界，用 3 个数字才够：(x, y, z)。

现在问题来了：这 3 个数字的"组合"到底是什么？它只是一堆数字并排站着，还是有内在结构？你能不能对它们做"加法"？能不能缩放它？它们之间有没有"相似程度"？

更具体地说：当你训练一个词嵌入模型，"猫"被表示成 [0.2, -0.8, 0.5, ...]，"狗"被表示成 [0.3, -0.7, 0.4, ...]——为什么说这两只动物"语义接近"？这里面藏着向量的核心。

### 朴素想法
最自然的想法是：用一组数字存储多维信息。Python 里就是个 list：`cat = [0.2, -0.8, 0.5]`。

想比较"猫"和"狗"多像？可以逐个数字对比，算差值之和。

### 卡住的地方
纯粹"一组数字"有严重问题：

1. **没有几何意义**：数字列表告诉你存了什么，但不告诉你它在空间里是什么方向。没有方向概念，就没法谈"这两个词往同一个语义方向走"。
2. **运算规则不清晰**：两个列表"相加"怎么算？乘以一个数怎么缩放？没有系统定义，随便你写，容易出错，也无法推广。
3. **相似度无法统一度量**：差值之和会受数字量纲影响（一个特征范围 0-1，另一个 0-100，就不公平了）。
4. **无法推广到高维**：词嵌入维度常常是 768 甚至 4096。"纯数字列表"没有数学框架支撑在这种高维空间里做运算。

所以我们需要**向量**——不只是"一组数字"，而是有严格运算规则和几何意义的数学对象。

### 1. 概念

**正式定义**

向量（Vector）是一个有序的数字序列，记为：

$$\mathbf{v} = \begin{bmatrix} v_1 \\ v_2 \\ \vdots \\ v_n \end{bmatrix}$$

其中 $n$ 称为向量的**维度**（dimension），每个 $v_i$ 称为一个**分量**（component）。

向量属于向量空间 $\mathbb{R}^n$（n 维实数空间）。它满足两大核心运算：

**向量加法**：两个同维向量逐分量相加

$$\mathbf{u} + \mathbf{v} = \begin{bmatrix} u_1 + v_1 \\ u_2 + v_2 \\ \vdots \\ u_n + v_n \end{bmatrix}$$

**标量乘法**：向量的每个分量乘以同一个数 $c$

$$c \cdot \mathbf{v} = \begin{bmatrix} c \cdot v_1 \\ c \cdot v_2 \\ \vdots \\ c \cdot v_n \end{bmatrix}$$

**通俗理解**

把向量想象成一支箭头：从原点射出，指向空间里的某一点。

- **大小**（模长/范数）告诉你箭头多长
- **方向**告诉你箭头指哪里

两支箭头可以"接尾巴"叠加（加法），也可以等比例拉伸或压缩（标量乘法）。

但类比不够，回到数学：箭头的"大小"用 **L2 范数**（欧几里得长度）来精确定义：

$$\|\mathbf{v}\|_2 = \sqrt{v_1^2 + v_2^2 + \cdots + v_n^2}$$

两个向量的"相似度"最常用**余弦相似度**来度量：

$$\cos\theta = \frac{\mathbf{u} \cdot \mathbf{v}}{\|\mathbf{u}\|_2 \|\mathbf{v}\|_2}$$

其中 $\mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^n u_i v_i$ 是**点积**（dot product）。结果范围 [-1, 1]，越接近 1 说明两个向量"方向越一致"，即语义越相近。

### 2. 推导与证明

#### 2.1 为什么余弦相似度比欧氏距离更好？

考虑两个词向量（简化为 2 维方便理解）：

$$\mathbf{a} = [1, 1], \quad \mathbf{b} = [10, 10], \quad \mathbf{c} = [1, -1]$$

欧氏距离：
- $\text{dist}(\mathbf{a}, \mathbf{b}) = \sqrt{(10-1)^2 + (10-1)^2} = \sqrt{162} \approx 12.7$（大！）
- $\text{dist}(\mathbf{a}, \mathbf{c}) = \sqrt{0^2 + (−1−1)^2} = 2$（小）

余弦相似度：
- $\cos(\mathbf{a}, \mathbf{b}) = \frac{1\cdot10 + 1\cdot10}{\sqrt{2} \cdot \sqrt{200}} = \frac{20}{20} = 1.0$（完全相同方向！）
- $\cos(\mathbf{a}, \mathbf{c}) = \frac{1\cdot1 + 1\cdot(-1)}{\sqrt{2} \cdot \sqrt{2}} = \frac{0}{2} = 0$（垂直，完全不相关）

**关键步骤**：

1. $\mathbf{a}$ 和 $\mathbf{b}$ 虽然模长差了 10 倍，但指向完全相同的方向。欧氏距离受模长影响大，余弦相似度只看方向。

2. 语义相似度更关心"这个词往什么方向偏"（方向），而不是"这个词的特征值大不大"（模长）。

3. 余弦相似度通过除以各自的模长，等效于先把两个向量都**归一化**到单位圆上，再取点积。

#### 2.2 点积（Dot Product）的几何意义

点积公式有两种等价写法：

**代数写法**（逐分量计算）：

$$\mathbf{u} \cdot \mathbf{v} = \sum_{i=1}^n u_i v_i$$

**几何写法**（用角度表达）：

$$\mathbf{u} \cdot \mathbf{v} = \|\mathbf{u}\|_2 \cdot \|\mathbf{v}\|_2 \cdot \cos\theta$$

其中 $\theta$ 是两向量之间的夹角。

两式等价，这是线性代数的基本定理之一。

**意义**：点积的值 > 0 表示夹角 < 90°（同向），= 0 表示垂直（正交），< 0 表示夹角 > 90°（反向）。

#### 2.3 向量空间的 8 条公理（为什么不能随便叫"向量"）

严格来说，向量不只是"一组数"，必须满足向量空间的 8 条公理（加法交换律、结合律、零向量存在、逆元素存在、标量乘法分配律等）。只有满足这些公理，我们才能安全地使用所有向量运算结论。普通实数列表 $\mathbb{R}^n$ 自然满足，但这也说明向量是有精确数学含义的，不只是"数字数组"。

### 3. 来源与历史动机

- **论文 / 起源**：向量概念不源自单一论文，而是 19 世纪数学系统化的产物
- **关键人物**：Hermann Grassmann（1844，《扩展论》）、William Rowan Hamilton（四元数，1843）、Josiah Willard Gibbs（确立现代向量符号，1880s）
- **时间**：19 世纪下半叶至 20 世纪初逐渐标准化
- **历史背景**：物理学家需要同时描述力的大小和方向，单个标量不够用。Grassmann 提出了 n 维"扩展量"，Gibbs 将其简化为今天的 3D 向量表示法，后来被推广到任意维度 $\mathbb{R}^n$。

在 AI 领域，向量真正爆发是在 **Word2Vec（2013，Mikolov 等人）**之后——他们发现，用神经网络训练出来的词向量，竟然能让 `国王 - 男人 + 女人 ≈ 女王`，完全靠向量运算实现了语义类比。这让人意识到向量不只是存储工具，而是语义关系的**几何编码**。

### 4. 核心用途

**用途一：词嵌入与语义表示**

NLP 中每个词被映射为一个高维向量（常见 100-4096 维）。语义相近的词，向量的余弦相似度高。Transformer 里每个 token 就是一个向量，整个序列是一个向量序列（矩阵）。

**用途二：注意力机制的 Query/Key 比较**

Transformer 的 Self-Attention 核心操作是：Query 向量和 Key 向量做点积，计算它们的"相关性分数"。这就是向量点积在实际系统中最密集的使用场景之一——你每次推理，都要对序列里每对位置做数千次高维向量点积。

**用途三：梯度（Gradient）**

你已经学过梯度——它本身就是一个向量！梯度 $\nabla f$ 是损失函数对所有参数的偏导数组成的向量，指向函数值上升最快的方向。向量加法和标量乘法在梯度下降中无处不在：$\theta \leftarrow \theta - \alpha \nabla L$，就是向量减去标量乘以向量。

### 5. 反直觉点与常见误区

**反直觉点：高维向量里，两个随机向量几乎总是"正交"的**

在低维（2D/3D）里，随机选两个方向，夹角可能大可能小。但当维度 $n$ 增大时，两个随机单位向量的点积 $\mathbf{u} \cdot \mathbf{v}$ 的期望值趋向于 0，且集中在 0 附近。换句话说，在 768 维空间里，绝大多数向量对之间的夹角都接近 90°！

这就是为什么词嵌入里，"不相关的词"大量存在，只有少数词对之间会有显著的余弦相似度——高维稀疏性带来的自然隔离。

**常见误区一：向量就是坐标点**

向量 $[3, 4]$ 不是"坐标点 (3,4)"，而是"从原点出发，指向 (3,4) 的箭头"。点没有加法，但向量有。点只是空间中的位置，向量携带方向和大小。

**常见误区二：模长大 = 向量更"重要"**

在词嵌入中，模长大只是说这个词的特征数值更大，不代表语义权重更高。语义关系靠方向（余弦相似度），不靠模长。归一化（将向量除以自身模长，得到单位向量）是消除这种干扰的标准手段。

**常见误区三：只有 2D/3D 才有几何意义**

4096 维向量同样有"方向"（就是各分量的比例关系）和"夹角"（余弦相似度）。几何直觉在高维同样成立，只是人类无法可视化，但数学结构完全一样。

### 6. 小实验

**手算验证余弦相似度**

设 $\mathbf{u} = [3, 4]$，$\mathbf{v} = [6, 8]$，$\mathbf{w} = [4, -3]$

1. 计算模长：
   - $\|\mathbf{u}\| = \sqrt{9+16} = 5$
   - $\|\mathbf{v}\| = \sqrt{36+64} = 10$
   - $\|\mathbf{w}\| = \sqrt{16+9} = 5$

2. 计算点积：
   - $\mathbf{u} \cdot \mathbf{v} = 3\cdot6 + 4\cdot8 = 18+32 = 50$
   - $\mathbf{u} \cdot \mathbf{w} = 3\cdot4 + 4\cdot(-3) = 12-12 = 0$

3. 余弦相似度：
   - $\cos(\mathbf{u}, \mathbf{v}) = 50 / (5 \times 10) = 1.0$（完全相同方向！$\mathbf{v}$ 就是 $\mathbf{u}$ 的 2 倍）
   - $\cos(\mathbf{u}, \mathbf{w}) = 0 / (5 \times 5) = 0$（完全正交，语义无关）

**Python 验证（3 行代码）**：

```python
import numpy as np
u, v, w = np.array([3,4]), np.array([6,8]), np.array([4,-3])
print(np.dot(u,v)/(np.linalg.norm(u)*np.linalg.norm(v)))  # 1.0
print(np.dot(u,w)/(np.linalg.norm(u)*np.linalg.norm(w)))  # 0.0
```

观察：$\mathbf{v} = 2\mathbf{u}$，所以余弦相似度为 1（方向完全一致）；$\mathbf{w}$ 和 $\mathbf{u}$ 正交（一个指东北，一个指东南，夹角恰好 90°）。

### 7. 批判性思考

**局限性**

向量假设空间是**线性**的——元素可以加、可以缩放，且加法和缩放均匀分布在整个空间里。但现实语义空间不一定是线性的：Word2Vec 的"国王-男人+女人=女王"是近似，而非精确；对于复杂语义关系（否定、反讽、模糊），纯向量点积表示能力有限。

向量表示也无法直接表达**结构关系**（如树状语法结构）。这是 GNN（图神经网络）和更复杂表示方法的动机之一。

**深度问题**

如果词嵌入向量是在高维空间里的"方向"，那么：
- 哪些方向有语义意义，哪些没有？是随机的还是有规律可循的？
- Transformer 在训练过程中如何自动调整向量，让语义相关的词靠近？（这和梯度下降、注意力机制密切相关）
- 为什么更大的词嵌入维度不一定更好？（维度灾难：高维空间里距离集中，余弦相似度区分度反而下降）

### 8. 和后续知识的连接

**已学内容的呼应**：
- **矩阵**：矩阵可以理解为一组向量排在一起（列向量组）
- **梯度**：损失函数对参数的梯度本身就是一个向量，梯度下降是向量减法
- **链式法则**：在神经网络里，反向传播计算的是高维向量空间里的方向导数

**即将在 Transformer 里再次出现**：
- **矩阵乘法（下一节课）**：矩阵乘法本质是对一批向量同时做线性变换；Q/K/V 就是对输入向量做三个不同的线性投影
- **Self-Attention**：核心操作是向量点积（Query·Key），测量每对 token 向量之间的相关性
- **位置编码**：给每个位置加一个特定向量，让模型知道词在序列中的顺序
- **词嵌入层**：Transformer 的第一步就是把每个 token 映射成一个向量（via 查找嵌入矩阵）
- **FFN（前馈层）**：对向量做两次线性变换 + 非线性激活，本质是在向量空间里做特征提取
