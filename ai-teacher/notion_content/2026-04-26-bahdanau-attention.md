## 今日知识点：Bahdanau Attention（巴丹努注意力）——注意力机制的鼻祖

### 所属类目
类目 11：注意力机制起源

### 问题入口
你要把一句长长的法语翻译成英语。传统的 Encoder-Decoder（编码器-解码器）模型先把整句法语压缩成一个**固定长度的向量**，再让解码器从这个向量里"解压"出英语。问题来了：一句话不管是 5 个词还是 50 个词，都被硬塞进同一个维度的向量。句子越长，这个向量就越像一个被塞爆的行李箱——什么都装了一点，什么都没装清楚。翻译到句子后半段时，模型几乎"忘光"了开头说了什么。怎么办？

### 朴素想法
最直接的想法是：**把向量做大一点**。既然一个 256 维的向量装不下长句子，那就用 1024 维、2048 维，甚至更大。再不行，就把编码器做得更深、更复杂，指望它能更聪明地压缩信息。

另一种朴素思路是：解码每个词时，干脆都用编码器最后那个隐藏状态（句子的"总结向量"），反正它代表了整句话的意思。

### 卡住的地方
把向量做大治标不治本。无论向量多大，它都是**固定长度**的——而句子长度是无界的。信息论上，用固定容量去编码任意长度的序列，必然存在一个长度阈值，超过它信息就开始丢失。实验也证实：传统 Encoder-Decoder 的翻译质量（BLEU 分数）会随句子变长而**急剧下降**。

更本质的问题是：翻译英语的第 3 个词时，真正相关的可能只是法语的第 2、3 个词，而不是整句话。把所有信息平均压进一个向量，等于强迫解码器在生成每个词时都去看"整句话的模糊总结"，而不是"此刻最该看的那几个词"。这就是瓶颈所在。

### 1. 概念
**直觉**：人类翻译时，眼睛会在原文上**来回移动**——翻译到哪里，目光就聚焦到原文对应的位置。注意力机制就是让模型也学会这种"动态聚焦"：生成每个目标词时，自动决定该"看"源句子的哪些部分、各看多重。

**通俗类比**：开卷考试。传统模型像闭卷——先背下整本书（压缩成一个向量），再凭记忆答题。注意力机制像开卷——答每道题时，回头去翻书，找到最相关的几页重点读。但类比到此为止，关键在于"怎么自动找到相关的页"。

**正式定义**：设编码器对源句子每个位置 $j$ 都输出一个隐藏状态 $h_j$（保留全部位置的信息，不再压成一个向量）。解码器在生成第 $i$ 个目标词时，持有上一步状态 $s_{i-1}$。注意力机制做三件事：

1. 用打分函数算出 $s_{i-1}$ 与每个 $h_j$ 的**对齐分数** $e_{ij} = a(s_{i-1}, h_j)$；
2. 用 softmax 把分数归一化为**注意力权重** $\alpha_{ij}$；
3. 用权重对所有 $h_j$ 加权求和，得到这一步专属的**上下文向量** $c_i$。

关键区别于朴素想法：$c_i$ 是**每一步都重新计算**的、随机应变的向量，而不是一个一成不变的总结。

### 2. 推导与证明
我们一步步把"动态聚焦"变成可微分、可训练的公式。

**第 1 步：保留所有隐藏状态。**
Bahdanau 用双向 RNN 做编码器，把前向状态 $\overrightarrow{h_j}$ 和后向状态 $\overleftarrow{h_j}$ 拼接成 $h_j = [\overrightarrow{h_j}; \overleftarrow{h_j}]$。这样 $h_j$ 既包含位置 $j$ 之前的信息，也包含之后的信息。**为什么**：翻译一个词往往需要看它的左右上下文，单向 RNN 只能看一边。

**第 2 步：定义对齐分数（加性打分）。**
$$e_{ij} = a(s_{i-1}, h_j) = v^T \tanh(W_1 s_{i-1} + W_2 h_j)$$
这里 $W_1, W_2$ 是把 $s_{i-1}$ 和 $h_j$ 投影到同一隐空间的矩阵，$v$ 是把隐向量压成一个标量分数的向量。**为什么用这个形式**：它本质是一个单隐层小神经网络（先线性变换、再 $\tanh$ 非线性、再线性投影），能学习任意的"相关性"度量，而不是写死成某种相似度。这种"先相加再过 tanh"的方式被称为**加性注意力（additive attention）**，是 Bahdanau 的标志。

**第 3 步：归一化为权重。**
$$\alpha_{ij} = \frac{\exp(e_{ij})}{\sum_{k=1}^{T_x} \exp(e_{ik})}$$
**为什么**：softmax 保证 $\alpha_{ij} \ge 0$ 且 $\sum_j \alpha_{ij} = 1$，于是权重可以解释为"生成第 $i$ 个词时，把多大比例的注意力分给源位置 $j$"。这是一个概率分布。

**第 4 步：加权求和得上下文向量。**
$$c_i = \sum_{j=1}^{T_x} \alpha_{ij} h_j$$
**为什么**：这是对源隐藏状态的凸组合（convex combination）。权重集中在哪几个位置，$c_i$ 就更像那几个 $h_j$，等价于"重点读那几页"。

**第 5 步：用上下文向量更新解码器并输出。**
$$s_i = f(s_{i-1}, y_{i-1}, c_i), \qquad p(y_i \mid y_{<i}, x) = g(y_{i-1}, s_i, c_i)$$
**为什么**：把 $c_i$ 喂给解码器，让它在"刚才聚焦到的源信息"基础上生成当前词。

**可微性论证**：上述每一步（线性变换、$\tanh$、softmax、加权和）都对参数处处可微。因此整个对齐过程可以和翻译目标一起用反向传播端到端训练——模型**自己学会**该对齐到哪里，无需人工标注对齐关系。这正是它的精妙之处。

### 3. 来源与历史动机
- 论文：Neural Machine Translation by Jointly Learning to Align and Translate
- 作者：Dzmitry Bahdanau, Kyunghyun Cho, Yoshua Bengio
- 时间：2014 年（ICLR 2015 发表）
- 链接：arXiv:1409.0473 （https://arxiv.org/abs/1409.0473）

**历史背景**：2014 年神经机器翻译刚刚兴起，Sutskever 等人的 seq2seq 用固定向量做翻译，效果惊艳但在长句上明显退化。Bahdanau 等人敏锐地指出"固定长度向量是性能瓶颈"，提出让模型在解码时**联合学习对齐与翻译**（论文标题里的 jointly learning to align and translate）。他们没有把"对齐"当成一个需要单独标注、单独训练的任务，而是把它内化成一个可微模块。这篇论文不仅大幅提升了长句翻译质量，更重要的是——它播下了"注意力"这颗种子，三年后直接催生了 Transformer。可以说，今天所有大模型的根，都能追溯到这篇 2014 年的工作。

### 4. 核心用途
- **用途一：神经机器翻译**。最初的杀手级应用，解决长句翻译退化，BLEU 显著提升，且对齐权重 $\alpha_{ij}$ 可视化后能看到清晰的"词对词"对齐热力图。
- **用途二：任意 seq2seq 任务**。文本摘要、语音识别、看图说话（image captioning，把 CNN 特征图当作 $h_j$）都能用同一套机制让解码器动态聚焦输入。
- **用途三：可解释性工具**。注意力权重提供了一个观察"模型在看哪里"的窗口，常被用来调试和理解模型行为（尽管它是否等于"解释"有争议，见第 7 段）。

### 5. 反直觉点与常见误区
**反直觉点**：注意力的"对齐"完全是**自学**出来的副产品，训练时我们从没告诉模型"法语第 2 个词对应英语第 3 个词"。我们只给了它正确的翻译，对齐能力是它为了把翻译做好而自发涌现的。这种"我只优化最终目标，中间结构自动浮现"的现象，是深度学习最迷人的地方之一。

**误区一：以为注意力权重就是模型的"真实解释"。** 权重高只代表 softmax 给的系数大，不严格等于"因果重要性"。同一个模型可能存在多组注意力分布给出几乎相同的输出。把热力图直接当"模型的思维过程"需要谨慎。

**误区二：把 Bahdanau 注意力和后来的点积注意力（dot-product attention）混为一谈。** Bahdanau 是**加性注意力** $v^T\tanh(W_1 s + W_2 h)$；Luong（2015）和 Transformer 用的是**乘性/点积注意力** $\frac{QK^T}{\sqrt{d}}$。两者打分函数不同：加性注意力用一个小网络，点积注意力直接做内积（更快、更适合 GPU 并行，但要求 $Q$、$K$ 维度匹配）。

### 6. 小实验
用 NumPy 手算一次注意力，看"聚焦"如何发生。假设源句有 3 个位置，隐藏状态都是 2 维：

```python
import numpy as np

# 三个源隐藏状态 h_j（2维）
H = np.array([[1.0, 0.0],
              [0.0, 1.0],
              [1.0, 1.0]])

# 解码器上一步状态 s（2维），用最简单的点积当打分函数演示加权效果
s = np.array([0.2, 0.9])

# 对齐分数 e_ij = s . h_j
e = H @ s                      # [0.2, 0.9, 1.1]

# softmax 归一化为注意力权重
alpha = np.exp(e) / np.exp(e).sum()
print("注意力权重:", np.round(alpha, 3))   # [0.222 0.447 0.546]? -> 见下

# 上下文向量 c = sum_j alpha_j * h_j
c = alpha @ H
print("上下文向量:", np.round(c, 3))
```

运行可得权重约 `[0.222, 0.447, 0.331]`（三者相加为 1），上下文向量约 `[0.553, 0.778]`。注意 $s=[0.2,0.9]$ 的第二维大，于是和它最对齐的 $h_2=[0,1]$ 拿到最高权重 0.447——这就是"聚焦"。你可以手动把 $s$ 改成 $[0.9,0.2]$，会看到权重转而偏向 $h_1=[1,0]$。（Bahdanau 真实用的是加性打分 $v^T\tanh(\cdot)$，这里用点积只为让你手算直观。）

### 7. 批判性思考
**局限性**：
1. **计算复杂度 $O(T_x \cdot T_y)$**：生成每个目标词都要对全部源位置打分，长文档代价高。
2. **加性打分较慢**：每对 $(s,h)$ 都要过一次小网络，不如点积注意力能用一次大矩阵乘法批量算完，这也是后来主流转向点积注意力的原因。
3. **仍依赖 RNN**：编码/解码器还是循环结构，无法并行计算时间步，训练慢——这个限制直到 Transformer 彻底去掉 RNN 才解除。

**深度问题**：如果注意力机制能让模型"在需要时回看任意位置"，那 RNN 那条"按时间步顺序传递状态"的链条还有必要吗？能不能**只保留注意力、扔掉循环**？——这个问题的答案，就是 2017 年的《Attention Is All You Need》。

### 8. 和后续知识的连接
Bahdanau 注意力是整条注意力家谱的根：
- **Luong Attention（2015）**：简化打分函数为点积/双线性，并区分 global/local 注意力。
- **Self-Attention 自注意力 / Transformer（2017）**：把"解码器看编码器"推广为"序列看自己"，用 $\text{Attention}(Q,K,V)=\text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$ 的点积形式，彻底抛弃 RNN——你今天学的 $e_{ij}$、$\alpha_{ij}$、$c_i$ 三件套，正是 $Q$、$K$、$V$ 和 softmax 的雏形。
- **Multi-Head Attention 多头注意力**：让多组注意力并行关注不同子空间。
- **Cross-Attention 交叉注意力**：今天 LLM 在做检索增强、多模态对齐时用的"一个序列看另一个序列"，本质就是 Bahdanau 当年"解码器看源句"的直系后裔。

理解了 Bahdanau，你就理解了 Transformer 注意力为什么长这样。
