## 今日知识点：偏导数（Partial Derivative）——微积分的多变量变化率探测器

### 所属类目
类目 2：微积分基础

### 问题入口
神经网络的损失函数 $L$ 同时依赖成千上万个参数 $w_1, w_2, \dots, w_n$。训练时我们想知道：如果只把某一个参数 $w_3$ 稍微调大一点点，损失会变好还是变坏、变多少？这是反向传播每一步都要回答的问题。可单变量微积分里的导数 $\frac{df}{dx}$ 只能处理「一个输入一个输出」的函数，面对几万个输入纠缠在一起的函数，它该怎么定义「某个方向上的变化率」？

### 朴素想法
学过一元导数的人会想：导数不就是「输出变化 ÷ 输入变化」吗？那我把所有参数打包，整体扰动一下，看损失变化除以扰动不就行了。或者更直接——既然函数有很多变量，那就把它们全都一起改，算个总的变化率。

### 卡住的地方
「全都一起改」立刻撞墙：

- **变化率不再是单一数字。** 沿不同方向走，函数升降完全不同。站在山坡上，向东可能上坡、向北可能下坡，「这一点的变化率」根本没有唯一答案，必须指定方向。
- **无法定位责任。** 损失变了，到底是 $w_3$ 的功劳还是 $w_{500}$ 的？整体扰动把所有变量的贡献混成一团，没法分别更新每个参数。

我们需要一种方法，能**单独**度量「只动一个变量、其余冻结不变」时函数的变化率。这正是偏导数。

### 1. 概念

**直觉：** 偏导数就是「把其他所有变量当成常数，只对你关心的那一个变量求普通导数」。相当于在多变量的函数曲面上，沿着平行于某一根坐标轴的方向切一刀，看那条截线的斜率。

**正式表达：** 对函数 $f(x_1, \dots, x_n)$，它对 $x_i$ 的偏导数定义为极限

$$\frac{\partial f}{\partial x_i} = \lim_{h \to 0} \frac{f(x_1, \dots, x_i + h, \dots, x_n) - f(x_1, \dots, x_i, \dots, x_n)}{h}$$

注意只有第 $i$ 个变量加了 $h$，其余原封不动。符号用花体 $\partial$（读作 "partial"）区别于一元导数的 $d$，正是为了强调「还有别的变量被按住没动」。

把所有偏导数收集成一个向量，就是**梯度**：

$$\nabla f = \left( \frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \dots, \frac{\partial f}{\partial x_n} \right)$$

**通俗类比但不止于类比：** 想象一块地形，海拔 $f$ 是经度和纬度的函数。偏导数 $\partial f/\partial x$ 是「只向正东走一步」的坡度，$\partial f/\partial y$ 是「只向正北走一步」的坡度。类比到此为止——真正关键的是：知道了这两个方向的坡度，靠下面要讲的全微分公式，你就能推算出**任意方向**的坡度，而不必每个方向都重新测一遍。这才是偏导数的威力。

### 2. 推导与证明

我们要建立从偏导数到「任意方向变化率」和「梯度下降」的桥梁。

**第 1 步：偏导数是单变量导数的特例。** 固定 $x_2,\dots,x_n$，定义单变量函数 $g(t) = f(t, x_2, \dots, x_n)$，则 $\frac{\partial f}{\partial x_1} = g'(x_1)$。
*为什么*：这说明计算偏导数不需要任何新规则，一元微积分的所有求导法则（乘法、链式）原样照搬，只是把别的变量看成常数。

**第 2 步：全微分——把各方向的贡献线性叠加。** 当 $f$ 足够光滑（可微）时，多个变量同时发生微小变化 $dx_1,\dots,dx_n$，函数的总变化近似为各偏导贡献之和：

$$df = \frac{\partial f}{\partial x_1}dx_1 + \frac{\partial f}{\partial x_2}dx_2 + \cdots + \frac{\partial f}{\partial x_n}dx_n = \nabla f \cdot d\mathbf{x}$$

*为什么*：这一步把「单独动一个」推广到「一起动」。它说明在足够小的尺度上，多变量函数的变化是**线性**的，每个变量各管一摊、互不干扰地叠加。这是反向传播能逐参数累加梯度的理论基础。

**第 3 步：方向导数与梯度方向的最优性。** 沿单位方向 $\mathbf{u}$ 的变化率为方向导数

$$D_{\mathbf{u}} f = \nabla f \cdot \mathbf{u} = \|\nabla f\| \, \|\mathbf{u}\| \cos\theta = \|\nabla f\|\cos\theta$$

其中 $\theta$ 是 $\mathbf{u}$ 与梯度的夹角。
*为什么重要*：当 $\theta = 0$（沿梯度方向）时 $\cos\theta=1$，变化率最大——**梯度方向是函数上升最快的方向**；反过来，$-\nabla f$ 是下降最快的方向。这就是**梯度下降**更新规则 $w \leftarrow w - \eta \nabla L$ 的数学根据。

**第 4 步：链式法则的多变量版本（反向传播的引擎）。** 若 $z$ 通过中间变量 $y_1,\dots,y_m$ 依赖 $x$，则

$$\frac{\partial z}{\partial x} = \sum_{j=1}^{m} \frac{\partial z}{\partial y_j}\frac{\partial y_j}{\partial x}$$

*为什么*：神经网络是层层复合函数，损失对某个深层权重的偏导无法直接写出，必须沿所有路径把局部偏导连乘再相加。这条公式就是反向传播算法的全部数学内核。

### 3. 来源与历史动机
- 论文：微积分经典概念，无单一原始论文。
- 作者：偏导数随多变量微积分发展而来，与 Leibniz、Euler、Lagrange、Clairaut 等数学家相关；现代符号 $\partial$ 的推广常归功于 Adrien-Marie Legendre 与 Carl Jacobi。
- 时间：17 世纪末微积分诞生后，18 世纪逐步成形。
- 链接：可参考任意数学分析教材，如 Walter Rudin《数学分析原理》（Principles of Mathematical Analysis）多元微分章节。

历史动机：18 世纪物理学要描述随空间和时间同时变化的量——热在金属棒上的传导、振动的弦、流体的运动。这些现象天然是多变量的，单变量导数不够用，于是需要「冻结其余变量、只看一个变量变化率」的工具，偏导数与偏微分方程应运而生，成为数学物理的通用语言。

### 4. 核心用途

**用途一：神经网络训练（反向传播 + 梯度下降）。** 损失对每个权重的偏导数 $\partial L/\partial w_i$ 告诉我们该参数往哪调、调多少。整个深度学习的优化就是反复计算这几亿个偏导数再沿负梯度更新，没有偏导数就没有训练。

**用途二：敏感性分析与特征重要性。** 偏导数衡量「输出对某个输入的局部敏感度」。在可解释性中，对输入像素求偏导得到的**显著图（saliency map）**能指出图像哪些区域最影响分类结果；金融、工程里也用它评估哪个因素对结果影响最大。

**用途三：物理建模与偏微分方程。** 扩散方程、波动方程、Navier-Stokes 方程都由偏导数搭成。科学计算、流体仿真乃至 Physics-Informed Neural Networks（PINN）都把偏导数作为基本砖块。

### 5. 反直觉点与常见误区

**反直觉点：** 一个函数在某点**所有方向**的偏导数（甚至所有方向导数）都存在，它在该点**仍可能不连续、不可微**。可微是比「偏导存在」强得多的条件——偏导只保证沿坐标轴方向有良好斜率，却不保证函数从各个角度逼近时行为一致。这常常颠覆初学者「偏导都有了函数肯定光滑」的直觉。

**误区一：以为偏导数是把所有变量一起求导。** 恰恰相反，求 $\partial f/\partial x_i$ 时其余变量全部当作**常数**。比如 $f = x^2 y$，$\partial f/\partial x = 2xy$（$y$ 当常数），$\partial f/\partial y = x^2$（$x$ 当常数）。

**误区二：把 $\partial$ 和 $d$ 当成同一个符号乱用。** 一元函数才写 $\frac{df}{dx}$；多元函数必须写 $\frac{\partial f}{\partial x}$。更进一步，全导数（total derivative）和偏导数在变量间存在依赖时数值不同，混用会在链式法则里酿成错误。

### 6. 小实验

**手算验证。** 取 $f(x, y) = x^2 + 3xy + y^2$，在点 $(1, 2)$ 处求偏导。

- 对 $x$：把 $y$ 看成常数，$\frac{\partial f}{\partial x} = 2x + 3y$，代入得 $2(1) + 3(2) = 8$。
- 对 $y$：把 $x$ 看成常数，$\frac{\partial f}{\partial y} = 3x + 2y$，代入得 $3(1) + 2(2) = 7$。

所以梯度 $\nabla f(1,2) = (8, 7)$，意味着此处沿 $(8,7)$ 方向上升最快。

**数值验证（用极限定义近似）。**

```python
def f(x, y):
    return x**2 + 3*x*y + y**2

h = 1e-6
x, y = 1.0, 2.0
df_dx = (f(x + h, y) - f(x, y)) / h   # 只动 x
df_dy = (f(x, y + h) - f(x, y)) / h   # 只动 y
print(df_dx, df_dy)   # 约 8.0 和 7.0
```

数值结果约为 `8.000003 7.000002`，与手算的 8 和 7 吻合，直接印证了「只扰动一个变量」的定义。

### 7. 批判性思考

**局限性：** 偏导数是**局部、线性**的近似，只在无穷小邻域可靠。损失曲面高度非凸时，某点梯度为零可能是局部极小、鞍点甚至平台，单凭一阶偏导无法分辨——这正是为什么深度学习还要研究二阶信息（Hessian）、动量、自适应学习率。此外，偏导数要求函数可微，而 ReLU 等在零点不可导，工程上只能用次梯度（subgradient）打补丁。

**深度问题：** 如果损失函数处处可微、梯度也处处可算，为什么单纯的梯度下降仍可能训练失败？思考方向：偏导数只给「当前点的瞬时方向」，不给「该走多远」也不给「全局地形」；步长太大越过极小、太小陷入鞍点平台，高维空间里坏条件数还会让不同方向尺度悬殊。这引出了对优化几何、预条件、二阶方法的整套研究。

### 8. 和后续知识的连接

偏导数是后续几乎所有内容的基石。**梯度、雅可比矩阵、Hessian 矩阵**都是偏导数的集合体；**反向传播**就是多变量链式法则的系统化实现；各种**优化器**（SGD、Adam）都在加工偏导数信号。理解 **Layer Normalization、Batch Normalization** 的反向传播、推导 **Attention** 对 query/key/value 的梯度、分析**梯度消失/爆炸**、乃至实现自动微分框架（PyTorch 的 autograd），无一不是在大规模、有组织地计算偏导数。可以说，掌握偏导数就是拿到了进入深度学习数学世界的门票。
