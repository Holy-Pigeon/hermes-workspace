## 今日知识点：向量（Vector）——把对象变成可计算坐标

### 所属类目
类目 1：线性代数基础

### 问题入口
我们以后讲 Transformer 时，会不断看到一句话：模型把词变成向量。

这句话听起来很抽象。为什么一句话里的“猫”“坐下”“银行”这些词，不能直接交给神经网络？因为神经网络最擅长处理的是数字，而不是文字、图片、声音这些人类感知到的对象。

真实问题是：如果我们想让机器比较两个对象是否相似、把多个特征组合起来、再通过矩阵乘法做变换，就需要先把对象放进一个可以计算的空间里。向量就是这个入口。

### 朴素想法
如果还不知道向量，我们可能会自然想到：给每个词一个编号。

比如：

- 猫 = 1
- 狗 = 2
- 银行 = 3
- 河流 = 4

这样模型终于能看到数字了，好像问题解决了。

### 卡住的地方
编号的问题是：数字大小会偷偷带入不存在的含义。

如果猫 = 1，狗 = 2，银行 = 3，那么模型可能误以为“狗比猫大 1”，“银行比猫大 2”，甚至“猫 + 狗 = 银行”。这些关系在语义上没有意义。

更麻烦的是，单个编号无法表达多个维度的信息。比如一个词可能同时有“动物性”“动作性”“金融相关性”“情绪色彩”等特征。只给一个编号，就像用一个门牌号描述一个人的身高、职业、性格和兴趣，信息太扁了。

### 1. 概念
向量可以理解为“一组按顺序排列的数字”。它通常写成：

```text
x = [x1, x2, x3, ..., xn]
```

这里：

- `x` 表示这个向量整体。
- `x1, x2, x3` 表示向量里的每一个数字，叫做分量。
- `n` 表示这个向量有多少个分量，也叫维度。

直觉上，向量不是一个孤零零的数字，而是一张“多维信息卡片”。

通俗理解：如果用一个数字描述一杯奶茶，只能说“评分 8 分”。但用向量可以写成：

```text
奶茶 = [甜度 7, 冰量 3, 茶味 8, 奶味 6, 价格 5]
```

这样我们就不只是知道它“好不好”，还知道它在哪些方面相似或不同。回到机器学习里，词向量也是类似的：每个词被表示成一组数字，模型通过这些数字进行比较、组合和变换。

### 2. 推导与证明
向量最重要的意义不是“把东西写成一串数”，而是让对象之间的关系可以被计算。

关键步骤：

1. 第一步：把对象拆成多个可比较的维度

如果我们只用一个编号表示词，词和词之间没有可靠的比较方式。但如果我们用多个维度，例如“动物相关”“金融相关”“动作相关”，就可以让每个词在这些维度上有不同分数。

例如：

```text
猫   = [0.9, 0.0, 0.2]
狗   = [0.8, 0.0, 0.3]
银行 = [0.0, 0.9, 0.1]
```

这里第一个维度表示动物相关性，第二个维度表示金融相关性，第三个维度表示动作或事件相关性。猫和狗在前两个维度上很接近，猫和银行差得很远。

2. 第二步：用距离或方向表达相似性

有了向量，我们可以比较两个对象是否接近。最直观的方法是看它们每个维度差多少。

猫和狗的差是：

```text
猫 - 狗 = [0.9 - 0.8, 0.0 - 0.0, 0.2 - 0.3]
        = [0.1, 0.0, -0.1]
```

差异很小，说明它们相似。

猫和银行的差是：

```text
猫 - 银行 = [0.9 - 0.0, 0.0 - 0.9, 0.2 - 0.1]
          = [0.9, -0.9, 0.1]
```

差异明显更大，说明它们不太相似。

3. 第三步：让模型可以通过矩阵变换这些信息

Transformer 里的每一层都在做一件事：把一批向量变成另一批更有用的向量。矩阵乘法就是常用工具。

如果 `x` 是一个词向量，`W` 是模型学出来的矩阵，那么：

```text
y = xW
```

可以理解为：模型把原来的信息卡片 `x` 重新整理成新信息卡片 `y`。后面讲 Q、K、V 矩阵时，你会看到它们本质上就是把同一个词向量投影成不同用途的向量。

### 3. 来源与历史动机
- 论文：Efficient Estimation of Word Representations in Vector Space
- 作者：Tomas Mikolov, Kai Chen, Greg Corrado, Jeffrey Dean
- 时间：2013
- 链接：https://arxiv.org/abs/1301.3781

历史背景：早期自然语言处理常把词看成离散符号，比如 one-hot 编码。one-hot 编码会给每个词一个很长的向量，只有对应位置是 1，其余位置全是 0。它能避免“编号大小”的错误含义，但仍然很难表达语义相似性，因为“猫”和“狗”的 one-hot 向量看起来和“猫”和“银行”一样不相似。

Word2Vec 这类方法的重要意义在于：它让词的含义通过训练自动落到连续向量空间里。相似词会在空间里更接近，语义关系也可能表现为方向关系。这为后来的 Transformer 词嵌入打下了基础。

### 4. 核心用途
用途一：表示输入对象

在 Transformer 里，文字不能直接进入模型。第一步通常是把 token，也就是切分后的文本单位，变成向量。比如“我喜欢猫”会先被切成若干 token，每个 token 再查表得到一个向量。

用途二：计算相似性

搜索、推荐、问答系统常把句子或文档表示成向量。用户问题和某篇文章的向量越接近，系统就越可能认为它们相关。RAG，也就是检索增强生成，就大量依赖这种向量相似性。

用途三：承载中间状态

在神经网络中，每一层都会产生新的向量。它们不只是输入表示，也像模型内部的“草稿纸”，保存当前层对词义、语法、上下文关系的理解。

### 5. 反直觉点与常见误区
反直觉点：向量里的每个维度不一定都有清楚的人类含义。

前面用“动物相关性”“金融相关性”是为了帮助理解。但真实大模型里的向量维度往往是训练自动学出来的，不一定能简单翻译成某个中文标签。它们更像许多混合特征的坐标轴。

常见误区：

- 误区一：以为向量只是“数组”。数组只是存储形式，向量的关键是它处在一个可以做加法、缩放、距离比较和线性变换的空间里。
- 误区二：以为维度越多一定越好。维度更多能容纳更多信息，但也会带来更多计算成本和过拟合风险。

### 6. 小实验
你可以不用写复杂代码，只用手算比较三个向量：

```text
猫   = [0.9, 0.0]
狗   = [0.8, 0.1]
银行 = [0.0, 0.9]
```

计算两个向量的差：

```text
猫 - 狗   = [0.1, -0.1]
猫 - 银行 = [0.9, -0.9]
```

你会发现，“猫 - 狗”的两个数字都很小，而“猫 - 银行”的两个数字都很大。这个小实验说明：向量能把“语义接近”变成“数字接近”。

如果用极短 Python，也可以这样验证：

```python
import math

cat = [0.9, 0.0]
dog = [0.8, 0.1]
bank = [0.0, 0.9]

def distance(a, b):
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))

print(distance(cat, dog))
print(distance(cat, bank))
```

第一个结果会明显小于第二个结果。

### 7. 批判性思考
局限性：向量表示很强，但它会压缩信息。一个词在不同上下文中可能有不同含义，比如“银行”可以是金融机构，也可以是河岸。单独一个固定向量很难同时完美表达这些含义。

深度问题：Transformer 为什么比早期固定词向量更强？一个关键原因是它会根据上下文动态更新向量。“苹果”在“吃苹果”和“苹果发布新手机”里，经过 Transformer 后会变成不同的上下文向量。

### 8. 和后续知识的连接
向量是后面几乎所有内容的底座。

在线性代数里，矩阵乘法会告诉我们如何批量变换向量；在神经网络里，前向传播就是向量和矩阵不断组合；在 Self-Attention 里，Q、K、V 都是从词向量变换出来的；在大语言模型应用里，向量数据库会用句子向量做语义检索。

所以今天这节课的核心不是记住“向量是一组数字”，而是理解：向量让抽象对象进入一个可以比较、组合、变换的计算空间。
