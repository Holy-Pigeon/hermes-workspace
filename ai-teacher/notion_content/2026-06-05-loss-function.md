## 今日知识点：损失函数（Loss Function）——把"模型猜得准不准"变成一个可优化的数字

### 所属类目
类目 7：反向传播算法

### 问题入口

你已经知道前向传播了：输入数据经过一层层计算，最终输出一个预测结果。

但问题来了——**模型输出了一个结果，你怎么告诉它"猜得好不好"？**

比如：
- 模型预测一张图是猫的概率是 0.3，但正确答案是"猫"。到底差多少？
- 模型预测明天气温是 25°C，实际是 22°C。这个 3°C 的误差算大还是小？

你需要一个**统一的标尺**，把"预测值和真实值之间的差距"变成**一个数字**。这个数字就是损失（Loss）。有了它，梯度下降才知道该往哪个方向调参数。

### 朴素想法

最直觉的办法：直接算"对了几个"。

比如 10 个样本，模型猜对 7 个，准确率 70%。这不就行了？

或者更简单：直接算 `预测值 - 真实值`，看差多少。

### 卡住的地方

朴素想法有三个致命问题：

**问题 1：准确率不可微。**
准确率是一个阶梯函数——预测概率从 0.49 变到 0.51，准确率突然从 0 跳到 1。梯度在大部分地方是 0，在跳变点不存在。梯度下降无法使用。

**问题 2：简单差值会正负抵消。**
如果直接用 `预测 - 真实`，一个样本偏高 +5，另一个偏低 -5，平均误差为 0。模型以为自己很完美，实际每个都猜错了。

**问题 3：不同任务需要不同"距离"定义。**
分类任务的"猜错"和回归任务的"猜错"含义完全不同。概率预测偏差 0.1 和温度偏差 0.1°C，不应该用同一个公式衡量。

所以我们需要一个**可微的、不会正负抵消的、适配任务类型的**标量函数。这就是损失函数。

### 1. 概念

**损失函数**（Loss Function）是一个映射：

$$L: (\hat{y}, y) \rightarrow \mathbb{R}$$

它把模型预测 $\hat{y}$ 和真实标签 $y$ 之间的"差距"映射为一个**非负实数**。

核心性质：
- **非负性**：$L(\hat{y}, y) \geq 0$，预测完全正确时取 0（或趋近 0）
- **可微性**：对模型参数可求导，这样梯度下降才能用
- **一致性**：预测越准，损失越小；越偏，损失越大

通俗理解：损失函数就像考试的**扣分规则**。你不是在算"得了多少分"，而是在算"被扣了多少分"。优化的目标是让扣分尽可能少。

但损失函数比普通扣分规则更聪明——它不仅告诉你"扣了多少分"，还告诉你"每道题该往哪个方向改才能少扣分"（梯度）。

### 2. 推导与证明

#### 回归任务：均方误差（MSE）

**场景**：预测连续值（温度、股价、距离）。

**推导动机**：我们需要一个不会正负抵消的误差度量。最自然的想法——先平方再求平均：

$$L_{MSE} = \frac{1}{N} \sum_{i=1}^{N} (\hat{y}_i - y_i)^2$$

关键步骤：
1. **计算差值**：$e_i = \hat{y}_i - y_i$，每个样本的偏差
2. **平方消除符号**：$e_i^2$ 保证正数，且惩罚大偏差（偏 2 的惩罚是偏 1 的 4 倍）
3. **求平均**：除以 $N$，让损失不受样本数量影响

为什么用平方而不用绝对值？因为 $|x|$ 在 $x=0$ 处不可微（左导数 -1，右导数 +1），而 $x^2$ 处处可微。

MSE 的梯度非常简洁：

$$\frac{\partial L_{MSE}}{\partial \hat{y}_i} = \frac{2}{N}(\hat{y}_i - y_i)$$

含义：偏得越多，梯度越大，参数调整的力度越猛。

#### 分类任务：交叉熵损失（Cross-Entropy Loss）

**场景**：预测离散类别（猫/狗、0-9 数字）。

**推导动机**：分类模型最后一层通常输出概率分布（通过 Softmax）。怎么衡量"预测概率分布"和"真实标签"之间的差距？

这正是信息论中交叉熵的用武之地（你在类目 4 已经学过！）。

对于 $C$ 个类别的分类，真实标签用 one-hot 向量表示：$y = [0, 0, 1, 0, ..., 0]$（假设第 3 类是正确类）。

$$L_{CE} = -\sum_{c=1}^{C} y_c \log(\hat{y}_c)$$

因为 $y$ 是 one-hot 的，只有正确类 $k$ 处 $y_k = 1$，其余都是 0，所以简化为：

$$L_{CE} = -\log(\hat{y}_k)$$

关键步骤：
1. **Softmax 归一化**：将原始输出（logits）变成概率：$\hat{y}_c = \frac{e^{z_c}}{\sum_{j=1}^{C} e^{z_j}}$
2. **取负对数**：$-\log(\hat{y}_k)$。当 $\hat{y}_k \to 1$ 时，损失趋近 0；当 $\hat{y}_k \to 0$ 时，损失趋近 $+\infty$
3. **梯度优雅**：对 logits 求导后，$\frac{\partial L}{\partial z_c} = \hat{y}_c - y_c$（Softmax + CE 的导数恰好是"预测概率 - 真实标签"，极其简洁）

#### 二分类特例：Binary Cross-Entropy（BCE）

当只有两个类时（正/负），只需要一个概率 $p$：

$$L_{BCE} = -[y \log(p) + (1-y) \log(1-p)]$$

- 当 $y=1$：损失 = $-\log(p)$，模型越确信是正例越好
- 当 $y=0$：损失 = $-\log(1-p)$，模型越确信是负例越好

### 3. 来源与历史动机

- **MSE 的渊源**：最小二乘法由高斯（Gauss, 1809）和勒让德（Legendre, 1805）独立提出，最初用于天文观测数据拟合。距今超过 200 年。
- **交叉熵的渊源**：来自香农（Shannon, 1948）的信息论。交叉熵最初用于衡量两个概率分布的差异，后被引入机器学习作为分类损失。
- **现代深度学习中的标准化**：Hinton 等人在 1980-90 年代系统地将交叉熵损失与 Softmax 结合，成为神经网络分类的标准做法。

历史背景：损失函数本质上回答了一个古老的统计学问题——**怎么定义"拟合得好"？** 高斯选择了最小化平方误差，因为它在数学上等价于假设误差服从正态分布后的最大似然估计。香农从通信编码的角度出发，给出了交叉熵这个度量。两条路径在深度学习中汇合。

### 4. 核心用途

**用途一：驱动梯度下降。** 损失函数是优化的起点。没有损失，就没有梯度，模型参数就无法更新。整个反向传播链条从损失函数开始往回走。

**用途二：选择模型和超参数。** 在验证集上比较不同模型的损失值，是模型选择的基本方法。损失曲线（training loss vs. validation loss）是诊断过拟合/欠拟合的核心工具。

**用途三：编码任务目标。** 不同的损失函数定义了"什么是好预测"。MSE 鼓励预测均值，MAE 鼓励预测中位数，交叉熵鼓励输出正确的概率分布。选择损失函数就是选择优化目标。

### 5. 反直觉点与常见误区

**反直觉点：损失函数不是越低越好。**

训练损失趋近 0 通常意味着**过拟合**——模型记住了训练数据的噪声，而不是学到了规律。真正重要的是**验证集损失**。训练损失降但验证损失升，就是在背答案。

**常见误区：**
- **误区一：分类任务用 MSE。** 虽然数学上可行（把 one-hot 和 softmax 输出算 MSE），但梯度会在预测接近 0 或 1 时变得极小（sigmoid/softmax 的饱和区），导致训练极慢。交叉熵的梯度 $\hat{y}_c - y_c$ 不会有这个问题。
- **误区二：只看损失不看指标。** 损失是优化工具，不一定直接反映业务目标。比如在不平衡分类中，损失低可能只是模型学会了"全猜多数类"。要结合 accuracy、F1、AUC 等指标一起看。
- **误区三：以为损失值可以跨任务比较。** 交叉熵损失 0.5 和 MSE 损失 0.5 完全没有可比性。即使同类损失，数据尺度不同（预测温度 vs 预测概率），数值也不可比。

### 6. 小实验

用 Python 手算感受 MSE 和交叉熵：

```python
import math

# === MSE 实验 ===
# 两个模型预测温度，真实值是 22°C
pred_a = 25  # 模型 A 预测 25
pred_b = 23  # 模型 B 预测 23
true_val = 22

mse_a = (pred_a - true_val) ** 2  # = 9
mse_b = (pred_b - true_val) ** 2  # = 1
print(f"模型 A 的 MSE: {mse_a}")  # 9
print(f"模型 B 的 MSE: {mse_b}")  # 1
# 偏 3 度的惩罚是偏 1 度的 9 倍，不是 3 倍！

# === 交叉熵实验 ===
# 三分类问题，真实类别是第 2 类（索引 1）
# 模型 A：比较确信 [0.1, 0.7, 0.2]
# 模型 B：不太确信 [0.3, 0.4, 0.3]
ce_a = -math.log(0.7)   # ≈ 0.357
ce_b = -math.log(0.4)   # ≈ 0.916
print(f"模型 A 的 CE: {ce_a:.3f}")  # 0.357
print(f"模型 B 的 CE: {ce_b:.3f}")  # 0.916
# 确信且正确 → 损失小；不确信 → 损失大

# === 感受交叉熵的极端行为 ===
print(f"预测概率 0.99 → CE: {-math.log(0.99):.4f}")   # 0.0101
print(f"预测概率 0.5  → CE: {-math.log(0.5):.4f}")    # 0.6931
print(f"预测概率 0.01 → CE: {-math.log(0.01):.4f}")   # 4.6052
# 从 0.5 到 0.01，损失暴涨！这就是 -log 的惩罚力度
```

**手算验证**：不需要电脑，你可以心算——$-\log(0.5) = \log(2) \approx 0.693$，$-\log(0.1) = \log(10) \approx 2.303$。预测概率每差一个数量级，损失大约增加 2.3。

### 7. 批判性思考

**局限性：**
- 标准损失函数假设每个样本同等重要。但在不平衡数据集中（99% 正常，1% 异常），损失被多数类主导。需要加权损失（Weighted Loss）或 Focal Loss 等变体。
- MSE 对异常值（outlier）极其敏感——一个偏差为 100 的点贡献的损失等于 10000 个偏差为 1 的点。实际中常用 Huber Loss 来兼顾敏感性和鲁棒性。
- 损失函数是人为选择的代理目标（proxy objective），不一定完美对齐真实目标。比如语言模型用交叉熵训练，但用户关心的是"回答是否有用"——这就是 RLHF 要解决的对齐问题。

**深度问题：**
- 为什么交叉熵在分类中比 MSE 好？从优化 landscape 的角度思考：交叉熵让错误预测的梯度更大，而 MSE 在 Sigmoid 饱和区梯度趋近 0。这是**损失函数和激活函数的配合问题**。
- 如果损失函数是可以选择的，那什么决定了"好的损失函数"？答案是：它应该让正确答案对应的参数空间成为"容易滚进去的坑"（良好的优化 landscape），而不仅仅是"数学上正确"。

### 8. 和后续知识的连接

- **向后连接（类目 6）**：前向传播算出预测值 $\hat{y}$，损失函数把 $\hat{y}$ 和 $y$ 的差距变成一个标量。这个标量就是反向传播的起点。
- **紧接下一步（类目 7）**：梯度下降需要知道"损失对每个参数的偏导数"，这就引出了反向传播算法（Backpropagation）——链式法则在计算图上的系统应用。
- **远期连接（类目 13-18）**：Transformer 的训练目标就是特定的损失函数。BERT 用 MLM 的交叉熵损失，GPT 用自回归的交叉熵损失。理解损失函数才能理解为什么不同预训练目标导致模型有不同的能力。
- **更远的连接（类目 20）**：RLHF 的核心动机是"交叉熵损失不能完全对齐人类偏好"，所以需要额外的奖励模型和强化学习来弥补损失函数的局限。
