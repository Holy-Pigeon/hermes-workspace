## 今日知识点：FFN 激活函数选择（ReLU → GELU → SwiGLU）——决定那台"小工厂"用什么样的开关

### 所属类目
类目 16：前馈神经网络

### 问题入口
上一课我们拆开了 Transformer 里那台"逐位置加工的两层小工厂"——前馈网络 FFN：先把每个位置的向量从维度 $d$ 升到 $4d$，过一个非线性激活，再压回 $d$。但我们一直含糊地把中间那个非线性写成"激活函数"。

真实问题是：**这个激活函数到底该选哪个？** 原始 Transformer 用的是 ReLU，BERT 和 GPT 改用了 GELU，而 LLaMA、PaLM 这些现代大模型又换成了 SwiGLU。它们看起来都是"在中间加个弯",为什么大家折腾不停？换一个激活函数，真的能让一个上千亿参数的模型变得更好吗？

### 朴素想法
最自然的想法是：**激活函数不就是为了引入非线性吗？随便挑个能"掰弯"的就行。** ReLU（$\max(0, x)$）简单、快、不会梯度消失，深度学习十年都靠它，那 FFN 里也用 ReLU，一劳永逸。

再朴素一点的人会说：既然只是为了非线性，那 sigmoid、tanh 也能掰弯，挑哪个不都一样吗？反正最后模型会自己学。

### 卡住的地方
"随便挑一个"在小网络上确实问题不大，但放到 Transformer 这个尺度就开始露馅：

1. **ReLU 在 $x<0$ 时梯度恒为 0**——这块区域的神经元一旦进去就可能"死掉"（dead ReLU），再也收不到梯度更新。在动辄几十上百层、几千亿参数的模型里，这种硬截断浪费了大量容量。
2. **ReLU 的"非黑即白"太粗暴**：输入只要小于 0 就完全清零，等于 0.001 和等于 $-100$ 被同等对待，丢掉了"接近阈值"这种细微信息。
3. **tanh / sigmoid 会饱和**：两端梯度趋近 0，深层网络里梯度一路乘下去就消失了——这正是当年 RNN 被诟病的老毛病，不可能在 Transformer 里重蹈覆辙。

换句话说：朴素想法把"激活函数"当成一个无所谓的开关，但实际上**这个开关的形状直接决定了梯度怎么流、信息怎么过、参数有没有被浪费**。在大模型尺度上，激活函数的微小差异会被放大成实打实的性能差距。

### 1. 概念

**激活函数选择**，指的是在 FFN 的中间层选用什么非线性函数，使得：①保留足够的非线性表达力；②让梯度能平滑地流过去（不死、不饱和）；③在硬件上算得快。

我们沿着历史顺序看三代主角：

**第一代：ReLU（Rectified Linear Unit）**

$$
\text{ReLU}(x) = \max(0, x)
$$

直觉：负的全部归零，正的原样保留。优点是计算极快、正区间梯度恒为 1（不消失）。缺点是负区间一刀切。

**第二代：GELU（Gaussian Error Linear Unit）**

$$
\text{GELU}(x) = x \cdot \Phi(x)
$$

其中 $\Phi(x)$ 是标准正态分布的累积分布函数（CDF），即"一个标准正态随机变量小于 $x$ 的概率"。直觉：**ReLU 是"按阈值硬性决定留不留"，GELU 是"按概率柔性决定留多少"**。当 $x$ 很大时 $\Phi(x)\to 1$，几乎原样保留；当 $x$ 很负时 $\Phi(x)\to 0$，几乎清零；在 0 附近则平滑过渡，还允许一点点负值通过。

常用的近似计算式（tanh 近似）：

$$
\text{GELU}(x) \approx 0.5 x \left(1 + \tanh\left[\sqrt{\tfrac{2}{\pi}}\left(x + 0.044715 x^3\right)\right]\right)
$$

**第三代：SwiGLU（Swish-Gated Linear Unit）**

这一代变了"玩法"——它不再是单纯一个激活函数，而是一个**门控（gating）结构**。先看它用到的 Swish（也叫 SiLU）：

$$
\text{Swish}(x) = x \cdot \sigma(x), \quad \sigma(x) = \frac{1}{1 + e^{-x}}
$$

Swish 长得跟 GELU 几乎一样平滑。SwiGLU 的关键创新是**把 FFN 的第一层拆成两条并行路径**，一条做"内容"，一条做"门":

$$
\text{SwiGLU}(x) = \big(\text{Swish}(x W_1)\big) \otimes \big(x W_3\big)
$$

这里 $\otimes$ 是逐元素相乘（element-wise），$W_1$ 和 $W_3$ 是两个独立的权重矩阵，输出再过 $W_2$ 压回去。直觉：**一条路径决定"信息内容"，另一条路径像阀门一样决定"这个信息放多少过去"**——这就是"门控"。

通俗理解：把 FFN 想成自来水管。ReLU 是个老式手动阀门，要么全开要么全关；GELU 是个能拧到任意角度的平滑阀门；SwiGLU 则是装了**两套系统**——一套调水质（内容），一套调水量（门控），两者相乘后才出水。但类比到此为止，回到机制：门控的本质是让模型对每个维度**动态地、依赖输入地**决定通过量，而不是用一个固定形状的函数一刀切。

### 2. 推导与证明

我们来推一下"为什么 GELU 比 ReLU 更平滑，以及 SwiGLU 为什么要拆成两路并保持参数量公平"。

**关键步骤：**

1. **从随机正则化推出 GELU。** 设想我们想随机地"保留或丢弃"输入 $x$：以概率 $\Phi(x)$ 保留它（乘 1），以概率 $1-\Phi(x)$ 丢弃（乘 0）。这个随机变量的**期望输出**是：
   $$
   \mathbb{E}[\text{mask} \cdot x] = \Phi(x)\cdot x + (1-\Phi(x))\cdot 0 = x\,\Phi(x)
   $$
   这正好就是 GELU。为什么用 $\Phi$？因为神经网络的输入经过归一化后近似服从正态分布，用正态 CDF 来决定"保留概率"是最自然的选择。这一步说明 GELU 不是拍脑袋造出来的，而是"带概率的 ReLU"的期望形式——它把 ReLU 的硬阈值变成了软概率。

2. **检查梯度的平滑性。** ReLU 在 $x=0$ 处不可导（左导数 0、右导数 1），GELU 处处可导且导数连续：
   $$
   \text{GELU}'(x) = \Phi(x) + x\,\phi(x)
   $$
   其中 $\phi(x)$ 是正态概率密度。注意 $x\,\phi(x)$ 这一项在 $x$ 略小于 0 时为负，使 GELU 的导数可以略大于 0 又允许少量负输出——这给了优化器更丰富的梯度信号，不会像 ReLU 那样在负区间彻底"断电"。

3. **门控为什么有用。** 普通 FFN 是 $W_2\,\sigma(W_1 x)$，非线性只作用在一个被线性变换后的标量上。SwiGLU 让输出的每个维度等于 $\text{Swish}(W_1 x)\otimes(W_3 x)$，**乘法本身就引入了输入之间的二阶交互**（一个维度可以"关掉"另一个维度），表达力比单纯叠加非线性更强。这类"门控线性单元"（GLU）在实验中持续小幅优于非门控版本。

4. **参数量要公平对比。** SwiGLU 多了一个矩阵 $W_3$，参数会变多。为了公平，实际工程里会把隐藏维度从 $4d$ 缩到约 $\tfrac{8}{3}d \approx 2.67d$，让 SwiGLU 版 FFN 的总参数量与原始 ReLU 版（$4d$）大致相等。**这是关键：SwiGLU 的优势是在"同等参数预算"下取得的，不是靠堆参数硬赢。** 这一步保证了对比的诚实性。

### 3. 来源与历史动机
- 论文一：《Gaussian Error Linear Units (GELUs)》，Dan Hendrycks & Kevin Gimpel，2016，链接：https://arxiv.org/abs/1606.08415
- 论文二：《Searching for Activation Functions》（提出 Swish/SiLU），Prajit Ramachandran et al.（Google Brain），2017，链接：https://arxiv.org/abs/1710.05941
- 论文三：《GLU Variants Improve Transformer》（提出并验证 SwiGLU），Noam Shazeer，2020，链接：https://arxiv.org/abs/2002.05202

历史背景：2017 年原始 Transformer（《Attention Is All You Need》）用的就是 ReLU。但 2018 年 BERT、GPT 不约而同换成了 GELU，因为实验显示 GELU 在 NLP 任务上稳定地略好。2020 年 Noam Shazeer（Transformer 原作者之一）系统对比了各种 GLU 变体，发现 SwiGLU 在同等参数下持续领先，并俏皮地在论文结尾写道这些方案的成功"归功于神的恩典"（divine benevolence）——意思是理论说不清为什么，但实验就是更好。此后 LLaMA、PaLM、Mistral、Qwen 等主流大模型几乎都采用了 SwiGLU。

### 4. 核心用途
用途一：**现代 LLM 的标配 FFN 激活。** 当你打开 LLaMA、Mistral、Qwen 的源码，FFN 部分基本都是 SwiGLU。理解它，你才看得懂这些模型的 `MLP` / `FeedForward` 模块为什么有三个权重矩阵（$W_1, W_2, W_3$）而不是两个。

用途二：**BERT/GPT 系列的理解。** Encoder-only（BERT）和早期 Decoder-only（GPT-2）用 GELU。读这些模型代码或微调时，激活函数的选择直接关系到你复现的精度。

用途三：**算力与精度的权衡决策。** 在资源受限（如手机端、边缘部署）时，ReLU 因为算得快、对量化友好，仍是合理选择。知道每代激活的取舍，你才能在"要精度还是要速度"之间做出正确工程判断。

### 5. 反直觉点与常见误区
反直觉点：**换一个看起来差不多的激活函数，居然能稳定提升大模型效果——而且至今没有干净的理论解释为什么。** 直觉上"非线性就是掰个弯，形状差不多就行",但在大模型尺度，激活函数的平滑度、是否允许负值、是否带门控，都会被几千亿次运算放大成可测量的性能差距。这提醒我们：深度学习里很多进步是实验驱动而非理论先行的。

常见误区：
- **误区一：以为 SwiGLU 只是"又一个激活函数"。** 不是——它是一个**门控结构**，把 FFN 第一层拆成两条并行路径再相乘，引入了输入维度之间的乘法交互。这是结构创新，不只是换函数。
- **误区二：以为 SwiGLU 更好是因为参数更多。** 公平对比时会把隐藏维度从 $4d$ 缩到约 $\tfrac{8}{3}d$，让总参数量持平。SwiGLU 是在同等预算下赢的。
- **误区三：以为 GELU 和 ReLU 在数值上差很多。** 在 $x$ 很大或很负时两者几乎重合，差异只集中在 0 附近的过渡区——但恰恰是这一小块"软过渡"带来了更顺滑的梯度。

### 6. 小实验
不用训练，手算/心算就能体会三者的差异。取几个输入值，比较输出：

对 $x = 2$：
- ReLU(2) = 2
- GELU(2) ≈ 2 × Φ(2) ≈ 2 × 0.977 ≈ 1.95
- Swish(2) = 2 × σ(2) ≈ 2 × 0.881 ≈ 1.76

对 $x = -2$：
- ReLU(−2) = 0（硬清零）
- GELU(−2) ≈ −2 × Φ(−2) ≈ −2 × 0.023 ≈ −0.046（漏出一点点负值）
- Swish(−2) = −2 × σ(−2) ≈ −2 × 0.119 ≈ −0.238（漏出更多负值）

观察：在正区间三者都接近 $x$；在负区间，ReLU 彻底归零，GELU 和 Swish 都"漏"出少量负值。正是这点"漏出"让梯度在负区间不至于完全断流。

想动手的话，几行 Python 就能画出三条曲线：
```python
import numpy as np
x = np.linspace(-4, 4, 100)
relu = np.maximum(0, x)
gelu = 0.5 * x * (1 + np.tanh(np.sqrt(2/np.pi) * (x + 0.044715 * x**3)))
swish = x / (1 + np.exp(-x))
# 用 matplotlib 画出来，会看到 GELU/Swish 在 0 附近平滑下凹，ReLU 是硬折角
```

### 7. 批判性思考
局限性：**至今没有令人满意的理论说明 SwiGLU 为何更好。** Shazeer 论文的实验提升幅度其实不大（困惑度改善个位数百分比），且依赖具体数据和规模。换句话说，它是一个稳定但温和的工程经验，不是颠覆性突破。此外门控结构增加了一个矩阵的访存和计算，在某些硬件上未必划算。

深度问题：如果激活函数的"门控"思想这么有效，那能不能把这种"用一条路径动态调节另一条路径"的思路推广到 Transformer 的其他部分？（提示：注意力机制本身就是一种"用 Query-Key 相似度动态加权 Value"的软门控——你会发现门控其实是整个 Transformer 反复出现的核心母题。）再追问：为什么大模型偏爱"平滑 + 允许少量负值"的激活？这和归一化后输入的分布、以及优化器对梯度连续性的偏好有没有关系？

### 8. 和后续知识的连接
- 这一课补全了类目 16（前馈神经网络）的"激活函数选择"。下一个知识点是同类目的**维度变换**（为什么是升到 $4d$ 再降回 $d$），届时你会看到 SwiGLU 把隐藏维度改成 $\tfrac{8}{3}d$ 的工程动机。
- "门控"思想会在你回看**注意力机制**（类目 11-12）时再次浮现——softmax 加权本质也是软门控。
- 进入**类目 18 Decoder-only 模型（GPT）**时，你会在每一个 Transformer block 里看到"注意力 + FFN"的循环，而 FFN 里用什么激活，正是区分 GPT-2（GELU）和 LLaMA（SwiGLU）的细节之一。
- 到**类目 20 大模型应用**做微调时，激活函数决定了你能否精确复现一个开源模型——选错激活，loss 就对不上。
