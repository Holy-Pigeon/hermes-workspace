# 涨价观测池（Price-Hike Watch Pool）

## 一句话定义
每日跟踪「下游资本开支驱动 / 供需缺口驱动」的产品涨价，**半年内产品涨价预期 > 20%** 的标的入池。

## 入池标准（同时满足）
1. **量化门槛**：未来 6 个月内，该产品价格的涨价预期 **≥ 20%**。
2. **逻辑来源**：涨价预期必须来自**自下而上**的可验证推导，而非单一资讯标题。优先级：
   - ① 下游资本开支(capex) → 上游需求 → 供需缺口模型测算
   - ② 供给端刚性约束（配额/减产/产能瓶颈）+ 真实价格数据（钢联/百川/财报）
   - ③ 行业研报/公司提价函等二手信息（仅作交叉验证，不单独入池）
3. **可证伪**：必须写明「证伪条件」——什么情况下该结论失效。

## 池子字段（每个标的）
| 字段 | 说明 |
|------|------|
| product | 产品名（如 HBM、钨精矿、DDR5） |
| sector | 所属板块 |
| hike_6m_pct | 半年涨价预期 %（核心门槛字段） |
| confidence | 置信度 H/M/L |
| driver | 涨价驱动逻辑（capex / 供给刚性 / 周期反转） |
| key_companies | 受益代表公司（A股优先，附代码） |
| evidence | 数据来源/锚点 |
| falsify | 证伪条件 |
| entry_date | 入池日期 |
| status | active / watching / removed |

## 每日观测动作
1. 抓取当日涨价相关资讯（东方财富资讯/研报）做**线索发现**（非结论）。
2. 对新线索 / 池内标的，用自下而上模型**重算涨价预期**。
3. 涨价预期跌破 15%（缓冲带）→ 移出池子，记 status=removed。
4. 新达标标的（≥20%）→ 入池。
5. 输出当日变动摘要（新增 / 移除 / 预期上调下调）。

## 反糊弄原则
- 资讯里的「已涨 X%」是**滞后数据**，只能当线索，不能当预测。
- 每个入池标的的 `hike_6m_pct` 必须能追溯到一个**可重算的模型或真实价格序列**。
- 数据源拿不到时，明确标注 confidence=L 并说明缺口，**绝不编造数字**。
