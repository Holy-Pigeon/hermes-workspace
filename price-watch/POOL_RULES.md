# 涨价观测池（Price-Hike Watch Pool）· v2 定性线索板

> **v2 定位变更（2026-06-16，用户拍板）**：本池**降级为「自下而上涨价逻辑的定性线索板」**，
> 不再伪装成可重算的量化模型。原 v1 要求每个标的的 `hike_6m_pct` 必须可追溯到
> 「可重算的真实价格序列（钢联/百川/财报）」——但严肃评估后确认：免费数据源（akshare）
> 拿不到商品现货的月度报价序列，钢联/百川是付费墙，自建爬虫违反「不建脆弱爬虫」原则。
> 强行保留该纪律的结果是池内标的的核心量化字段全是**估算冒充重算**＝伪精确。
> v2 坦诚放弃伪精确，把本池明确为**线索发现 + 逻辑追踪**工具，量化数字仅作粗略参考。

## 一句话定义
跟踪「下游资本开支驱动 / 供需缺口驱动」的产品涨价**逻辑线索**，半年涨价预期 > 20% 的标的入池观察。
**这是线索板，不是估值模型**——`hike_6m_pct` 是定性判断的粗略量化锚，不承诺可重算精度。

## 入池标准
1. **方向门槛**：未来 6 个月内，该产品价格的涨价预期**粗估 ≥ 20%**（量级判断，非精确测算）。
2. **逻辑来源（核心，必须有）**：涨价预期必须来自**自下而上**的可验证**逻辑链**：
   - ① 下游资本开支(capex) → 上游需求 → 供需缺口（逻辑推导，数据能拿到多少算多少）
   - ② 供给端刚性约束（配额/减产/产能瓶颈）
   - ③ 行业研报/公司提价函等二手信息（作交叉验证）
   - **逻辑链站得住 > 数字精确**。逻辑不成立的，再漂亮的涨价标题也不入池。
3. **可证伪**：必须写明「证伪条件」——什么情况下该逻辑链失效。

## 池子字段（每个标的）
| 字段 | 说明 |
|------|------|
| product | 产品名（如 HBM、钨精矿、DDR5） |
| sector | 所属板块 |
| hike_6m_pct | 半年涨价预期 %（**粗略量化锚，非可重算精度，仅供量级参考**） |
| confidence | 置信度 H/M/L（**数据缺口大时一律 L 并写明缺什么**） |
| driver | 涨价驱动逻辑（capex / 供给刚性 / 周期反转） |
| key_companies | 受益代表公司（A股优先，附代码） |
| evidence | 数据来源/锚点（**如实标注是线索还是可验证数据**） |
| falsify | 证伪条件 |
| entry_date | 入池日期 |
| status | active / watching / removed |

## 观测节奏（v2 降频：日 → 周）
> v1 是每日 agent 型 cron，但实测连续 5 天产出 0 量化更新、结论雷同——
> 涨价逻辑链是慢变量，日频是过度采样制造噪音。v2 降为**周频**，每次做一次有内容的逻辑复审。

1. 抓取近一周涨价相关资讯（东方财富资讯/研报）做**线索发现**（非结论）。
2. 对新线索 / 池内标的，复审**逻辑链是否仍成立**（不强求重算数字）。
3. 涨价逻辑链被证伪 / 预期明显跌破 → 移出池子，记 status=removed。
4. 新达标标的（逻辑链成立 + 粗估 ≥20%）→ 入池。
5. 输出当周变动摘要（新增 / 移除 / 逻辑变化）。

## 反糊弄原则（v2 强化诚实）
- 资讯里的「已涨 X%」是**滞后数据**，只能当线索，不能当预测。
- `hike_6m_pct` 是**定性判断的量级锚**，不假装是可重算模型的输出——这是 v2 最重要的诚实。
- 数据源拿不到时，明确标注 confidence=L 并说明缺口，**绝不编造数字**。
- **本池是情报采集层，非组合决策层**：发现的涨价链若要影响建仓，须经 StockChoose/stock-discovery
  独立尽调后决定，本池不直接产出买卖信号。
