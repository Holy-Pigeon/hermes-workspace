# allocation-discipline · 资本配置纪律引擎

## 它解决什么(系统级 gap,非缺某只票)
整个系统有 8+ 个**信号生成**工具(估值分位 / 已实现α / 护城河 / 相关性 /
筹码集中度 / 南向资金 / 反向DCF / 发现管线),但价投的**第二支柱——资本配置**
此前只有一次性诊断 note(研究第六篇),从未工程化、从未定期跑、从未编码
用户自陈的行为护栏(马丁/满仓负偏度教训)。

「选哪只」被 8 个工具反复回答;「钱怎么分、有没有放在 edge 最大处、有没有
违反预承诺护栏」从无可复用引擎。本项目补这个洞。

## 它做什么(纯只读,绝不下单、绝不改库)
`allocate.py`:
1. 从 `paper-trading/paper_trading.db` 读真实持仓 → 算市值权重(fx 折算)。
2. 对照**行为护栏**(预承诺参数):
   - `MAX_SINGLE_WEIGHT=0.35` 单标的封顶(🔴 对抗满仓单押)
   - `EQUAL_WEIGHT_CV=0.12` 等权惰性检测(🟡 权重雷同=从未做配置决策)
   - `MIN_NAMES=3` 有效持仓数下限(🟡 分散底线)
3. 给定信号 JSON(`{symbol: score}`)→ **信号-权重对齐诊断**:
   - edge 最强是否拿了最大权重?(🟡 排名错配)
   - 负信号是否还在重仓?(🔴 钱押在 edge 为负处)

## 用法
```
/usr/bin/python3 allocate.py --account stockchoose
/usr/bin/python3 allocate.py --account stockchoose --signals signals.json
/usr/bin/python3 allocate.py --all --quiet     # cron 友好: 仅 🔴 才 surface
```
退出码: 出现 🔴 硬告警 → 1;否则 0。`--quiet` 下慢性 🟡 不刷屏(只 🔴 出声)。

## 设计哲学
引擎**不喊买/卖**,只把「钱有没有放在 edge 最大处」和「有没有违反预承诺护栏」
算成数字 surface 给人拍板。这与 note#6 的一次性诊断互补:note 是某天快照,
本引擎是可复跑的纪律基础设施。

## 数据诚实
- 权重 = DB 一手持仓市值,不估算。
- 信号分需人工/上游工具喂入 JSON,**绝不凭印象编造**;无信号文件时只做护栏体检。
- 护栏阈值是防御性预承诺参数(可调),非预测,非买卖指令。

## 首跑硬结果(2026-06-15, stockchoose 账户)
当前 4 持仓权重 紫金27.2/小商品城26.6/工业富联25.4/康方20.8% → CV=0.101
→ 🟡 近乎等权(独立复现 note#6 核心发现:从未做主动配置决策)。
喂入示例信号后,引擎独立标出工业富联负 edge 却 25% 重仓的硬错配。

## 边界 / 待扩展
