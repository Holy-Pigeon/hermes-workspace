# 科技股价值-成长发现管线 (stock-discovery)

## 为什么存在
整个投研系统此前有 6 个工具 + 6 篇研究 note，**全部向内监控已持有的 4 只股票**
（康方/工业富联/小商品城/紫金），从无任何"向组合外发现新机会"的管线。
一个只有 4 只持仓的组合却没有候选标的漏斗——这是结构盲区。

本项目在**能力圈内的策展科技白名单**上做逐只价值-成长扫描，等优质科技股
进入自身历史便宜区。不撒网全市场（非价投，且东财全截面端口本轮持续断连），
而是盯一个人工策展的优质池。

## 数据源（全部一手 akshare，拉不到即跳过+告警，绝不编造）
- `stock_value_em`：个股历史估值序列（PE-TTM/PB/PS/PEG）→ 算自身历史分位
- `stock_financial_abstract`：多季营收/净利/ROE/毛利率 → 算单季 YoY 增速

## 判定逻辑（候选漏斗，非买卖指令）
| 标记 | 含义 | 条件 |
|------|------|------|
| ⭐ | 价值成长候选 | PE 自身史分位<35% 且 净利YoY>20% 且 ROE>12% |
| 📈 | 成长但偏贵 | 净利YoY>25% 但 PE分位>70% |
| ⚠️ | 潜在价值陷阱 | PE分位<25% 但 净利YoY<0（便宜可能有原因）|
| · | 中性 | 其余 |

⭐ 仅代表"值得深研"，**必须再做反向DCF(reverse_dcf.py)+护城河尽调+二阶验证增速质量**
才能进入候选池。单季 YoY 受基数/一次性影响大，高增速务必拆穿是否基数假象。

## 用法
```bash
python3 tech_screener.py            # 全白名单扫描
python3 tech_screener.py --quiet    # 只在有⭐候选时输出（可挂cron）
python3 tech_screener.py --symbol 688111
```

## 维护
编辑 `watchlist.json` 增删标的（覆盖代码内默认池）。入池标准：有技术壁垒/
平台或生态/数据飞轮的能力圈内科技股，非主题炒作。刻意不含已持仓标的避免重复。

## 已知局限
- 未挂 cron（发现新标的是 cadence 决策，留用户拍板；估值分位漂移慢，按需/周跑即可）
- ROE/净利为单季口径，跨周期股慎用；ROE 阈值对单季偏严（宁严勿松，⭐门槛高）
- 不替代 thesis/catalyst/reverse_dcf，是它们上游的发现漏斗
